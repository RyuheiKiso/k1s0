# エラーコードカタログ

## このドキュメントについて

全サービスが返すエラーコードの一覧。アラートやログにエラーコードが出現した際の
初動判断（どのサービスの何の問題か）と対処法の参照先をまとめている。

エラーコードの体系は [ADR-0005: エラーレスポンス形式](../adr/0005-error-response-format.md) で定義されている。

### エラーコードの命名規則

```
{TIER}_{SERVICE}_{ERROR_TYPE}
```

- **Tier プレフィックス**: `SYS_`（system）/ `BIZ_`（business）/ `SVC_`（service）
- **SERVICE**: サービスを識別する略称（例: `AUTH`, `SESSION`, `ORDER`）
- **ERROR_TYPE**: エラーの種類（例: `NOT_FOUND`, `VALIDATION_FAILED`）

### HTTP ステータスコード対応

| HTTP ステータス | エラー種別 | 典型的なコードパターン |
|----------------|----------|---------------------|
| 400 Bad Request | 不正なリクエスト | `*_QUERY_PARSE_FAILED`, `*_INVALID_QUERY` |
| 401 Unauthorized | 認証失敗 | `SYS_AUTH_INVALID_TOKEN`, `SYS_AUTH_MISSING_CLAIMS`, `SYS_SESSION_EXPIRED` |
| 403 Forbidden | 権限不足 | `*_PERMISSION_DENIED`, `*_ACCESS_DENIED`, `*_FORBIDDEN` |
| 404 Not Found | リソース未存在 | `*_NOT_FOUND` |
| 409 Conflict | 競合・重複 | `*_ALREADY_EXISTS`, `*_VERSION_CONFLICT`, `*_CONFLICT` |
| 422 Unprocessable | バリデーション失敗 | `*_VALIDATION_FAILED`, `*_VALIDATION_ERROR`, `*_INVALID_STATUS_*` |
| 429 Too Many Requests | レート超過 | `*_RATE_EXCEEDED`, `*_QUOTA_EXCEEDED`, `*_MAX_*_EXCEEDED` |
| 500 Internal Error | 内部エラー | `*_INTERNAL_ERROR`, `*_EVALUATION_FAILED`, `*_EXECUTION_FAILED` |
| 502 Bad Gateway | 上流エラー | `*_UPSTREAM_ERROR`, `*_MODEL_REQUEST_FAILED`, `*_SEND_FAILED` |
| 503 Service Unavailable | サービス不可 | `*_STORAGE_ERROR` |

## System Tier (SYS_*)

### auth-server（認証・認可）

| エラーコード | HTTP | 説明 | 初動対応 |
|------------|------|------|---------|
| `SYS_AUTH_INVALID_TOKEN` | 401 | JWT トークンの形式・署名が不正 | Keycloak/公開鍵確認。[運用FAQ Q1](../observability/運用FAQ.md#q1) |
| `SYS_AUTH_MISSING_CLAIMS` | 401 | JWT に必要な claims が不足 | クライアントのスコープ設定確認 |
| `SYS_AUTH_PERMISSION_DENIED` | 403 | ロール・権限が不足 | Keycloak のロール割当確認。[運用FAQ Q2](../observability/運用FAQ.md#q2) |
| `SYS_AUTH_NOT_FOUND` | 404 | ユーザー・ロールが未登録 | Keycloak 管理画面でユーザー存在確認 |
| `SYS_AUTH_VALIDATION_FAILED` | 422 | リクエストのバリデーション失敗 | リクエストパラメータの値を確認 |
| `SYS_AUTH_INTERNAL_ERROR` | 500 | auth サーバー内部エラー | auth-server のログ確認。Runbook: [エラー率高騰](../observability/runbooks/common/high-error-rate.md) |

## エラーコードカタログのメンテナンス

### 更新方法

新しいサービスやエラーコードが追加された場合は、以下の手順で更新する:

1. 該当サービスの `src/domain/error.rs` の `From<XxxError> for ServiceError` 実装を確認
2. 新しい `ErrorCode::new("...")` 呼び出しを本カタログに追加
3. HTTP ステータスコードは `ServiceError` の変換先バリアントから導出（`NotFound` → 404 等）

### 自動検索コマンド

```bash
# 全サービスのエラーコードを一覧表示
grep -r 'ErrorCode::new("' regions/ \
  --include='error.rs' \
  -h | grep -oP '"[^"]+"' | sort | uniq
```
