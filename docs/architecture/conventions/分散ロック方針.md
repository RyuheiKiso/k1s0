# 分散ロック方針

## 概要

k1s0-distributed-lock ライブラリ（`regions/system/library/rust/distributed-lock/`）が提供する
3種類のバックエンドの使い分けと、ロック設計のベストプラクティスを定義する。

## バックエンド選択フロー

```
テスト環境か?
  YES → InMemoryDistributedLock を使用する
  NO  → ロック頻度が高い、または DB 接続がないか?
          YES → RedisDistributedLock を使用する
          NO  → PostgresDistributedLock を使用する（既存 DB 接続を再利用）
```

## バックエンド比較

| バックエンド | 実装方式 | 精度 | 推奨シナリオ | 前提条件 |
|------------|---------|------|------------|---------|
| **InMemory** | `Mutex` + `HashMap` | プロセス内のみ | テスト、単一インスタンス | なし |
| **PostgreSQL** | Advisory Lock | DB トランザクション | 中〜低頻度、既存 DB 環境 | PostgreSQL 接続 |
| **Redis** | SET NX + Lua スクリプト | ミリ秒単位 | 高頻度、低レイテンシ要求 | Redis クラスタ |

## DistributedLock トレイト API

```rust
// k1s0-distributed-lock の DistributedLock トレイト定義
pub trait DistributedLock: Send + Sync {
    // ロックを取得する（取得できるまでブロックまたはタイムアウト）
    async fn acquire(&self, key: &str, ttl: Duration) -> Result<LockGuard, LockError>;
    // ロックを解放する（LockGuard の RAII で自動解放も可能）
    async fn release(&self, guard: LockGuard) -> Result<(), LockError>;
    // ロックの TTL を延長する（長時間処理でのロック継続）
    async fn extend(&self, guard: &LockGuard, ttl: Duration) -> Result<(), LockError>;
    // ロックが保持されているか確認する
    async fn is_locked(&self, key: &str) -> Result<bool, LockError>;
}
```

## ロックキー命名規約

```
{service}:{resource-type}:{resource-id}
```

| 例 | 説明 |
|----|------|
| `inventory:stock:item-123` | 在庫アイテム 123 の更新ロック |
| `saga:workflow:order-456` | Saga ワークフロー order-456 の実行ロック |
| `scheduler:job:daily-report` | 日次レポートジョブのロック |
| `config:reload:service-abc` | サービス ABC の設定リロードロック |

## タイムアウトポリシー

| 用途 | 推奨 TTL | 根拠 |
|------|---------|------|
| 短期処理（在庫更新など） | 10秒 | タイムアウト後の自動解放で安全性を確保 |
| 中期処理（ファイルエクスポートなど） | 60秒 | 処理時間に余裕を持たせる |
| Saga ステップ | 30秒（workflow YAML で定義） | `step.timeout_secs` パラメータと合わせる |
| 最大上限 | 300秒 | これ以上は設計を見直すこと |

## 使用パターン

```rust
// 正常なロック使用パターン: スコープベースで RAII 管理する
async fn reserve_stock(
    lock: &dyn DistributedLock,
    item_id: &str,
) -> Result<(), Error> {
    // ロックキーはサービス名:リソース種別:リソースID の形式で命名する
    let lock_key = format!("inventory:stock:{}", item_id);
    // 10秒 TTL でロックを取得する（タイムアウト時は LockError::Timeout を返す）
    let _guard = lock.acquire(&lock_key, Duration::from_secs(10)).await?;
    // _guard がスコープを抜けると自動的にロックが解放される（RAII）
    // ここで排他的な処理を実行する
    Ok(())
}
```
