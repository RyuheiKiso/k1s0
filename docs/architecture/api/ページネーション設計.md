# ページネーション設計

本ドキュメントは k1s0 プロジェクトにおける統一ページネーションパターンを定義する。

---

## 概要

全ての一覧取得 API（gRPC / REST / GraphQL）で統一されたページネーション方式を採用する。
オフセットベースのページネーションを標準とし、Proto の共通メッセージ型を通じて一貫性を確保する。

## Proto 共通メッセージ型

### リクエスト: `Pagination`

パッケージ: `k1s0.system.common.v1`

| フィールド | 型 | デフォルト | 説明 |
|-----------|-----|-----------|------|
| `page` | int32 | 1 | ページ番号（1 始まり） |
| `page_size` | int32 | 20 | 1 ページあたりの件数（1〜100） |

### レスポンス: `PaginationResult`

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `total_count` | int64 | 全件数 |
| `page` | int32 | 現在のページ番号 |
| `page_size` | int32 | 1 ページあたりの件数 |
| `has_next` | bool | 次ページが存在するか |

---

## バリデーションルール

| パラメータ | ルール | エラーメッセージ |
|-----------|--------|--------------|
| `page` | >= 1 | `page must be >= 1` |
| `page_size` | 1 <= x <= 100 | `page_size must be between 1 and 100` |

バリデーションはユースケース層で実施する。インフラ層（リポジトリ）に不正な値が到達しないようにする。

---

## REST API マッピング

```
GET /api/v1/{resource}?page=1&page_size=20&search=keyword
```

レスポンス:
```json
{
  "data": [...],
  "pagination": {
    "total_count": 42,
    "page": 1,
    "page_size": 20,
    "has_next": true
  }
}
```

---

## GraphQL マッピング

```graphql
query {
  configs(namespace: "system.auth", page: 1, pageSize: 20) {
    entries { key value updatedAt }
    pagination { totalCount page pageSize hasNext }
  }
}
```

## 実装パターン

### ユースケース層

```rust
// バリデーション → リポジトリ呼び出し → 結果返却
if params.page < 1 {
    return Err(ValidationError("page must be >= 1"));
}
if params.page_size < 1 || params.page_size > 100 {
    return Err(ValidationError("page_size must be between 1 and 100"));
}
```

### リポジトリ層

```rust
// OFFSET/LIMIT 計算
let offset = (page - 1) * page_size;
let limit = page_size;
```
