# ADR-0046: Tauri GUI 実装の延期

## ステータス

承認済み

## コンテキスト

k1s0 プロジェクトには Tauri 2 ベースのデスクトップ GUI アプリケーション設計書（`docs/cli/gui/TauriGUI設計.md`）が存在する。この設計書では Tauri + React + TypeScript を組み合わせた GUI の全機能仕様が定義されており、README にも「Tauri デスクトップ GUI」として言及されている。

しかし現時点では、CLI バイナリ（`CLI/crates/k1s0-cli/`）と Rust コア（`CLI/crates/k1s0-core/`）の実装は進んでいるが、Tauri GUI の実装コードは一切存在しない。外部技術監査において、設計書のみが存在し実装が未着手であることが指摘された。

## 決定

Tauri デスクトップ GUI の実装を**延期**する。現時点では Rust 製対話式 CLI（`k1s0-cli`）のみを提供し、Tauri GUI は計画中の機能として扱う。

## 理由

1. **コアサーバー群の安定化を優先**: System Tier の Rust サーバー群（auth, config, saga 等）は継続的な監査対応・機能拡張中であり、GUI 開発のリソースを割り当てる段階ではない。
2. **実験的機能**: Tauri GUI は実験的な拡張機能であり、設計書のみが存在する段階では実装コストに見合う確証が取れていない。
3. **CLI の機能完全性**: Rust 製の対話式 CLI（dialoguer）は既に全機能を提供しており、ターミナル操作に慣れた開発者向けには十分な開発者体験を実現している。
4. **技術的リスク管理**: Tauri 2 + React 19 の組み合わせは比較的新しく、エコシステムの安定性を見極める期間が必要。

## 影響

**ポジティブな影響**:

- コアサーバー・CLI の開発リソースを集中できる
- 設計書が存在することで将来の実装開始時のコストを削減できる
- README に「計画中・実装未着手」の注記を追加することで利用者への誤解を防止できる

**ネガティブな影響・トレードオフ**:

- ターミナル操作に不慣れなユーザーへの対応が遅れる
- README に言及があることで実装済みと誤解されるリスクがあった（注記追加で解消）

## 代替案

| 案 | 概要 | 採用しなかった理由 |
|----|------|-----------------|
| 案 A: 即時実装 | Tauri GUI を今すぐ実装する | コアサーバーの安定化が優先であり、GUI 開発のリソースを確保できない |
| 案 B: 設計書削除 | TauriGUI設計.md を削除して計画自体を破棄する | 将来の実装可能性を残すため設計書は維持する |
| 案 C: 延期（本決定） | 設計書は維持しつつ、README に「計画中・実装未着手」を明記する | 採用 |
