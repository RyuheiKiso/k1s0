# ADR-0091: JWT Token Introspection ハイブリッド方式

## ステータス

承認済み

## コンテキスト

現在の JWT 検証はローカルで JWKS 公開鍵検証のみを行っている。Keycloak 側でトークンが失効（管理者強制ログアウト・セッション無効化等）した場合でも、JWT の有効期限が切れるまでの最大 5 分間、ローカル検証ではトークンが有効と判定される。

この問題は特に管理者操作・設定変更系のエンドポイントで重大なセキュリティリスクとなる。侵害されたトークンが管理者によって無効化された後も、TTL 内であればそのトークンを使用した操作が続行できてしまう。

外部技術監査 H-002 において当該リスクが指摘された。

## 決定

ハイブリッド方式を採用する。

- **通常 API**: JWT ローカル検証のみ（JWKS 公開鍵による署名検証）。パフォーマンスを重視する。
- **高価値エンドポイント**: JWT ローカル検証に加えて、Keycloak の Token Introspection エンドポイント（RFC 7662）に問い合わせる。対象は管理者操作・設定変更系のエンドポイント。

実装は以下の 3 フェーズで進める。

- **Phase 1**: 本 ADR の策定
- **Phase 2**: auth ライブラリ（`regions/system/library/go/auth`）に Token Introspection 機能を追加する
- **Phase 3**: 各サービスの高価値エンドポイントに introspection ミドルウェアを適用する

## 理由

Token Introspection を全 API に適用した場合、すべてのリクエストが Keycloak への HTTP 往復を必要とし、Keycloak が単一障害点となる。負荷集中によるパフォーマンス劣化・可用性リスクが高く、採用は困難である。

リスクベースアプローチにより、セキュリティ上のリスクが高い操作（管理者操作・設定変更系）にのみ Token Introspection を適用することで、最小限の実装コストで最大のセキュリティ効果を得られる。

管理者強制ログアウト等の高価値操作の即時無効化は業務上の要件であり、通常の閲覧系 API では 5 分以内の失効猶予は許容範囲と判断する。

## 影響

**ポジティブな影響**:

- 高価値エンドポイントにおける侵害トークンの悪用リスクを即時排除できる
- 管理者が強制ログアウトした後、そのセッションが即座に無効化される
- 通常 API のパフォーマンスは現状維持される
- RFC 7662 標準に準拠した実装となる

**ネガティブな影響・トレードオフ**:

- 高価値エンドポイントへのリクエストに Keycloak との往復レイテンシが追加される（通常 1〜10ms）
- Keycloak が停止した場合、高価値エンドポイントが利用不能となる（フェイルクローズ設計）
- introspection 対象エンドポイントの定義・メンテナンスコストが発生する

## 代替案

| 案 | 概要 | 採用しなかった理由 |
|----|------|-----------------|
| 全エンドポイントに Token Introspection 適用 | すべての API リクエストで Keycloak に問い合わせる | Keycloak への負荷が高く単一障害点となる。スループットが大幅に低下する |
| JWT の TTL を短縮（例: 60 秒） | JWT 有効期限を短くして失効リスクを小さくする | リフレッシュ頻度が上がり UX が悪化する。ネットワーク負荷も増加する |
| 現状維持（ローカル検証のみ） | 変更なし | 管理者強制ログアウト後も最大 5 分間、侵害トークンが有効となるリスクを放置することになる |
