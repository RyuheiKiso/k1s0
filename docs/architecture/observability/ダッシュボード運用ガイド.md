# ダッシュボード運用ガイド

Grafana ダッシュボードの配置ルール、更新手順、トラブルシューティング時の活用方法を定義する。

---

## ダッシュボード配置ルール

プロジェクト内にダッシュボードが以下の3箇所に存在する。**用途が異なるため**、混同しないこと。

### 本番・ステージング用（正規）

```
infra/observability/grafana/dashboards/
├── overview.json          # 全サービス概要（RED メトリクス一覧）
├── slo.json               # SLO / エラーバジェット
├── kafka.json             # Kafka トピック別スループット・ラグ
├── kong.json              # API Gateway (Kong) のリクエスト統計
├── database.json          # PostgreSQL クエリ統計・接続数
└── istio.json             # Istio サービスメッシュ
```

**Kubernetes 環境に自動プロビジョニングされる正規のダッシュボード。**
変更はこちらのファイルを編集する。

### ローカル開発用

```
infra/docker/grafana/dashboards/
├── k1s0-overview.json     # 全体概要（ローカル向け簡易版）
├── k1s0-service.json      # サービス詳細（ローカル向け）
└── system-overview.json   # system Tier 概要
```

**Docker Compose でのローカル開発環境専用。**
本番への適用は行わない。

### デモ・検証用

```
infra/demo/kiali/dashboards/
└── （13ファイル）
```

**デモンストレーション・検証目的のみ。**
本番・ローカル開発環境に適用しないこと。

---

## ダッシュボード一覧と用途

### 本番環境用ダッシュボード

| ファイル | Grafana タイトル | 主な用途 | 主要パネル |
|---------|----------------|---------|-----------|
| `overview.json` | k1s0 Overview | インシデント発生時の全体把握 | リクエスト率、エラー率、レイテンシ（全サービス） |
| `slo.json` | SLO Dashboard | エラーバジェット確認・月次レビュー | バーンレート、エラーバジェット残量、可用性 |
| `kafka.json` | Kafka Dashboard | Kafka コンシューマーラグ確認 | Consumer Lag、スループット |
| `kong.json` | Kong Dashboard | API ゲートウェイの状況確認 | リクエスト数、エラー率、レイテンシ |
| `database.json` | Database Dashboard | DB 状況確認・スロークエリ調査 | 接続数、プール使用率、クエリ時間 |
| `istio.json` | Istio Dashboard | サービスメッシュの通信状況確認 | サービス間トラフィック、エラー率 |

## ダッシュボード更新手順

### 手順 1: Grafana UI で編集

1. Grafana にアクセス（ローカル: http://localhost:3200、本番: http://grafana.k1s0.internal）
2. 編集したいダッシュボードを開く
3. 編集モードに切り替えてパネルを修正
4. 変更内容を確認

### 手順 2: JSON をエクスポート

1. ダッシュボードの設定アイコン（歯車）→ JSON Model を開く
2. JSON をコピー
3. 対応するファイルに貼り付けて保存

```bash
# 本番用ダッシュボードを更新する場合
# infra/observability/grafana/dashboards/{name}.json を編集

# ローカル用ダッシュボードを更新する場合
# infra/docker/grafana/dashboards/{name}.json を編集
```

### 手順 3: プロビジョニング確認

```bash
# ローカル環境でのプロビジョニング確認
docker compose restart grafana
# → Grafana を再起動してダッシュボードが反映されることを確認

# Kubernetes 環境
kubectl rollout restart deployment/grafana -n observability
```

### 注意事項

- ダッシュボード JSON 内の `id` と `uid` フィールドを変更しないこと（URL と参照が壊れる）
- バージョン管理のため、変更は必ず git でコミットする

## CLI によるダッシュボード自動生成

新規サービス追加時は CLI でダッシュボードのテンプレートを生成できる:

```bash
k1s0 generate grafana --service-name {service-name} --tier {tier}
```

生成されるファイル:
- `infra/observability/grafana/dashboards/{service-name}-overview.json`
- `infra/observability/grafana/dashboards/{service-name}-slo.json`

生成後は Grafana に取り込んで動作確認すること。

## トラブルシューティング時のダッシュボード活用

### インシデント対応時のダッシュボード活用フロー

```
[アラート発火]
    │
    ▼
[1. Overview ダッシュボードで全体把握]
  → どのサービスでエラー率・レイテンシが上昇しているか確認
    │
    ▼
[2. Service Detail ダッシュボードで詳細調査]
  → 特定サービスのエンドポイント別メトリクスを確認
    │
    ▼
[3. SLO ダッシュボードでエラーバジェット確認]
  → SLA 違反リスクを評価
    │
    ▼
[4. Database / Kafka ダッシュボードで依存確認]
  → DB プール使用率、Kafka ラグが原因かを確認
```

### よく使うダッシュボードリンク（本番環境）

| 用途 | URL |
|------|-----|
| 全体概要 | http://grafana.k1s0.internal/d/k1s0-overview |
| SLO 確認 | http://grafana.k1s0.internal/d/k1s0-slo |
| Kafka 確認 | http://grafana.k1s0.internal/d/k1s0-kafka |
| DB 確認 | http://grafana.k1s0.internal/d/k1s0-db |

### よく使うダッシュボードリンク（ローカル環境）

| 用途 | URL |
|------|-----|
| 全体概要 | http://localhost:3200/d/k1s0-overview |
| サービス詳細 | http://localhost:3200/d/k1s0-service |
