# 90. 付録

本カテゴリは監査・稟議・外部説明時のエビデンスとなる支援文書を格納する。設計本体ではなく、設計が何の標準に準拠しているかを証明する層である。

## 本カテゴリの位置付け

JTC の稟議レビュアーや外部監査人は、設計書の技術内容を直接検証する代わりに、「標準準拠を証明する対応表があるか」「設計判断の根拠となる ADR に辿れるか」「参照した文献が信頼できるか」を確認する傾向がある。本カテゴリはこの種の参照要求に対応するエビデンス層である。

要件定義書側の付録（[../../03_要件定義/90_付録/](../../03_要件定義/90_付録/)）が共通フレーム対応表・非機能要求グレード判定・参考文献を格納しているのと同様、本カテゴリは概要設計フェーズの対応表・ADR 索引・参考文献を格納する。

## 格納予定ドキュメント

3 ファイルを格納する。

- `01_共通フレーム2013対応表.md` — IPA 共通フレーム 2013 の設計プロセス（6.4.3 システム方式設計 / 7.1.2 ソフトウェア方式設計 / 6.4.10 移行プロセス）と本概要設計書の章立ての対応を明示する。各アクティビティが本書のどのファイルでカバーされるか、一対一で対応付ける。要件定義書側の同名ファイル（[../../03_要件定義/90_付録/01_共通フレーム2013対応表.md](../../03_要件定義/90_付録/01_共通フレーム2013対応表.md)）と重複しないよう、本書では設計プロセスのみを対象とする。
- `02_ADR索引.md` — 構想設計 [../../02_構想設計/adr/](../../02_構想設計/adr/) の既存 ADR と本概要設計の章対応を索引化する。各章が参照している ADR 番号、概要設計フェーズで新規起票された ADR、既存 ADR からの逸脱があれば記録する。
- `03_参考文献.md` — 概要設計で参照した IPA 原典・RFC・ベンダー資料・OSS 仕様書の一覧。URL とアクセス日付を併記し、監査時の再現性を担保する。

## 執筆方針

- 対応表は章立て変更時に必ず同期改訂する。対応表が設計本体とずれると、監査対応時に混乱を招く。
- ADR 索引は ADR 番号・タイトル・関連章の 3 列を最低限含める。ADR 発行日と承認者は構想設計側の原本に委ね、本索引では重複管理しない。
- 参考文献はアクセス日付を必ず記録する。IPA 資料などはバージョン更新で URL が変わるため、アクセス日を証跡として残す。

## 他カテゴリとの関係

- **横断**: 本カテゴリは他の全カテゴリから参照される共通基盤であり、他章の執筆者は本カテゴリの対応表・ADR 索引・参考文献を参照しながら執筆する。
- **上流**: [../../03_要件定義/90_付録/](../../03_要件定義/90_付録/) と対になる構造であり、稟議時は両者をセットで提示する。
- **下流**: 詳細設計フェーズで、本カテゴリの対応表をベースに詳細設計プロセス対応表が追加される想定。
