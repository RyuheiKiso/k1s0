# 03. 参考文献

本ファイルは概要設計書で参照した公的規格・OSS 公式ドキュメント・業界標準・法令・学術文献の一次情報源を列挙する参考文献リストである。設計判断の根拠を外部一次情報に遡って確認できる状態を保つことで、監査・外部レビューの場面でのエビデンスとして機能させる。

## 本ファイルの位置付け

概要設計書は内部文書だけで自己完結することは原則できない。共通フレーム 2013 の準拠を主張する以上は IPA の原典を引用し、OSS の動作仕様を前提とする以上は OSS 公式ドキュメントを引用し、法令遵守を謳う以上は法令条文を引用する必要がある。これらの引用元が本ファイルで一元管理されていないと、採用検討で「なぜそう判断したか」を問われた際に資料散逸が発生する。

本ファイルは引用元を以下 5 カテゴリに分類し、各文献について「発行元」「版・年」「アクセス日」「参照箇所」の 4 要素を付記する。オンライン文献は URL を付記し、アクセス日を全項目に付与する。URL は恒久的に維持される保証がないため、改訂時に死リンクチェックを実施する。

- **公的規格**（IPA / ISO / IEC 等）
- **OSS 公式ドキュメント**（Dapr / Kubernetes / Istio 等）
- **業界標準**（OWASP / CNCF / CIS 等）
- **法令**（個人情報保護法 / GDPR / J-SOX / 電帳法 等）
- **学術文献**（Saga Pattern 原典、分散システム理論 等）

本章は索引層のため設計 ID を採番しない。

## アクセス日の維持

本ファイルに記録される URL は恒久的に維持される保証がない。改訂時に以下 3 ルールでアクセス日と URL 有効性を維持する。

- **四半期死リンクチェック**: 四半期レビューで全 URL の HTTP 200 応答を CI ジョブで確認。404 / 5xx / タイムアウトのいずれかが発生した URL は代替 URL（Web Archive 等）を探索する。
- **アクセス日更新**: 文献内容を実際に参照した場合、アクセス日列を更新する。参照なしに URL のみ確認する場合はアクセス日を更新しない。
- **版追随**: OSS 公式ドキュメントは常にその時点での最新安定版を参照対象とする。参照対象版を変更した場合、版列を更新し、必要に応じて概要設計本文を再確認する。

## 参考文献の分類基準

文献を 5 カテゴリに分類する基準は以下である。分類の判定が曖昧な場合は、主用途で判定する。

- **公的規格**: 政府機関・標準化団体が発行し、準拠が調達・監査の基準となる文献。
- **OSS 公式ドキュメント**: 採択 OSS の開発元または公式コミュニティが発行する一次情報。
- **業界標準**: 業界団体・セキュリティコミュニティが発行するベストプラクティス集。法的拘束力はないが事実上の基準。
- **法令**: 法的拘束力を持つ条文。遵守が必須。
- **学術文献**: 査読付き論文・書籍。設計パターンや理論的根拠として参照。

同一文献が複数カテゴリに該当する場合（例: NIST 文書は公的規格であると同時に業界標準としても機能する）は、主用途のカテゴリに分類し、他カテゴリの注記を参照箇所列に記載する。

## 引用書式

概要設計書本文から本ファイルを引用する際の書式は以下で統一する。引用書式の統一は監査時の機械的な整合確認を容易にする。

- **規格引用**: `IPA 共通フレーム 2013, §6.4.3.1（参考文献 公的規格 1 番目）`
- **OSS 引用**: `Dapr v1.14 公式ドキュメント, Service Invocation API（参考文献 OSS ドキュメント Dapr）`
- **法令引用**: `個人情報保護法 第 17 条（参考文献 法令 1 番目）`
- **学術文献引用**: `Sagas, Garcia-Molina et al. 1987（参考文献 学術 1 番目）`

引用文中の章節番号・条文番号・ページ番号は一次情報源の番号をそのまま使用し、意訳番号は禁止する。

## 運用ルール

本参考文献リストの継続運用は以下の 4 ルールで固定化する。

- **文献追加タイミング**: 概要設計書本文で新規引用が必要となった時点で、同じ PR で本ファイルに追加する。引用先未登録のまま本文を追加することは禁止する。
- **文献削除タイミング**: 概要設計書本文から引用が削除された場合でも、本文献リストからは削除せず、「参照箇所」列を空欄に書き換える。削除履歴は Git commit で追跡可能な状態を保持する。
- **四半期再チェック**: 四半期レビューで死リンクチェック・版追随を実施する。
- **監査対応**: 監査で本ファイルから引用された場合、参照箇所列の一致を監査チェックリストで確認する。
