# k1s0 紹介ドキュメント

クラウドネイティブ・アプリケーションプラットフォーム **k1s0** の紹介資料群。k1s0 は起案者が業務外で個人的に開発している OSS であり、Apache 2.0 ライセンスで GitHub に公開している。会社（起案者の雇用主）が業務システムとして利用しているが、k1s0 は会社の予算・工数で開発されたものではなく、会社が自発的に採用しているという関係にある。

最上位の紹介資料は [`企画書.md`](./企画書.md)（Marp スライド）、技術深掘りは [`../02_構想設計/`](../02_構想設計/)、要件化は [`../03_要件定義/`](../03_要件定義/) を参照する。

---

## 本ディレクトリの位置づけ

`01_企画/` は **k1s0 とは何か・なぜ作ったか・どう作られているか** を、初見の読み手が掴むための紹介資料を配置する。技術選定や内部アーキテクチャの詳細は [`../02_構想設計/`](../02_構想設計/) で扱う。

紹介の論点は「**なぜ作るか・既存品で代替できないか・どう運用するか・どう責任を分けるか**」の 4 点に絞っている。

---

## 読み方

初読時は以下の順で読むと k1s0 の全体像が掴める。

1. [`企画書.md`](./企画書.md) — 10 分で読み切れる紹介資料（Marp スライド）
2. [`01_背景と目的/`](./01_背景と目的/) — 何を解決するか、誰のためか、なぜ個人開発の OSS として作るか
3. [`02_競合と差別化/`](./02_競合と差別化/) — 既存製品との比較と k1s0 の設計特性
4. [`05_法務サマリ/`](./05_法務サマリ/) — Apache 2.0 ライセンス・AGPL 依存 OSS の構成・知財・監査
5. [`全体構成図.md`](./全体構成図.md) — インフラ＋アーキテクチャの 1 枚俯瞰図（技術面の概観）

技術深掘り（アーキテクチャ・tier1 設計・OSS 選定・CI/CD・法務）は [`../02_構想設計/`](../02_構想設計/) を参照。
