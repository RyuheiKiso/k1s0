# k1s0 企画ドキュメント

JTC (Japanese Traditional Company) 情報システム部門向けマイクロサービス基盤プラットフォーム **k1s0** の **稟議向け企画資料群**。
最上位の提案書は [`企画書.md`](./企画書.md) (Marp スライド)、技術深掘りは [`../02_構想設計/`](../02_構想設計/)、要件化は [`../03_要件定義/`](../03_要件定義/) を参照する。

---

## 本ディレクトリの位置づけ

`01_企画` は **決裁者・稟議レビュアーが判断するための最小構成** に絞った章のみを配置する。技術選定や内部アーキテクチャの詳細は稟議通過後に着手する設計領域であり、[`../02_構想設計/`](../02_構想設計/) に分離してある。

決裁に必要な論点を過不足なく揃えることを優先し、「なぜ作るか」「既存品で代替できないか」「いつ誰がどれだけの費用で作るか」「走り出した後に何で測るか」の 4 点に絞った構成とする。

---

## 読み方

初読時は以下の順で読むと稟議判断に必要な情報が漏れなく掴める。

1. [`企画書.md`](./企画書.md) — 10 分で読み切れるエグゼクティブサマリ (Marp スライド)
2. [`01_背景と目的/`](./01_背景と目的/) — 何を解決したいか、誰のためか、やらない選択と撤退条件
3. [`02_競合と差別化/`](./02_競合と差別化/) — 既存製品との比較と k1s0 の勝ち筋 (Build vs Buy の定性根拠)
4. [`03_ロードマップと体制/`](./03_ロードマップと体制/) — フェーズ計画・MVP スコープ・体制
5. [`04_定量試算/`](./04_定量試算/) — 5 年 TCO・開発工数・運用工数の実数試算 (Build vs Buy の数的根拠)
6. [`05_法務サマリ/`](./05_法務サマリ/) — ライセンス適合・法令適合・知財帰属・監査応答の稟議向け要約（4 論点即答）
7. [`全体構成図.md`](./全体構成図.md) — インフラ＋アーキテクチャの 1 枚俯瞰図 (技術面の概観)

技術深掘り (アーキテクチャ・tier1 設計・OSS 選定・CI/CD・法務) は [`../02_構想設計/`](../02_構想設計/) を参照。
