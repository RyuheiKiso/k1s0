# k1s0 企画ドキュメント

JTC (Japanese Traditional Company) 情報システム部門向けマイクロサービス基盤プラットフォーム **k1s0** の企画資料群。
最上位の提案書は [`企画書.md`](./企画書.md) (Marp スライド) を参照し、詳細は本ディレクトリ配下の各章を参照する。

## 読み方

初読時は以下の順で読むと全体像が掴みやすい。

1. [`企画書.md`](./企画書.md) — 10 分で読み切れるエグゼクティブサマリ (Marp スライド)
2. [`01_背景と目的/`](./01_背景と目的/) — 何を解決したいか、誰のためか
3. [`02_アーキテクチャ/`](./02_アーキテクチャ/) — 全体像 (レイヤ構成・依存ルール・配置形態)
4. [`03_tier1設計/`](./03_tier1設計/) — k1s0 の核心である tier1 の設計思想
5. [`04_技術選定/`](./04_技術選定/) — 採用 OSS と選定根拠
6. [`05_CICDと配信/`](./05_CICDと配信/) — CI/CD パイプラインと 2 種類のポータル
7. [`06_競合と差別化/`](./06_競合と差別化/) — 既存製品との比較と k1s0 の勝ち筋
8. [`07_ロードマップと体制/`](./07_ロードマップと体制/) — フェーズ計画・MVP スコープ・体制

## 記載方針

- 図表は drawio で作成し、同階層の `img/` に `.drawio` と `.svg` を両方格納する。md からは svg を埋め込む。
- 日付は `YYYY-MM-DD` 形式で記載する。
- 本ディレクトリは「Proposed 状態の企画資料」である。確定した技術決定は `docs/02_infra/00_ADR/` 配下の ADR として別途記録する。
