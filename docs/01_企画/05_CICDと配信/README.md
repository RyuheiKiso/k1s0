# 05_CICDと配信

04 章で採用を決めた GHA / Harbor / Trivy / Argo CD / Backstage / Renovate を「1 本のパイプライン」と「2 種類のポータル（開発者向け / アプリ配信向け）」に統合する章。GHA self-hosted runner で build/scan/push、Argo CD で GitOps デプロイ、Renovate で自動更新 PR、Backstage で全状態を可視化する。「何を使うか」ではなく「運用時に何が動き、誰が何を見るか」を設計している。

## 想定読者

DevOps 担当、システム基盤チーム、アプリ配信担当（エンドユーザー向け配信ポータル運用者）、および k1s0 上で開発を始める tier2/tier3 開発者。
