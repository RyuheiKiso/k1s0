# 試算前提

## 目的

本フォルダ `04_定量試算/` 配下の各ファイル（TCO 比較 / 開発工数 / 運用工数）が共通して参照する前提条件を一枚に集約する。試算の透明性と更新可能性を確保するため、規模感・人月単価・インフラ単価・商用ライセンス単価を出典付きで明示し、「どの数字をどう動かすと結論がどう変わるか」を企画書の読者が独立して検証できる状態を目指す。

本資料で採用する数字はいずれも公開情報・業界ヒアリング・自社経験則の三点に基づく推定値であり、実際の契約・見積もり段階では大きく変動する。本企画書は「決裁者の判断材料として十分な精度」を目的とし、会計精度の試算ではないことを明記する。

---

## 1. 企業規模の 3 段階想定

k1s0 の採用対象は「中堅以上の日系大企業 (JTC)」であり、単一サイズで試算を固めると汎用性を失う。本試算では、従業員数の異なる 3 段階で並行試算を行うことで、費用優位性が「どの規模から効くのか」を可視化する。3 段階の選定根拠は以下の散文に示す。

従業員 1,000 名は、商用 IDP のライセンス費が年 3,000 万円を超え「実質的に選択肢が狭まる」ボーダーにあたる。Excel 運用や属人化スクリプトでは業務システム数の増加に耐えられず、かといって商用製品を買えない「狭間」の企業群が該当する。3,000 名は、複数事業部が並行して内製着手し、共通基盤への投資判断が稟議に上がる JTC の典型サイズで、本企画書の主要ターゲット。10,000 名は、商用 IDP の本格検討が進む規模だが、ライセンス費が年 1 億円を超えて「稟議が通らない」段階に入る。k1s0 の費用優位性が最も顕在化するゾーンである。

### 1.1 規模別の想定パラメータ

| 区分 | 小 (1,000 名) | 中 (3,000 名) | 大 (10,000 名) | 根拠 / 出典 |
|---|---|---|---|---|
| 社内業務システム数 | 30 | 80 | 200 | 経産省「DX 白書 2024」中央値を従業員比で外挿（1 システム/30〜50 名） |
| 情シス人員 (全体) | 5 名 | 15 名 | 50 名 | 日本情報システム・ユーザー協会「企業 IT 動向調査 2024」 情シス比率 0.5% |
| 内製開発エンジニア | 2 名 | 10 名 | 30 名 | 情シス内 内製比率 30〜40%（JUAS 調査 2024） |
| アプリ配信ポータル想定 DAU | 600 | 1,800 | 6,000 | 業務系ポータルの DAU/全従業員比 60%（自社経験則） |
| tier1 API ピーク RPS | 50 | 150 | 500 | DAU × 1 時間 1 操作 + ピーク係数 2 倍 |
| 同時 PoC 案件数 | 1〜2 | 3〜5 | 10〜15 | 中堅 SI ヒアリング（5 社平均） |
| パイロット業務数 (初年度) | 1 | 2 | 3 | Phase 1c 完了までに稼働させる業務数の想定 |

### 1.2 パラメータ選定の妥当性

社内業務システム数は「DX 白書 2024」の中央値を従業員比で外挿したもので、JTC の業務系システム（人事 / 経費 / 勤怠 / 申請 / 受発注 等）の実数を直感的に再現する。情シス人員比率 0.5% は経団連・JUAS の複数調査で繰り返し観測されている JTC の構造的制約であり、「内製したくてもできない」根拠そのもの。アプリ配信ポータルの DAU は、社内業務アプリ全般の利用実態から 60% を採用する（営業外勤・現場作業員を除くと従業員の過半が日次アクセスする想定）。

## 2. 人月単価

人月単価は経産省「IT 人材白書 2025」および情報サービス産業協会 JISA「IT サービス産業 経営概況調査 2024」のレンジに基づき、以下の 4 区分で試算する。社員単価は年収・賞与・社会保険・諸経費込みの会社負担ベースで、協力会社単価は契約単価そのもの（会社負担を含まない）である点に注意。

| 区分 | 月額単価 (税込) | 内訳 | 出典 |
|---|---|---|---|
| 社員 (中堅) | 100 万円 | 年収 600 万 + 賞与 200 万 + 社保 25% + 諸経費 | JISA 経営概況 2024 中央値 |
| 社員 (シニア) | 150 万円 | 年収 900 万 + 賞与 300 万 + 同上 | JISA 経営概況 2024 上位 25% |
| 協力会社 (SES 中堅) | 130 万円 | IT 人材派遣相場 (中堅 SE) | レバテック市場レポート 2025 / パーソル IT 単価表 |
| 協力会社 (SES シニア) | 180 万円 | アーキテクト級 SES | クラスメソッド / REVSAS 単価表 2025 |

### 2.1 単価適用方針

- Phase 0〜1a の起案者は社員 (シニア) として 150 万円/月を適用
- Phase 1b 以降の協力者は社員 (中堅) として 100 万円/月を適用
- Phase 2 以降の拡張メンバーは社員 (中堅) と協力会社 (SES 中堅) を併用し、加重平均で 115 万円/月を適用
- 残業・休日出勤は単価に含むとみなし、別途積み増ししない

これらの方針は「JTC 情シスの内製開発は社員主導・協力会社補完」という実態を反映する。単価を機械的に一律で置くと大手 SIer 相場（200 万円超）になり、JTC 自社試算としての実感を失う。

## 3. インフラ単価

k1s0 はオンプレ前提の基盤だが、比較対象として AWS 東京リージョン価格も参考値として併記する。オンプレ単価は自社データセンター運用を想定し、サーバ調達費を 5 年で償却、電力・空調・ラック費・運用人件費を加味した「TCO 単価」を採用する。為替レートは 1 USD = 150 円で固定する。

### 3.1 コンピュート単価

| リソース | オンプレ月額 | AWS 月額 (オンデマンド) | オンプレ内訳 / AWS 対応 |
|---|---|---|---|
| 4 vCPU / 16 GB / 200 GB SSD | 5,000 円 | 約 40,000 円 | サーバ償却 3,000 + 電力/空調 1,500 + ラック 500 / m6i.xlarge + EBS gp3 |
| 8 vCPU / 32 GB / 500 GB SSD | 10,000 円 | 約 80,000 円 | 同上スケール / m6i.2xlarge + EBS gp3 |
| 16 vCPU / 64 GB / 1 TB SSD | 20,000 円 | 約 160,000 円 | 同上スケール / m6i.4xlarge + EBS gp3 |

#### 3.1.1 オンプレ単価の詳細内訳

オンプレ単価「4 vCPU / 16 GB / 200 GB SSD = 5,000 円/月」の算出根拠を 4 要素で分解する。1 台の物理サーバに VM 8 台を収容する想定 (ハイパースレッド含めコア 32 / メモリ 128 GB の典型的な 2U ラックサーバ) で按分する。

| 内訳項目 | 月額 (円) | 計算根拠 | 出典 |
|---|---|---|---|
| サーバハードウェア償却 | 3,000 | 物理サーバ本体 40 万円 (Dell PowerEdge R660 / HPE ProLiant DL360 相当、2U/32 コア/128GB/4TB NVMe) + 保守 5 年 25 万円 = 65 万円を 60 か月で償却し、VM 8 台で按分 (65 万 ÷ 60 ÷ 8 ≒ 1,354 円) … に加え、クラスタのコントロールプレーン冗長化・予備機オーバーヘッド (1.5 倍) で 2,030 円、ラックスイッチ・KVM 等の共通装置償却で追加 1,000 円 | Dell / HPE 2025 年国内定価・大企業値引き率 65% 適用後 |
| 電力・空調 | 1,500 | サーバ 1 台の平均消費電力 500 W (ピーク 800 W の 60% 稼働想定) + 空調の PUE 1.6 × 24h × 30 日 × 産業用電力単価 22 円/kWh = 500 × 1.6 × 720 × 22 / 1,000 = 約 12,670 円/サーバ/月 → VM 8 台按分で約 1,584 円 | 日本データセンター協会 TCO レポート 2024 / 経産省電力単価 |
| ラック・床面積 | 500 | ラック 1 本 (42U) の月額 3 万円 (自社データセンター、設備費按分) を 20 台のサーバで按分 = 1,500 円 → 2U サーバなので 1,500 × 2 / 42 × 20 / 8 VM = 約 446 円 | 自社データセンター相場 / NTT Com / IIJ の Colo 価格参考値 |
| 運用人件費の按分 | — (次項参照) | 運用人件費は規模別 FTE 内訳 ([`../03_ロードマップと体制/02_体制と役割.md`](../03_ロードマップと体制/02_体制と役割.md)) で TCO 試算内に別計上。コンピュート単価には含めない | — |
| **合計 (VM 単位)** | **5,000 円** | 端数調整 | — |

運用人件費を単価に含めないのは、本試算の TCO 比較 ([`01_TCO5年試算.md`](./01_TCO5年試算.md)) で人件費を別項目で計上するためである。二重計上を避ける方針として、コンピュート単価はハードウェア・電力・施設の物理コストのみを含む。

#### 3.1.2 AWS 単価の見直し (RI / Savings Plan 適用)

オンデマンド単価 (m6i.xlarge 4 万円/月) は AWS Pricing API の 2026-04 時点の東京リージョン価格。ただし実務では Reserved Instances (RI) や Savings Plan (SP) の適用で実効単価が大幅に下がるため、商用案比較の公平性を確保するため RI / SP 適用シナリオを並行試算する。

| 契約形態 | 割引率 (AWS 公称) | m6i.xlarge 実効月額 | 5 年 TCO への反映 |
|---|---|---|---|
| オンデマンド | 0% | 40,000 円 | 基準値 |
| RI 1 年 (All Upfront) | 約 30% | 28,000 円 | 1 年ごとの購入更新が前提 |
| RI 3 年 (All Upfront) | 約 60% | 16,000 円 | 3 年契約ロックイン |
| Compute Savings Plan 1 年 | 約 27% | 29,200 円 | インスタンスタイプ変更柔軟 |
| Compute Savings Plan 3 年 | 約 55% | 18,000 円 | 同上 |

### 3.1.3 AWS 単価の実務想定

AWS を商用 IDP の比較対象として用いる場合、JTC の実務ではオンデマンド単価をそのまま採用することは稀で、以下のいずれかを選ぶ。

- **短期 PoC**: オンデマンド (〜 3 か月)
- **本番 1〜2 年**: RI 1 年 / SP 1 年
- **本番 3 年以上**: RI 3 年 / SP 3 年

k1s0 の TCO 比較は 5 年試算のため、「3 年契約 + 2 年目以降も RI 更新」が JTC の妥当な想定となる。本企画書では以下の 3 シナリオを並行試算する。

| シナリオ名 | AWS 単価前提 | 採用根拠 |
|---|---|---|
| オンデマンド | 40,000 円/月 (m6i.xlarge) | 旧来の試算 (3.1 節の数値) |
| RI 1 年更新 | 28,000 円/月 | 事業継続性優先、柔軟性確保 |
| RI 3 年固定 | 16,000 円/月 | 事業計画が 3 年以上確定 |

RI 3 年固定シナリオでは AWS 単価が 60% 引き下がる。この場合のオンプレ単価とのギャップは 1/3.2 (5,000 ÷ 16,000) となり、従来の「1/8」よりは縮まるが依然としてオンプレが優位。

ただし RI 3 年ロックインは「事業の柔軟性を失う」トレードオフがあり、JTC が実際に採用するかは個別判断となる。Broadcom 統合後の VMware Tanzu 価格改定のような予期せぬ変動が AWS でも起きる可能性 (過去事例: S3 値下げ継続、EC2 は値上げなし) を考慮すると、RI 3 年ロックインのリスクプレミアムは 5〜10% 相当と見積もれる。

### 3.1.4 TCO 試算での採用値

[`01_TCO5年試算.md`](./01_TCO5年試算.md) の基準シナリオは **AWS オンデマンド** を採用する。ただし感度分析章で RI 1 年 / RI 3 年適用時の商用案 TCO を並行掲示することで、「AWS の値引きを適用しても k1s0 が優位か」を明示する。

### 3.2 ネットワーク・ストレージ単価

| リソース | オンプレ月額 | AWS 月額 | 備考 |
|---|---|---|---|
| L4 ロードバランサ | 0 円 (MetalLB) | 約 20,000 円 (NLB 1 リスナー) | オンプレは MetalLB + kube-vip で内製 |
| バックアップストレージ (S3 互換) | 5 円/GB | 約 3 円/GB | オンプレは MinIO + 自社 SAS HDD、AWS は S3 標準 |
| ドメイン + SSL | 833 円 (年 1 万円) | 833 円 (Route53 + ACM) | cert-manager + 社内 CA 想定 |
| 帯域 (外部接続) | 月額固定 2 万円 (1 Gbps 専用線) | 従量 100 円/GB | オンプレは既存回線相乗り想定 |

### 3.3 単価選定の妥当性

オンプレ単価が AWS オンデマンド単価の 1/8 になるのは、経産省「ICT 投資動向調査 2024」および日本データセンター協会の TCO レポートで繰り返し観測される構造的なギャップである。サーバ本体価格（1 台 30〜50 万円を 5 年償却）と電力コスト（日本の産業用電力単価は北米の 1.5 倍）を合わせても、クラウドの粗利・マージンが除かれる分、オンプレの単価優位性は維持される。

AWS に RI 3 年ロックインを適用した場合のギャップは 1/3.2 に縮まるが、それでもオンプレが優位である事実は変わらない。加えて RI 3 年は事業計画の柔軟性を失うロックインを伴うため、JTC の実務選択としてはむしろ採用しづらい側面がある。この構造は k1s0 の「オンプレで完結する IDP」というバリュープロポジションの根幹であり、試算でも主軸に据える。

## 4. 商用製品ライセンス費の前提

Build (k1s0) vs Buy (商用 IDP) の比較に用いるライセンス費は、各ベンダーの公開価格および代理店ヒアリングに基づく推定値である。JTC 向けの実際の見積もりは規模・契約期間・値引き交渉で大きく変動するため、本試算は「公開情報ベースの想定上限」として扱う。

| 製品 | ライセンス形態 | 採用単価 | 公開 / 推定根拠 |
|---|---|---|---|
| Red Hat OpenShift Container Platform (Standard) | サブスクリプション | 1,500 USD/コア/年 | Red Hat List 価格 5,500 USD × 値引き率 70% (大企業代理店ヒアリング) |
| Red Hat OpenShift Application Foundations | アドオン | 500 USD/コア/年 | Red Hat List 価格より推定（Service Mesh / GitOps / Pipelines 相当） |
| VMware Tanzu Application Platform | サブスクリプション | 800 USD/CPU/年 | Broadcom 統合後の代理店価格（2025 年国内取引例） |
| Humanitec Platform Orchestrator | SaaS | 基本 60,000 USD/年 (10 ユーザー) + 1,500 USD/追加ユーザー | Humanitec 公開価格 (Enterprise tier) |

### 4.1 補足: 機能スコープの不一致

商用製品は各社で内包機能が異なるため、単純なライセンス費比較では意図を読み誤る。本試算では「k1s0 と同等の機能セットを揃えるのに必要な追加費用」を各商用案に加算する。具体的には以下の対応関係を想定する。

- OpenShift = K8s 基盤 + GitOps + Service Mesh + Pipelines（k1s0 の infra + tier1 の一部に相当）。残差として「アプリ配信ポータル」「tier1 ファサード独自 API」は内製追加が必要
- Tanzu Application Platform = K8s 基盤 + 開発者ポータル + Service Mesh（k1s0 の infra + 05_CICDと配信 の一部に相当）
- Humanitec = 開発者ポータル のみ。K8s 基盤は別途 (EKS または OpenShift) 必要

この機能スコープの差分は `01_TCO5年試算.md` で加算項目として明示する。

## 5. 試算の不確実性と更新方針

本試算は公開情報ベースの推定であり、以下の不確実性を含む。決裁段階では実見積もりによる更新を前提とする。

- 商用ライセンスの企業値引き（公開価格の 50〜70% が JTC の一般的水準）
- AWS のリザーブドインスタンス / Savings Plan 値引き（1 年 30% / 3 年 60%）
- 為替レート変動（本試算は 1 USD = 150 円固定、±10% で試算結果が変動）
- 人月単価の地域差（東京 / 地方で 20〜30% の差）
- OSS の採用変更（例: Valkey が商用化される、Keycloak のライセンス変更等）

### 5.1 更新タイミング

- 半期ごとの見直し（4 月・10 月）
- 主要ベンダーの価格改定時（Broadcom 統合のような構造変化）
- 為替レートが ±10% 動いた時
- 主要 OSS のライセンス変更時（前例: Redis → Valkey 2024）
