# 02_アーキテクチャ

k1s0 の全体像を「tier1 → tier2 → tier3 → infra」の 4 レイヤ構造で捉え、各レイヤが上位レイヤをどう楽にするかを可視化する章。本章のゴールは、読者が「この構造なら 01 章の痛みは解ける」と納得できる地点まで抽象度を下げること。個別 OSS の選定根拠には踏み込まず、構造上の決定（依存方向・配置形態・非機能要件）に限定する。

## 想定読者

アーキテクト級の技術リード、インフラチーム、tier1 実装担当。経営層は [`00_概念アーキテクチャ.md`](./00_概念アーキテクチャ.md) の全体図と [`../全体構成図.md`](../全体構成図.md) だけで全体像を掴める粒度で書いている。
