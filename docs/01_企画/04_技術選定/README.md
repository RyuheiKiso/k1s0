# 04_技術選定

02・03 章で定めた構造を、実際に動く OSS にマッピングする章。選定方針は「オンプレ前提・OSI 承認済みライセンス・CNCF 推奨・運用実現性」の 4 軸に固定されており、本章はその軸で 40+ OSS を分類・採否判定している。既存の [`00_選定方針.md`](./00_選定方針.md) が章内ナビの役割を兼ねるため、本 README はそれを補完する短い索引である。

## 想定読者

技術リード（調達・ライセンス・スキル獲得の判断）、経営層（稟議根拠として「なぜこれか」を確認）、将来の差し替え判断者（長期戦略を確認する者）。

## 前章と後章

- 前章: [`../02_アーキテクチャ/`](../02_アーキテクチャ/) / [`../03_tier1設計/`](../03_tier1設計/) — 本章で構造を OSS にマッピングする。
- 後章: [`../05_CICDと配信/`](../05_CICDと配信/) — 選定した OSS を CI/CD に組み込む。

## 読み順

本章は **俯瞰 → タイムライン → 詳細** の順で読むことを推奨する。個別 OSS の根拠ファイルが 15 本に分岐しているため、先に 04 と 19 で全体像を取らないと迷子になる。

1. [`00_選定方針.md`](./00_選定方針.md) — 判断軸（**必ず最初に読む**）
2. [`04_選定一覧.md`](./04_選定一覧.md) — **責務マトリクス + 採用 40+ OSS のインデックス表 (俯瞰の起点)**
3. [`19_フェーズ別採用タイムライン.md`](./19_フェーズ別採用タイムライン.md) — **Phase 1a/1b/1c/2/3 × OSS の時系列マトリクス**
4. [`01_実行基盤中核OSS.md`](./01_実行基盤中核OSS.md) — k8s / Istio / Envoy Gateway / Kafka / Dapr / Go / Rust の詳細
5. [`18_言語選定の定量分析.md`](./18_言語選定の定量分析.md) — Go+Rust ハイブリッドの定量根拠
6. [`17_OSS長期戦略.md`](./17_OSS長期戦略.md) — 継続性リスクと差し替え・自作移行シナリオ
