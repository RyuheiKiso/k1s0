## 3. tier1 公開 API と内部実装の対応表

tier2 / tier3 が利用する API が、内部でどの OSS / サービスにマッピングされるか。

| tier1 公開 API | 内部実装 | 依存 OSS |
|---|---|---|
| `k1s0.Log.*` | tier1 各言語クライアントライブラリ内 (構造化ログ) | Loki (バックエンド) |
| `k1s0.Telemetry.*` | tier1 各言語ライブラリが OTel SDK をラップ | OpenTelemetry / OTel Collector / Grafana Tempo / Prometheus / Grafana Pyroscope |
| `k1s0.State.*` | Go ファサード → Dapr Go SDK | Dapr / Valkey (バックエンド) |
| `k1s0.PubSub.*` | Go ファサード → Dapr Go SDK → Apicurio 検証 | Dapr / Kafka / Apicurio Registry |
| `k1s0.Secrets.*` | Go ファサード → Dapr Go SDK | Dapr / OpenBao (MVP-1b 以降) |
| `k1s0.Workflow.*` | Go ファサード → Temporal Go SDK (Saga / 長期実行) / Dapr Workflow (軽量短期処理) | Temporal / Dapr / PostgreSQL |
| `k1s0.Service.Invoke*` | Go ファサード → Dapr Go SDK | Dapr / Istio |
| `k1s0.Auth.*` | Go サービス (認証プロキシ) | Keycloak / oauth2-proxy / PostgreSQL |
| `k1s0.Audit.*` | Rust サービス (監査改ざん防止) | ZEN Engine (評価) / Loki (保管) / PostgreSQL (永続化) |
| `k1s0.Decision.*` | Rust サービス (ZEN Engine in-process) | ZEN Engine |
| `k1s0.Settings.*` | Go ファサード → Dapr Configuration / State | Dapr / Valkey |
| `k1s0.Feature.*` | Go ファサード → flagd gRPC | flagd |

> **補足**: Rate Limiting は Go ファサード層で処理される。詳細は [`../02_アーキテクチャ/10_レート制限.md`](../02_アーキテクチャ/10_レート制限.md) を参照。
