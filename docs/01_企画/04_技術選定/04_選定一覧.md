## 3. tier1 公開 API と内部実装の対応表

tier2 / tier3 が利用する API が、内部でどの OSS / サービスにマッピングされるか。

| tier1 公開 API | 内部実装 | 依存 OSS |
|---|---|---|
| `k1s0.Log.*` | tier1 各言語クライアントライブラリ内 (構造化ログ) | Loki (バックエンド) |
| `k1s0.Telemetry.*` | tier1 各言語ライブラリが OTel SDK をラップ | OpenTelemetry / Jaeger / Prometheus |
| `k1s0.State.*` | Go ファサード → Dapr Go SDK | Dapr / Valkey (バックエンド) |
| `k1s0.PubSub.*` | Go ファサード → Dapr Go SDK → Apicurio 検証 | Dapr / Kafka / Apicurio Registry |
| `k1s0.Secrets.*` | Go ファサード → Dapr Go SDK | Dapr / Vault or k8s secrets |
| `k1s0.Workflow.*` | Go ファサード → Dapr Workflow | Dapr |
| `k1s0.Service.Invoke*` | Go ファサード → Dapr Go SDK | Dapr / Istio |
| `k1s0.Auth.*` | Go サービス (認証プロキシ) | Keycloak / oauth2-proxy / PostgreSQL |
| `k1s0.Audit.*` | Rust サービス (監査改ざん防止) | ZEN Engine (評価) / Loki (保管) / PostgreSQL (永続化) |
| `k1s0.Decision.*` | Rust サービス (ZEN Engine in-process) | ZEN Engine |
| `k1s0.Settings.*` | Go ファサード → Dapr Configuration / State | Dapr / Valkey |
| `k1s0.Feature.*` | Go ファサード → flagd gRPC | flagd |

> **補足**: Rate Limiting は Go ファサード層で処理される。詳細は [`../02_アーキテクチャ/10_レート制限.md`](../02_アーキテクチャ/10_レート制限.md) を参照。

## 4. ライセンス分布

| ライセンス種別 | 該当 OSS | 備考 |
|---|---|---|
| Apache 2.0 | k8s / Istio / Envoy Gateway / Kafka / Jaeger / Prometheus / Dapr / Backstage / Keycloak / Argo CD / GHA runner / Tekton / Harbor / Trivy / containerd / Kyverno / cert-manager / Apicurio Registry / Tilt / OpenFeature / flagd / Litmus / Longhorn / Argo Events | 最多 |
| AGPL-3.0 | Grafana / Loki (一部) / MinIO / Renovate (self-hosted) | 改変配布時の義務に留意。社内利用は問題なし |
| MPL 2.0 | OpenTofu | |
| BSD-3-Clause | Valkey / Go | |
| PostgreSQL License | PostgreSQL | OSI 承認。BSD 系に近い許容的ライセンス |
| MIT | ZEN Engine / 一部 Rust クレート / Tilt (一部) | |
| ISC | PgBouncer | BSD 系に近い許容的ライセンス |

全採用 OSS は OSI 承認済み。非 OSI (RSALv2 / SSPL / BSL) は含まれていない。
