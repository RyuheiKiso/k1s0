## 0. 責務マトリクス (読み順の推奨)

OSS の個別ファイルが 15 本に分岐しているため、**先に責務別の俯瞰** を取ってから詳細に進むことを推奨する。本節は「この責務は誰が担うのか」「代替候補は何か」「Phase 何で本番水準に上げるのか」を 1 枚で答えるマトリクス。Phase 列は [`19_フェーズ別採用タイムライン.md`](./19_フェーズ別採用タイムライン.md) の Intro/Prod 水準に対応する。

| 責務カテゴリ | 担当 OSS | 代替候補 (差し替え時) | 本番水準フェーズ | 選定の肝 |
|---|---|---|---|---|
| **クラスタ実行** | Kubernetes / containerd / kubeadm | k3s (軽量) / RKE2 (エンタープライズ OSS) | Phase 1c | オンプレ + kubeadm HA の成熟度 |
| **ネットワーク境界** | Envoy Gateway / Istio / MetalLB / kube-vip | NGINX Ingress / Linkerd / Cilium | Phase 1c (Envoy) / Phase 2 (Istio) | Gateway API 準拠、CNCF 標準 |
| **メッセージング** | Apache Kafka (Strimzi) / Apicurio Registry | NATS JetStream / Redpanda | Phase 1c | 既存 JTC 資産互換性、スキーマレジストリ成熟 |
| **データ永続化** | CloudNativePG + PostgreSQL / pgvector / pg_partman / PgBouncer | Postgres Operator by Zalando / StackGres | Phase 1c (PG) / Phase 2 (拡張) | Operator 成熟度とバックアップ自動化 |

| **オブジェクトストレージ** | MinIO | SeaweedFS / Garage | Phase 2 | AGPL 制約下の社内利用整合 (第 2 節) |
| **マイクロサービス抽象** | Dapr (Control Plane + sidecar) | 自作ファサードのみ / 直接 Istio | Phase 1c | tier1 内部隠蔽で将来差し替え可能 |
| **tier1 内部言語** | Go (ファサード) / Rust (自作領域) | — (ハイブリッド固定) | Phase 1c (Go) / Phase 2 (Rust) | [`18_言語選定の定量分析.md`](./18_言語選定の定量分析.md) |

| **コンテナレジストリ / スキャン** | Harbor + Trivy / Cosign / Syft / Grype | Quay / Kraken | Phase 1c | SBOM / 署名の統合 |
| **IaC** | OpenTofu | Terraform (ライセンス変更済) / Pulumi | Phase 1c | [`05_IaC.md`](./05_IaC.md) |

| **オブザーバビリティ** | Prometheus / Grafana / Loki / Grafana Tempo / Grafana Pyroscope / kube-state-metrics / OTel Collector | VictoriaMetrics / SigNoz / ClickHouse Stack | Phase 1c (主要) / Phase 2 (Pyroscope) | AGPL 制約下の社内利用整合 (第 2 節) |
| **ポリシー / ガバナンス** | Kyverno | OPA Gatekeeper | Phase 1c | YAML ベース、k8s ネイティブ |
| **スキーマ / 契約** | Buf (Protobuf) / Apicurio Registry (イベント) | buf-alt なし / Confluent Schema Registry | Phase 1c | 言語間契約の単一起点 ([`../03_tier1設計/`](../03_tier1設計/)) |
| **ワークフロー** | Temporal (長期 Saga) / Dapr Workflow (軽量短期) | Argo Workflows / Cadence | Phase 2 | [`12_マニフェストとワークフロー.md#Z`](./12_マニフェストとワークフロー.md) |
| **Feature Flag** | OpenFeature (SDK) / flagd (評価) | Unleash / Flagsmith | Phase 1c | ベンダー中立の SDK 標準 |
| **分散ストレージ (ブロック)** | Longhorn | Rook + Ceph / OpenEBS | Phase 1c | VM 環境での運用容易性 |
| **開発者ポータル / TechDocs** | Backstage | Port (商用) / 自作 | Phase 2 | 業界標準、プラグインエコシステム |

| **テスト基盤** | Testcontainers / Litmus (Chaos) | Chaos Mesh | Phase 1c (Testcontainers) / Phase 2 (Litmus) | CI 統合の容易性 |

| **ローカル開発** | Tilt / Kustomize / Helm | Skaffold / Devspace | Phase 1c | k8s 上開発の標準セット |
| **依存管理** | Renovate | Dependabot (GitHub) | Phase 2 | self-hosted で OSS プロジェクト対応 |

全 40+ OSS を本マトリクスに含める。**代替候補欄** は OSS 側のライセンス変更や開発停止に備え、[`17_OSS長期戦略.md`](./17_OSS長期戦略.md) で想定する差し替え候補を最低 1 つ併記する。候補が「—」の行は方針固定であり、差し替えには別途 RFC が必要。

## 3. tier1 公開 API と内部実装の対応表

tier2 / tier3 が利用する API が、内部でどの OSS / サービスにマッピングされるか。

| tier1 公開 API | 内部実装 | 依存 OSS |
|---|---|---|
| `k1s0.Log.*` | tier1 各言語クライアントライブラリ内 (構造化ログ) | Loki (バックエンド) |
| `k1s0.Telemetry.*` | tier1 各言語ライブラリが OTel SDK をラップ | OpenTelemetry / OTel Collector / Grafana Tempo / Prometheus / Grafana Pyroscope |
| `k1s0.State.*` | Go ファサード → Dapr Go SDK | Dapr / Valkey (バックエンド) |
| `k1s0.PubSub.*` | Go ファサード → Dapr Go SDK → Apicurio 検証 | Dapr / Kafka / Apicurio Registry |
| `k1s0.Secrets.*` | Go ファサード → Dapr Go SDK | Dapr / OpenBao (MVP-1b 以降) |
| `k1s0.Workflow.*` | Go ファサード → Temporal Go SDK (Saga / 長期実行) / Dapr Workflow (軽量短期処理) | Temporal / Dapr / PostgreSQL |
| `k1s0.Service.Invoke*` | Go ファサード → Dapr Go SDK | Dapr / Istio |
| `k1s0.Auth.*` | Go サービス (認証プロキシ) | Keycloak / oauth2-proxy / PostgreSQL |
| `k1s0.Audit.*` | Rust サービス (監査改ざん防止) | ZEN Engine (評価) / Loki (保管) / PostgreSQL (永続化) |
| `k1s0.Decision.*` | Rust サービス (ZEN Engine in-process) | ZEN Engine |
| `k1s0.Settings.*` | Go ファサード → Dapr Configuration / State | Dapr / Valkey |
| `k1s0.Feature.*` | Go ファサード → flagd gRPC | flagd |

> **補足**: Rate Limiting は Go ファサード層で処理される。詳細は [`../02_アーキテクチャ/04_非機能とデータ/03_レート制限.md`](../02_アーキテクチャ/04_非機能とデータ/03_レート制限.md) を参照。
