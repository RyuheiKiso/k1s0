# 05. レガシーラップ配置

本ファイルは `src/tier3/legacy-wrap/` 配下の .NET Framework ラッパー配置を確定する。ADR-MIG-001 で規定された Dapr sidecar パターンの物理配置である。

## レガシーラップの目的

JTC の既存 .NET Framework 4.8 資産を、k1s0 基盤で運用しつつ段階的に移行するための仕組み。以下の 2 形態を採用する。

- **Sidecar パターン**: 既存 .NET Framework プロセスに Dapr sidecar（Dapr runtime）を併設し、tier1 公開 API を Dapr 経由で利用可能にする
- **Wrapper パターン**: 既存 .NET Framework ライブラリを呼び出す .NET 8 のラッパーサービスを構築し、そのラッパーが Pod として k1s0 基盤上で動作する（sidecar ではなく完全な Pod 化）

Phase 1b では Sidecar パターン、Phase 2 以降で順次 Wrapper パターンまたは完全な .NET 8 / .NET MAUI 再実装に移行する。

## sidecar パターンの構造

既存 .NET Framework プロセスは変更せず、その隣に Dapr sidecar container が同 Pod 内で動作する。.NET Framework 側は Dapr の HTTP API（localhost:3500）を経由して k1s0 公開 API を利用する。

- .NET Framework 側は `K1s0.Legacy.Sidecar` プロジェクトで HTTP 薄ラッパーを実装
- Dapr sidecar が認証・リトライ・mTLS などを透過的に扱う
- Kubernetes Pod 定義で `dapr.io/sidecar-inject: "true"` アノテーションを付与（Dapr Control Plane が自動注入）

## Dockerfile（Windows container）

```dockerfile
# sidecars/K1s0.Legacy.Sidecar/Dockerfile.windows
FROM mcr.microsoft.com/dotnet/framework/sdk:4.8-windowsservercore-ltsc2022 AS builder
WORKDIR /workspace
COPY . .
RUN nuget restore LegacyWrap.sln
RUN msbuild sidecars/K1s0.Legacy.Sidecar/K1s0.Legacy.Sidecar.csproj ^
    /p:Configuration=Release ^
    /p:OutDir=C:\publish\ ^
    /p:WebPublishMethod=FileSystem

FROM mcr.microsoft.com/dotnet/framework/aspnet:4.8-windowsservercore-ltsc2022
WORKDIR /inetpub/wwwroot
COPY --from=builder C:/publish/ ./
EXPOSE 80
```

tag は `4.8-windowsservercore-ltsc2022` で Windows Server OS version を固定する。`:4.8` のみだと `floating` tag となり、upstream の Windows Server 更新時に Node との host compatibility が壊れる可能性がある（Windows container は host の Windows Server build number と一致する必要がある）。

Windows container は Kubernetes Windows Node で動作する。Phase 1b の環境整備で Windows Node 2 台以上を用意する（NFR-A-AVL-\* の要件に応じて HA 化）。

## wrappers/（Phase 2 以降）

完全な Pod 化を目指す場合、以下の手順で移行する。

1. 既存 .NET Framework ライブラリ（DLL）をそのまま `.NET 8` のプロジェクトから参照（マネージド相互運用）
2. ラッパーが `src/sdk/dotnet/` を使って tier1 公開 API にアクセス
3. ラッパー自体は Linux container として k1s0 基盤上で動作（Windows Node 不要）

これにより Windows Node の依存から脱出できる。ただし全ライブラリが .NET Standard 2.0+ で参照可能である必要があるため、.NET Framework 固有 API（WCF Server / WPF / Forms）を使っている場合は再実装が必要。

## migration-guide/

既存資産から k1s0 基盤への移行手順を Markdown で整備する。

- `01_sidecar_setup.md`: Dapr sidecar の注入と初期動作確認
- `02_gradual_api_exposure.md`: 既存 API を段階的に k1s0 公開 API 経由に置換
- `03_framework_to_net8.md`: .NET Framework → .NET 8 本格移行

これらは実装ガイドであり、顧客の .NET Framework 資産ごとに個別にカスタマイズする想定。

## 依存方向

- Legacy ラッパーは `src/sdk/dotnet/` を介して tier1 にアクセス
- tier2 / tier3 の他部分を参照することは禁止
- 既存 .NET Framework ライブラリは `third_party/` に vendoring するか、社内 NuGet server から参照する（リポジトリには含めない）
