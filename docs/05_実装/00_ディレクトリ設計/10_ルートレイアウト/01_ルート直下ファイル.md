# 01. ルート直下ファイル

本ファイルはリポジトリルート（`k1s0/`）直下に配置するファイルと、その役割・Phase 導入タイミングを定める。ディレクトリではなくファイル単位の配置判断を本章で固定することにより、Phase 1a 以降の追加タイミングで「どこに置くか」の迷いを解消する。

## 必要性

ルート直下は `git clone` 直後に目に入る場所であり、OSS 公開時の顔にもなる。一方で「ルートに何をどこまで置くか」の判断は曖昧になりがちで、以下の破綻が頻発する。

- 似た名前のファイルが複数生まれる（`CODEOWNERS.md` / `owners.txt` / `AUTHORS` 等）
- OSS リポジトリに期待されるファイル（`CONTRIBUTING.md` / `SECURITY.md`）が欠落している
- 開発者が日常的に編集するファイル（`Makefile` / `.gitattributes`）とアーキテクト専用のファイル（`GOVERNANCE.md`）が混在して、読みにくさが増す

本ファイルはルート直下ファイルを 3 カテゴリ（プロジェクト規約 / OSS 標準ファイル / ビルド・Git 設定）に分けて整理する。

## カテゴリ 1: プロジェクト規約

プロジェクト固有の規約とメタ情報。

- `CLAUDE.md` — Claude Code / Agents 向けの規約。プロジェクト共通規約として参照される
- `README.md` — 第一印象となるプロジェクト概要。Phase 0 時点では最小限、Phase 1a で本格化
- `CHANGELOG.md` — リリース履歴。Phase 1b 以降で運用開始（`release-please` 等の自動化を検討）

## カテゴリ 2: OSS 標準ファイル

OSS プロジェクトで期待される標準ファイル。CNCF / OpenSSF の推奨に準拠する。

- `LICENSE` — プロジェクトライセンス。Apache License 2.0 を想定（Phase 0 稟議で確定予定）
- `NOTICE` — 利用 OSS ライセンスの集約。SBOM から自動生成される予定（Phase 1b）
- `SECURITY.md` — セキュリティ報告手段。Phase 1a で整備
- `CODE_OF_CONDUCT.md` — 行動規範。Phase 1b 以降で整備（Contributor Covenant 2.1 準拠）
- `CONTRIBUTING.md` — コントリビューションガイド。Phase 1b 以降
- `GOVERNANCE.md` — プロジェクト統治。Phase 1c 以降（正式なスポンサー / PMC 体制成立時）
- `SUPPORT.md` — サポート窓口。Phase 1b 以降

## カテゴリ 3: ビルド・Git 設定

ビルドツールと Git の設定ファイル。

- `Makefile` — ルート Makefile。`bootstrap` / `test-all` / `lint-all` / `build-all` / `deploy` 等の横断ターゲットを提供
- `.gitattributes` — LFS / 改行コード / linguist-generated 等の属性定義
- `.gitignore` — 生成物・ローカル設定の除外
- `.git-blame-ignore-revs` — 大規模整形コミットの blame 除外（formatter 一斉適用時に記録）
- `.editorconfig` — エディタ共通設定（文字コード / 改行 / インデント）

### .gitattributes の推奨設定サンプル

```
# 既定値: 自動的な改行コード変換を無効化
* -text

# テキストファイルは明示的に text 指定
*.md        text eol=lf
*.rs        text eol=lf
*.go        text eol=lf
*.cs        text eol=lf
*.ts        text eol=lf
*.tsx       text eol=lf
*.json      text eol=lf
*.yaml      text eol=lf
*.yml       text eol=lf
*.toml      text eol=lf
*.sh        text eol=lf
Dockerfile  text eol=lf

# 生成物は linguist-generated でカウント除外
src/tier1/go/internal/proto/v1/**           linguist-generated=true
src/tier1/rust/crates/proto-gen/src/v1/**   linguist-generated=true

# バイナリ資産
*.png       binary
*.jpg       binary
*.gif       binary
*.svg       text eol=lf
*.drawio    text eol=lf
```

上記は雛形。Phase A（ADR 起票直後）に実ファイルとして配置する別タスクでそのまま使用する。

### .git-blame-ignore-revs の用途

大規模整形コミット（formatter 一斉適用、ディレクトリ移動など）のコミットハッシュをここに列挙する。`git blame` 実行時に対象コミットをスキップし、真の変更者を特定できる。

Phase 0 時点は空ファイル。Phase 1a で以下のような整形コミットが発生した時点で追記する。

```
# rustfmt + gofmt + prettier 一斉適用（YYYY-MM-DD）
abc123def456...

# src/tier1/contracts/ から src/contracts/ へ移動（ADR-DIR-001）
def456abc789...
```

## Phase 導入タイミング

以下のタイミングで各ファイルを整備する。

| Phase | 追加するファイル |
|---|---|
| Phase 0 （本章確定時） | CLAUDE.md（既存）/ README.md / LICENSE / SECURITY.md / .gitattributes / .gitignore / .git-blame-ignore-revs / .editorconfig / Makefile |
| Phase 1a | NOTICE（SBOM 自動生成体制構築後） |
| Phase 1b | CHANGELOG.md / CODE_OF_CONDUCT.md / CONTRIBUTING.md / SUPPORT.md |
| Phase 1c | GOVERNANCE.md |
