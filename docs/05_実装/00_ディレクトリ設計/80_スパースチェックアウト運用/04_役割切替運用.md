# 04. 役割切替運用

本ファイルは役割間の切替、複数役割の兼任、チーム横断作業時の運用を規定する。

## 役割切替の典型シナリオ

- **コードレビュー時**: 自分の tier 以外の PR をレビューするため一時的に該当役割に切替
- **障害対応時**: インシデントで複数 tier を調査するため一時的に full 役割に切替
- **兼任開発者**: tier1 Rust + tier2 を兼任、常時両方を include
- **アーキテクト**: 常時 full 役割
- **BFF 開発者が SDK 公開 API 改修が必要になった場合**: tier3-web-dev から `tier3-web-dev + sdk-dev` の合成 cone へ切替（`src/sdk/go/` の内部構造・契約テスト・他言語 SDK との整合が必要なため）

## 単一役割切替

```bash
# 現在の役割確認
git sparse-checkout list

# 別役割に切替（既存の cone を上書き）
./tools/sparse/checkout-role.sh tier2-dev

# 切替後、不要なディレクトリは物理的に削除される
ls src/tier1/  # tier1/ 配下は空（ディレクトリエントリのみ）
```

切替時、sparse-checkout 範囲外のディレクトリはワーキングツリーから削除される。未 commit の変更があればエラーになる。事前に commit / stash が必須。

## 複数役割の兼任

`checkout-role.sh -m` オプションで複数役割をマージ。

```bash
# tier1 Rust + tier2 開発を兼任
./tools/sparse/checkout-role.sh -m tier1-rust-dev tier2-dev

# 内部処理:
cat .sparse-checkout/roles/tier1-rust-dev.txt \
    .sparse-checkout/roles/tier2-dev.txt \
    | sort -u > /tmp/merged.txt
git sparse-checkout set --stdin < /tmp/merged.txt
```

Dev Container も複数役割兼用の `tools/devcontainer/profiles/combined/` を `-m` 指定で自動生成する。

## 一時的 full 切替

インシデント調査で全コードを見たい場合:

```bash
# 現在の役割を退避
git sparse-checkout list > /tmp/saved-cone.txt

# full に切替
./tools/sparse/checkout-role.sh full

# 作業後、元の役割に復帰
git sparse-checkout set --stdin < /tmp/saved-cone.txt
```

`tools/sparse/checkout-role.sh --push / --pop` で stack 管理を提供:

```bash
./tools/sparse/checkout-role.sh --push full  # 現在を save、full に切替
# ... 作業 ...
./tools/sparse/checkout-role.sh --pop         # 元に戻す
```

## 役割追加時の運用

新役割の追加（例: Phase 1c で `tier3-web-bff-dev` を tier3-web-dev から分離）は以下の手順:

1. `.sparse-checkout/roles/tier3-web-bff-dev.txt` を作成（PR）
2. `tools/devcontainer/profiles/tier3-web-bff-dev/` を作成
3. `docs/05_実装/00_ディレクトリ設計/80_スパースチェックアウト運用/02_役割別cone定義.md` を更新
4. `@k1s0/arch-council` のレビュー・承認
5. merge 後、`tools/sparse/list-roles.sh` の出力に自動反映

## 役割切替時の注意

### 未 commit の変更

```bash
# 未 commit があると切替失敗
$ ./tools/sparse/checkout-role.sh tier2-dev
error: Your local changes to the following files would be overwritten by sparse-checkout:
        src/tier1/rust/crypto/src/lib.rs
Please commit your changes or stash them before you switch role.
```

対処:

```bash
git stash
./tools/sparse/checkout-role.sh tier2-dev
# 元の役割に戻ったら stash pop
```

### IDE / LSP の再起動

役割切替後、IDE の LSP インデックスを再構築する必要がある。

- **VS Code**: Rust Analyzer / Go / TypeScript の LSP 再起動（`Ctrl+Shift+P → "Reload Window"`）
- **JetBrains IntelliJ**: Invalidate Caches → Restart

Dev Container 切替（役割変更時）の場合は自動で LSP 再起動される。

### git log の見え方

sparse-checkout 配下にないディレクトリの変更も `git log` には出る（cone は working tree だけ絞り、history は全体）。PR review 時に想定外の diff が表示されても驚かないこと。

## チーム横断作業の運用

四半期に 1 回、全員が一時的に full 役割に切替えて横断 refactoring を行う「Full Week」を `ops/runbooks/weekly/full-week.md` として Runbook 化。

## 役割別使用率メトリクス

`tools/ci/role-metrics.sh` で全開発者の使用役割を集計（Backstage の Software Catalog と PagerDuty ローテーションから推定）。以下を監視:

- 各役割の使用者数
- 複数役割兼任率
- full 使用率（高いと役割分離が機能していないサイン）

Phase 1c で full 使用率 20% 以下を閾値に、cone mode の必須化を判断する。
