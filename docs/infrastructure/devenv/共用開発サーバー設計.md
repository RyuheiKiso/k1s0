# 共用開発サーバー設計

開発者のローカル PC にリソース集約型サービス（Kafka・OTel スタック等）を Docker で起動する容量がない場合、共用開発サーバーを活用して柔軟に開発環境を構築する。

## 背景・課題

| 課題 | 詳細 |
| --- | --- |
| ローカル PC のリソース不足 | Kafka・Jaeger・Prometheus・Grafana・Loki を全て起動するとメモリ 8GB 以上を消費する |
| ポート競合 | 共用サーバーで複数開発者が同一ポートを使用するとバインドエラーが発生する |
| 環境差異 | 開発者ごとに異なる設定が docker-compose.yaml 本体に入り込むと管理不能になる |

## 方式比較

| 方式 | 概要 | メリット | デメリット |
| --- | --- | --- | --- |
| **VS Code Remote SSH（推奨）** | SSH で共用サーバーに接続し、サーバー上で Docker を操作 | ポートフォワード自動、localhost 扱いで透過的 | VS Code 必須 |
| Docker Context | `docker context create` でリモート Docker Engine を指定 | IDE 非依存、CLI ベース | ポートフォワード手動、ボリュームマウント不可 |
| ポートオフセット | 開発者ごとにポート範囲を割り当て（例: +100） | 設定が単純 | ポート管理が煩雑、スケールしない |

## 推奨構成: VS Code Remote SSH + Compose Project 分離

```
[開発者ローカル PC]                    [共用開発サーバー]
  VS Code ──── SSH ────────────>  Docker Engine
  Remote-SSH 拡張                  docker compose -p <username>
  ポートフォワード（自動）            ├── postgres, kafka, redis
                                   ├── keycloak, vault, kong
                                   ├── auth-rust, config-rust, ...
                                   └── jaeger, prometheus, grafana
```

### 動作原理

1. VS Code Remote SSH でサーバーに接続すると、ターミナルはサーバー上で動作する
2. `docker compose` コマンドはサーバーの Docker Engine を直接操作する
3. VS Code が `forwardPorts` やターミナル出力からポートを自動検出し、ローカル PC にフォワードする
4. 開発者はローカル PC の `localhost:8083` 等でサービスにアクセスできる

### COMPOSE_PROJECT_NAME による分離

`COMPOSE_PROJECT_NAME` を設定すると、以下のリソースに自動的にプレフィックスが付与され、開発者間で完全に分離される。

| リソース | 例（COMPOSE_PROJECT_NAME=alice） |
| --- | --- |
| コンテナ名 | `alice-postgres-1`, `alice-auth-rust-1` |
| ボリューム名 | `alice_postgres-data`, `alice_kafka-data` |
| ネットワーク名 | `alice_default`（`COMPOSE_NETWORK_NAME` 未設定時） |

```bash
# .env に設定
COMPOSE_PROJECT_NAME=alice
```

## セットアップ手順

### サーバー側（初回のみ）

```bash
# 1. Docker Engine インストール（Ubuntu の場合）
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 2. 開発者ユーザーを docker グループに追加
sudo usermod -aG docker $USER

# 3. リポジトリのクローン
git clone <repository-url> ~/k1s0
cd ~/k1s0

# 4. .env の設定
cp .env.example .env
# COMPOSE_PROJECT_NAME にユーザー名を設定
echo "COMPOSE_PROJECT_NAME=$(whoami)" >> .env

# 5. サービス起動
docker compose --profile infra up -d
docker compose --profile infra --profile observability up -d  # OTel スタックも使う場合
```

### PC 側

```bash
# 1. VS Code に Remote-SSH 拡張をインストール
code --install-extension ms-vscode-remote.remote-ssh

# 2. SSH 設定（~/.ssh/config）
# Host shared-dev
#     HostName <server-ip>
#     User <username>
#     IdentityFile ~/.ssh/id_ed25519

# 3. VS Code で接続
# コマンドパレット > "Remote-SSH: Connect to Host..." > shared-dev

# 4. サーバー上のリポジトリを開く
# File > Open Folder > ~/k1s0
```

## 運用ルール

### リソース上限

共用サーバーのリソースを公平に使用するため、以下のガイドラインを設ける。

| 項目 | 推奨値 |
| --- | --- |
| 同時起動プロファイル | infra + observability + system まで |
| ボリュームサイズ目安 | 開発者あたり 20GB 以内 |
| 長期未使用コンテナ | 1 週間以上未使用の場合はクリーンアップ |

### クリーンアップ手順

```bash
# 自分のコンテナ・ネットワークを停止・削除
docker compose down

# データも含めて完全削除
docker compose down -v

# 未使用イメージの削除
docker image prune -f
```

### リソース使用状況の確認

```bash
# 全開発者のコンテナ一覧
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

# ディスク使用量
docker system df
```

## 代替方式: Docker Context

VS Code を使用しない場合、Docker Context でリモート Docker Engine に接続できる。

```bash
# 1. Docker Context の作成
docker context create shared-dev --docker "host=ssh://user@server-ip"

# 2. Context の切り替え
docker context use shared-dev

# 3. 以降の docker コマンドはリモートで実行される
docker compose --profile infra up -d

# 4. ポートフォワード（手動）
ssh -L 5432:localhost:5432 -L 8083:localhost:8083 -L 16686:localhost:16686 user@server-ip

# 5. ローカルに戻す
docker context use default
```

> **注意**: Docker Context 方式ではボリュームマウント（`./infra/docker/...`）がローカルパスを参照するため、ビルドコンテキストやマウント設定に注意が必要。サーバー側にリポジトリをクローンしておくことを推奨する。

## 推奨プロファイル構成

### 共用サーバー（フルスタック）

```bash
# インフラ + 可観測性 + system サーバー
docker compose --profile infra --profile observability --profile system up -d
```

### ローカル PC（軽量）

```bash
# インフラのみ（OTel なし）
docker compose --profile infra up -d

# 可観測性は共用サーバーの OTEL_EXPORTER_OTLP_ENDPOINT を参照
# config.dev.yaml:
#   otel:
#     endpoint: "http://<shared-server>:4317"
```
