## テンプレートエラーハンドリング

テンプレート条件評価時のエラーは沈黙的に無視せず、ユーザーに通知する。

### 実装パターン

```rust
// テンプレート条件の評価
match tera.render_str(&condition, &context) {
    Ok(result) => result == "true",
    Err(e) => {
        eprintln!("テンプレート条件評価エラー: {e}");
        false
    }
}
```

テンプレートレンダリングで発生するエラーは以下のカテゴリに分類される:

| カテゴリ | 例 | 処理 |
| --- | --- | --- |
| 構文エラー | 不正な Tera 構文 | eprintln で通知、false を返す |
| 未定義変数 | コンテキストに存在しない変数参照 | eprintln で通知、false を返す |
| 型エラー | フィルタへの不正な型渡し | eprintln で通知、false を返す |

## テンプレートコンテキストバリデーション

`TemplateContextBuilder` はビルド前に必須変数のバリデーションを実行する。

### 必須変数

| 変数 | 説明 |
| --- | --- |
| service_name | サービス名 |
| tier | ティア（system / business / service） |
| language | 言語（go / rust / typescript / dart） |
| kind | 種別（server / client / library / database） |

`try_build()` メソッドで Result を返し、`build()` メソッドではバリデーション失敗時に panic する。
