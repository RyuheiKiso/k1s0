# CLI テンプレート設計

本ドキュメントは k1s0 CLI のテンプレートコンテキスト設計とバリデーション仕様を記載する。

---

## テンプレートコンテキスト

CLI の `generate` コマンドは対話フローで収集した入力値から `TemplateContext` を構築し、
Tera テンプレートエンジンに渡してコード生成を行う。

### 必須変数

以下の変数はテンプレートレンダリングに必須であり、`TemplateContextBuilder` で構築時にバリデーションされる。

| 変数名 | 型 | 説明 | バリデーション |
|--------|-----|------|--------------|
| `service_name` | String | サービス名（kebab-case） | 空文字列禁止 |
| `tier` | String | 配置階層（system / business / service） | 空文字列禁止 |
| `language` | String | 言語識別子（go / rust / typescript / dart） | 空文字列禁止 |
| `kind` | String | 種別（server / client / library / database） | 空文字列禁止 |

### 自動導出変数

必須変数から以下が自動導出される。

| 変数名 | 導出ルール |
|--------|-----------|
| `service_name_snake` | kebab-case → snake_case |
| `service_name_pascal` | kebab-case → PascalCase |
| `service_name_camel` | kebab-case → camelCase |
| `module_path` | tier + kind + language + service_name から導出 |
| `go_module` | Go 言語時のみ: `{go_module_base}/{module_path}` |
| `rust_crate` | Rust 言語時のみ: service_name |
| `docker_project` | `k1s0-{tier}` |
| `namespace` | `k1s0-{tier}` |
| `helm_path` | service_name |

### オプション変数

| 変数名 | 型 | デフォルト | 説明 |
|--------|-----|-----------|------|
| `domain` | String | `""` | 業務領域名（business tier 時） |
| `framework` | String | `""` | フレームワーク（react / flutter） |
| `api_style` | String | `""` | API 方式（rest / grpc / graphql） |
| `api_styles` | Vec | `[]` | 複数 API 方式 |
| `has_database` | bool | `false` | DB 有無 |
| `database_type` | String | `""` | RDBMS 種別 |
| `has_kafka` | bool | `false` | Kafka 有無 |
| `has_redis` | bool | `false` | Redis 有無 |
| `docker_registry` | String | `harbor.internal.example.com` | Docker レジストリ |

## コンテキストバリデーション

### `validate()` メソッド

`TemplateContextBuilder` に `validate()` メソッドを提供し、必須変数の事前チェックを行う。

```rust
let builder = TemplateContextBuilder::new(name, tier, lang, kind);
// 明示的なバリデーション（エラーハンドリング可能）
if let Err(e) = builder.validate() {
    eprintln!("テンプレートコンテキストエラー: {}", e);
}
```

### `build()` vs `try_build()`

| メソッド | 戻り値 | バリデーション失敗時 |
|---------|--------|-------------------|
| `build()` | `TemplateContext` | panic |
| `try_build()` | `Result<TemplateContext, String>` | `Err(String)` を返却 |

テスト・内部コードでは `build()` を使用し、ユーザー入力を処理する箇所では `try_build()` を推奨する。

## module_path 導出ルール

| tier | パターン |
|------|---------|
| system | `regions/system/{kind}/{language}/{service_name}` |
| business | `regions/business/{domain}/{kind}/{language}/{service_name}` |
| service | `regions/service/{service_name}/{kind}/{language}` |

## テンプレートバージョン管理ポリシー

テンプレート内の依存バージョンは **ワークスペース (`regions/system/Cargo.toml`) と常に一致させる**（ADR-0095 参照）。

### 現在の主要バージョン（2026-04-04 時点）

| 依存 | バージョン | 対象テンプレート |
|------|----------|---------------|
| axum | 0.8 | server/rust, bff/rust |
| opentelemetry | 0.27 | server/rust (system tier) |
| opentelemetry-otlp | 0.27 | server/rust (system tier) |
| opentelemetry_sdk | 0.27 | server/rust (system tier) |
| utoipa-swagger-ui | 9 | server/rust (REST) |
| mockall | 0.14 | server/rust (dev-dependencies) |
| async-graphql-axum | 7 | bff/rust (GraphQL) |
| gqlgen | v0.17.55 | server/go, bff/go (GraphQL) |
| gqlparser/v2 | v2.5.19 | server/go, bff/go (GraphQL) |

### BFF Rust テンプレートのフレームワーク

BFF Rust テンプレートは **axum 0.8** を使用する（H-012 監査対応）。
actix-web は使用禁止。ワークスペース全体が axum に統一されているため。

### バージョン更新手順

1. `regions/system/Cargo.toml` の `[workspace.dependencies]` を更新する。
2. 同時に `CLI/crates/k1s0-cli/templates/` 配下の対応テンプレートを更新する。
3. PR レビューチェックリストで「テンプレートバージョン同期確認」を実施する。
