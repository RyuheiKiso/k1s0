# イベントコード生成設計

## 概要

`events.yaml` 定義ファイルから Kafka イベント連携に必要なコードを一括自動生成する CLI コマンド。
Tera テンプレートエンジンを用いて、Proto スキーマ・Producer・Consumer・Outbox マイグレーション・Schema Registry 設定を生成する。

> **関連設計書**: ライブラリとしての設計は [event-codegen 設計書](../../libraries/codegen/event-codegen.md) を参照。

## 実装パス

`CLI/crates/k1s0-core/src/commands/generate_events/`

| ファイル | 責務 |
|----------|------|
| `mod.rs` | モジュール定義・公開 API |
| `types.rs` | `EventsConfig`, `EventDefinition`, `EventSchema`, `SchemaField`, `ConsumerDefinition` の型定義 |
| `parser.rs` | `events.yaml` の読み込み・バリデーション |
| `context.rs` | Tera テンプレートコンテキストの構築 |
| `execute.rs` | コード生成の実行・ファイル書き込み |

## 入力

### events.yaml

プロジェクトルートに配置する YAML 定義ファイル。

```yaml
domain: accounting
tier: business
service_name: domain-master
language: rust  # rust | go

events:
  - name: master-item.created
    version: 1
    description: "マスタアイテムが作成された時に発行されるイベント"
    partition_key: item_id
    outbox: true
    schema:
      fields:
        - name: item_id
          type: string
          number: 1
          description: "アイテムID"
    consumers:
      - domain: fa
        service_name: asset-manager
        handler: on_accounting_master_item_created
```

### バリデーションルール

| ルール | 詳細 |
|--------|------|
| `domain` | 非空、kebab-case |
| `tier` | `system` / `business` / `service` |
| `service_name` | 非空、kebab-case |
| `language` | `rust` / `go` |
| `events` | 1つ以上 |
| `event.name` | kebab-case + ドット区切り（例: `master-item.created`） |
| `event.version` | >= 1 |
| `schema.fields` | 1つ以上 |
| `field.number` | >= 1、イベント内で重複なし |
| `field.type` | proto3 有効型（`string`, `int32`, `int64`, `uint32`, `uint64`, `bool`, `float`, `double`, `bytes`, `google.protobuf.Timestamp`） |
| `partition_key` | schema 内のフィールド名と一致 |
| `consumer.handler` | snake_case |
| イベント名 | 重複なし |

## 出力

### 共通生成ファイル

| ファイル | 内容 |
|----------|------|
| `proto/{domain}/events/v{ver}/{name_snake}.proto` | Proto スキーマ |
| `migrations/{n}_create_outbox.up.sql` | Outbox UP マイグレーション（outbox: true 時のみ） |
| `migrations/{n}_create_outbox.down.sql` | Outbox DOWN マイグレーション（outbox: true 時のみ） |
| `config/schema-registry.yaml` | Schema Registry 設定 |

### Rust 固有生成ファイル

| ファイル | 内容 |
|----------|------|
| `src/events/mod.rs` | モジュール定義 |
| `src/events/types.rs` | イベント型定義 |
| `src/events/producer.rs` | Producer + Outbox 関数 |
| `src/events/consumers/{handler}.rs` | Consumer ハンドラ（consumers 定義のあるイベントのみ） |

### Go 固有生成ファイル

| ファイル | 内容 |
|----------|------|
| `internal/events/producer.go` | Producer + Outbox 関数 |
| `internal/events/consumers/{handler}.go` | Consumer ハンドラ |

## 処理フロー

```
1. events.yaml を読み込み (parser::parse_events_yaml)
2. YAML をパースして EventsConfig にデシリアライズ
3. バリデーション実行 (parser::validate)
   - domain/tier/service_name/language の形式チェック
   - イベント名の重複チェック
   - フィールド番号の重複チェック
   - partition_key の存在チェック
   - consumer.handler の命名チェック
4. テンプレートコンテキスト構築 (context::build_template_context)
   - ケース変換（snake_case, PascalCase）
   - トピック名生成
   - Proto パッケージ名生成
   - Proto 型 → Rust/Go 型マッピング
5. Tera テンプレートレンダリング + ファイル書き込み (execute::execute_event_codegen)
   - Proto ファイル: イベントごとに個別コンテキストで生成
   - 言語別コード: Rust / Go に分岐して生成
   - Outbox マイグレーション: 既存マイグレーション番号の次番号を自動採番
   - Schema Registry 設定: 全イベント分を一括生成
6. 生成サマリー表示 (execute::format_generation_summary)
```

## テンプレートコンテキスト

### グローバルコンテキスト

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `domain` | String | ドメイン名 |
| `domain_snake` | String | ドメイン名（snake_case） |
| `tier` | String | 配置階層 |
| `service_name` | String | サービス名 |
| `service_name_snake` | String | サービス名（snake_case） |
| `language` | String | 言語識別子 |
| `has_outbox` | bool | Outbox パターンを使用するイベントがあるか |
| `has_consumers` | bool | Consumer 定義があるイベントがあるか |
| `events` | Array | イベント定義配列 |

### イベントコンテキスト（events 配列内）

| 変数名 | 型 | 説明 |
|--------|-----|------|
| `name` | String | イベント名（原形） |
| `name_snake` | String | snake_case 名 |
| `name_pascal` | String | PascalCase 名（Proto メッセージ名） |
| `version` | u32 | バージョン |
| `topic` | String | Kafka トピック名 |
| `partition_key` | String | パーティションキー |
| `partition_key_pascal` | String | パーティションキー（PascalCase） |
| `outbox` | bool | Outbox 有効フラグ |
| `proto_package` | String | Proto パッケージ名 |
| `fields` | Array | フィールド定義配列 |
| `consumers` | Array | Consumer 定義配列 |

### トピック命名規約

```
k1s0.{tier}.{domain}.{name-hyphenated}.v{version}
```

例: `k1s0.business.accounting.master-item-created.v1`

## テンプレートディレクトリ

テンプレートファイルは `templates/events/` 配下に配置される。

```
templates/events/
├── proto/
│   └── event.proto.tera
├── rust/
│   ├── mod.rs.tera
│   ├── types.rs.tera
│   ├── producer.rs.tera
│   └── consumer.rs.tera
├── go/
│   ├── producer.go.tera
│   └── consumer.go.tera
├── sql/
│   ├── outbox.up.sql.tera
│   └── outbox.down.sql.tera
└── config/
    └── schema-registry.yaml.tera
```
