# service tier（個別サービス層）概要

## service tier の役割と責務

service tier は k1s0 の 3 層 Tier アーキテクチャにおける最下位層であり、**実際にデプロイ・稼働する個別の業務サービス**を担う。エンドユーザーに価値を届ける最前線として、業務ロジックの実装からクライアントアプリの提供までを一貫して行う。

```
service (tier3) → business (tier2) → system (tier1)
```

service tier は system tier と business tier の両方に依存し、それらが提供する共通基盤・領域共通機能を活用してサービスを構築する。

## 個別サービスの位置づけ

各サービスは `regions/service/{サービス名}/` に配置される。サービスごとに独立したサーバー、クライアント、データベースを持ち、自律的にデプロイ・運用される。

- **独立性**: 各サービスは自身のデータベースを所有し、他サービスのデータベースに直接アクセスしない
- **自律的デプロイ**: サービス単位で CI/CD パイプラインを持ち、他サービスに影響を与えずにデプロイ可能
- **BFF 間通信禁止**: サービス内の BFF は独立したエンドポイントとして動作し、他サービスの BFF と直接通信しない

## 提供するもの

| 種別 | 内容 | 配置先 |
| --- | --- | --- |
| 業務サーバー（Rust） | ドメインロジック、gRPC API | `server/rust/` |
| BFF（Rust） | GraphQL BFF（async-graphql） | `server/rust/bff/` |
| BFF（Go） | GraphQL BFF（gqlgen）または REST BFF | `server/go/bff/` |
| React クライアント | Web アプリケーション | `client/react/` |
| Flutter クライアント | モバイル/デスクトップアプリ | `client/flutter/` |
| サービス DB | サービス固有の業務データ（PostgreSQL/MySQL/SQLite） | `database/` |

### ディレクトリ構成例（order サービス）

```
regions/service/order/
├── server/
│   ├── rust/           # Rust 業務サーバー
│   │   └── bff/        # Rust GraphQL BFF（オプション）
│   └── go/
│       └── bff/        # Go BFF（オプション）
├── client/
│   ├── react/          # React Web アプリ
│   └── flutter/        # Flutter アプリ
└── database/           # DB スキーマ・マイグレーション
```

## system tier / business tier との関係

### system tier からの利用

| 利用内容 | 方式 |
| --- | --- |
| 認証トークンの検証（JWT） | system library を import |
| ユーザー情報の取得 | system server へ gRPC 呼出 |
| 設定管理（config） | system library を import |
| ログ・トレース・メトリクス | system library を import |
| キャッシュ・メッセージング | system library を import |
| ファイルストレージ | system library を import |

### business tier からの利用

| 利用内容 | 方式 |
| --- | --- |
| 領域共通サーバー API | gRPC / REST 呼出 |
| 領域共通 UI コンポーネント | business client を import |
| 領域共通ライブラリ | business library を import |

service tier から system tier への直接通信は、business tier を経由せずに許可されている。

## 担当開発者に求められるスキル

| 分野 | 必要なスキル |
| --- | --- |
| 言語 | Go / Rust / TypeScript / Dart のうち担当部分 |
| サーバー | axum（Rust）、gqlgen / net/http（Go）、tonic（gRPC）、sqlx（DB） |
| クライアント | React（TanStack Query, Zustand, Tailwind 等）または Flutter（Riverpod, go_router 等） |
| 設計 | クリーンアーキテクチャ、DDD の基礎、TDD |
| API | GraphQL、REST、gRPC の基礎 |
| データベース | PostgreSQL / MySQL の設計・マイグレーション |
| インフラ | Docker、Kubernetes の基礎概念 |
| ツール | Git、k1s0 CLI の基本操作 |
