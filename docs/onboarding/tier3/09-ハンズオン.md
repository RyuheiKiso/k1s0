# service tier ハンズオン

ゼロから 1 つの機能を追加する手順をステップバイステップで実践する。2 つのシナリオを用意した。

- **シナリオ A**: サーバーサイド -- task サービスに「タスクステータスを更新する API」を追加
- **シナリオ B**: クライアントサイド（React）-- タスク一覧画面に「ステータスフィルター」を追加

各ステップで「どのファイルを編集するか」「何を書くか」のコード例を示す。

### Step 4: domain 層にモデル追加

**編集ファイル**: `regions/service/task/server/rust/src/domain/model/task.rs`

ステータスの列挙型と遷移ロジックを追加する。

```rust
/// タスクステータス
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum TaskStatus {
    Pending,     // 保留中
    Confirmed,   // 確定
    Shipped,     // 発送済み
    Delivered,   // 配達完了
    Cancelled,   // キャンセル
}

impl Task {
    /// ステータスを更新する
    /// ビジネスルール: 許可された遷移のみ受け付ける
    pub fn update_status(&mut self, new_status: TaskStatus) -> Result<(), DomainError> {
        // ステータス遷移の検証
        let allowed = match (&self.status, &new_status) {
            (TaskStatus::Pending, TaskStatus::Confirmed) => true,
            (TaskStatus::Pending, TaskStatus::Cancelled) => true,
            (TaskStatus::Confirmed, TaskStatus::Shipped) => true,
            (TaskStatus::Confirmed, TaskStatus::Cancelled) => true,
            (TaskStatus::Shipped, TaskStatus::Delivered) => true,
            _ => false,
        };

        if !allowed {
            return Err(DomainError::InvalidStatusTransition {
                from: self.status.clone(),
                to: new_status,
            });
        }

        self.status = new_status;
        self.updated_at = Utc::now();
        Ok(())
    }
}
```

**ポイント**: ビジネスルール（「保留中からしか確定にできない」等）はドメイン層に書く。DB やフレームワークに依存しない純粋なロジック。

### Step 5: application 層にユースケース追加

**新規ファイル**: `regions/service/task/server/rust/src/application/usecase/update_status.rs`

```rust
use crate::domain::model::task::TaskStatus;
use crate::domain::repository::task::TaskRepository;
use crate::application::dto::{UpdateStatusInput, UpdateStatusOutput};

pub struct UpdateTaskStatusUsecase<R: TaskRepository> {
    repository: Arc<R>,
}

impl<R: TaskRepository> UpdateTaskStatusUsecase<R> {
    pub fn new(repository: Arc<R>) -> Self {
        Self { repository }
    }

    pub async fn execute(&self, input: UpdateStatusInput) -> Result<UpdateStatusOutput, UsecaseError> {
        // 1. タスクを取得
        let mut task = self.repository
            .find_by_id(input.task_id)
            .await
            .map_err(|_| UsecaseError::NotFound)?;

        // 2. ステータスを更新（ドメインロジック）
        let new_status = TaskStatus::try_from(input.new_status.as_str())
            .map_err(|_| UsecaseError::InvalidInput("不正なステータス値".into()))?;
        task.update_status(new_status)?;

        // 3. 保存
        self.repository.save(&task).await?;

        // 4. 結果を返す
        Ok(UpdateStatusOutput {
            task_id: task.id,
            status: task.status.to_string(),
            updated_at: task.updated_at,
        })
    }
}
```

**編集ファイル**: `regions/service/task/server/rust/src/application/dto/request.rs`

### Step 6: infrastructure 層にリポジトリ実装

**編集ファイル**: `regions/service/task/server/rust/src/infrastructure/repository/task.rs`

既存の `TaskRepository` 実装に `save` メソッドを追加（まだ無い場合）。

```rust
impl TaskRepositoryImpl {
    pub async fn save(&self, task: &Task) -> Result<(), RepositoryError> {
        sqlx::query!(
            r#"
            UPDATE tasks
            SET status = $1, updated_at = $2
            WHERE id = $3
            "#,
            task.status.to_string(),
            task.updated_at,
            task.id
        )
        .execute(&self.pool)
        .await?;

        Ok(())
    }
}
```

**ポイント**: `sqlx::query!` マクロを使うと、コンパイル時に SQL の文法チェックが行われる。DB に接続できる環境でビルドする必要がある。

### Step 7: presentation 層に gRPC ハンドラー追加

**編集ファイル**: `regions/service/task/server/rust/src/presentation/grpc/task.rs`

```rust
use crate::proto::task::{
    UpdateTaskStatusRequest, UpdateTaskStatusResponse,
};

#[tonic::async_trait]
impl TaskService for TaskServiceImpl {
    // 既存のハンドラー...

    async fn update_task_status(
        &self,
        request: Request<UpdateTaskStatusRequest>,
    ) -> Result<Response<UpdateTaskStatusResponse>, Status> {
        let req = request.into_inner();

        // リクエストからDTOへ変換
        let input = UpdateStatusInput {
            task_id: Uuid::parse_str(&req.task_id)
                .map_err(|_| Status::invalid_argument("不正なtask_id"))?,
            new_status: req.new_status,
        };

        // ユースケース実行
        let output = self.update_status_usecase
            .execute(input)
            .await
            .map_err(|e| match e {
                UsecaseError::NotFound => Status::not_found("タスクが見つかりません"),
                UsecaseError::InvalidInput(msg) => Status::invalid_argument(msg),
                _ => Status::internal("内部エラー"),
            })?;

        // レスポンスを返す
        Ok(Response::new(UpdateTaskStatusResponse {
            task_id: output.task_id.to_string(),
            status: output.status,
            updated_at: output.updated_at.to_rfc3339(),
        }))
    }
}
```

**ポイント**: presentation 層の役割は「リクエスト → DTO 変換 → ユースケース呼び出し → レスポンス変換」の 4 ステップ。エラーを適切な gRPC Status に変換するのも重要。

### Step 3: useQuery フックの作成

**編集ファイル**: `regions/service/task/client/react/src/features/task/api/use-tasks.ts`

既存のフックにフィルターパラメータを追加する。

```typescript
import { useQuery } from "@tanstack/react-query";
import { api } from "@/shared/lib/api";
import type { Task, TaskListParams } from "../types/task";

export const taskKeys = {
  all: ["tasks"] as const,
  lists: () => [...taskKeys.all, "list"] as const,
  list: (params: TaskListParams) => [...taskKeys.lists(), params] as const,
  detail: (id: string) => [...taskKeys.all, "detail", id] as const,
};

export function useTasks(params: TaskListParams = {}) {
  return useQuery({
    // paramsをキーに含めることで、フィルター変更時に自動再取得される
    queryKey: taskKeys.list(params),
    queryFn: async () => {
      const searchParams = new URLSearchParams();
      if (params.status) {
        searchParams.set("status", params.status);
      }
      const query = searchParams.toString();
      const url = `/api/v1/tasks${query ? `?${query}` : ""}`;
      return api.get<Task[]>(url);
    },
  });
}
```

**ポイント**: `queryKey` にパラメータを含めることで、フィルター条件が変わるたびに TanStack Query が自動的にデータを再取得してくれる。

```typescript
import { renderHook, waitFor } from "@testing-library/react";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { http, HttpResponse } from "msw";
import { setupServer } from "msw/node";
import { describe, it, expect, beforeAll, afterAll } from "vitest";
import { useTasks } from "./use-tasks";

const mockTasks = [
  { id: "1", status: "pending", total: 1000, createdAt: "2026-01-01" },
  { id: "2", status: "confirmed", total: 2000, createdAt: "2026-01-02" },
];

const server = setupServer(
  http.get("/api/v1/tasks", ({ request }) => {
    const url = new URL(request.url);
    const status = url.searchParams.get("status");

    if (status) {
      const filtered = mockTasks.filter((o) => o.status === status);
      return HttpResponse.json(filtered);
    }
    return HttpResponse.json(mockTasks);
  }),
);

beforeAll(() => server.listen());
afterAll(() => server.close());

function createWrapper() {
  const queryClient = new QueryClient({
    defaultOptions: { queries: { retry: false } },
  });
  return ({ children }: { children: React.ReactNode }) => (
    <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
  );
}

describe("useTasks", () => {
  it("フィルターなしで全件取得する", async () => {
    const { result } = renderHook(() => useTasks(), {
      wrapper: createWrapper(),
    });

    await waitFor(() => expect(result.current.isSuccess).toBe(true));
    expect(result.current.data).toHaveLength(2);
  });

  it("ステータスフィルターで絞り込む", async () => {
    const { result } = renderHook(
      () => useTasks({ status: "pending" }),
      { wrapper: createWrapper() },
    );

    await waitFor(() => expect(result.current.isSuccess).toBe(true));
    expect(result.current.data).toHaveLength(1);
    expect(result.current.data?.[0].status).toBe("pending");
  });
});
```
