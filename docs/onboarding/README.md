# オンボーディングガイド

k1s0 プロジェクトへの参加者向けオンボーディング資料。Tier アーキテクチャに基づき、担当する階層ごとに必要な知識をまとめている。

## Tier アーキテクチャと対象読者

| Tier | 階層 | 対象読者 | 概要 |
| --- | --- | --- | --- |
| [tier1 (system)](./tier1/) | 共通基盤層 | 認証・設定・API ゲートウェイ等の共通基盤を開発する人 | 全サービスが依存する最上位層。認証、共通ライブラリ、インフラ基盤を担当 |
| [tier2 (business)](./tier2/) | 領域共通層 | 業務ドメインの共通サーバー・クライアント・ライブラリを開発する人 | 特定の業務領域（経理、FA 等）内で共通利用する基盤を担当 |
| [tier3 (service)](./tier3/) | 個別サービス層 | 業務サービスの実装・デプロイを担当する人 | 実際にデプロイされる個別業務サービスの開発を担当 |

## 依存方向

```
service (tier3) → business (tier2) → system (tier1)
```

下位層は上位層に依存する。逆方向の依存は禁止。

## 読み進め方

1. 自分が担当する Tier のフォルダを開き、`01-概要.md` から順に読む
2. 下位 Tier の開発者は、依存先の上位 Tier の資料にも目を通すことを推奨
   - tier3 開発者 → tier2 + tier1 も参照
   - tier2 開発者 → tier1 も参照
3. 各章末の「関連ドキュメント」から詳細な設計書へ進む

## 前提知識

全 Tier 共通で以下を前提とする。

| 分野 | 内容 |
| --- | --- |
| 言語 | Go / Rust / TypeScript / Dart のうち担当言語 |
| 設計 | クリーンアーキテクチャ、DDD の基礎概念 |
| ツール | Git、Docker、k1s0 CLI の基本操作 |
| インフラ | コンテナ、Kubernetes の基礎概念 |
