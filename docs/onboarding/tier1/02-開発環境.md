# 開発環境セットアップ

## 必要なツール一覧

system tier の開発には以下のツールが必要である。

### 言語・ランタイム

| ツール | バージョン | 用途 |
|-------|-----------|------|
| Rust (rustup) | stable latest | サーバー・CLI・ライブラリ開発 |
| Go | 1.22+ | BFF サーバー・ライブラリ開発 |
| Node.js | 20 LTS | TypeScript ライブラリ・React クライアント |
| Flutter | 3.x | Dart ライブラリ・Flutter クライアント |

### 開発ツール

| ツール | 用途 |
|-------|------|
| Docker / Docker Compose | ローカル環境構築・コンテナビルド |
| buf | Protocol Buffers コンパイル・lint・breaking change 検出 |
| sqlx-cli | DB マイグレーション管理（`cargo install sqlx-cli`） |
| protoc | Protocol Buffers コンパイラ |
| grpcurl | gRPC エンドポイントのテスト |
| kubectl | Kubernetes クラスタ操作 |
| helm | Helm Chart 管理 |
| terraform | インフラプロビジョニング |
| k1s0 CLI | プロジェクト構築・コード生成・運用操作 |

### セキュリティ脆弱性スキャンツール

依存関係やコードに含まれる既知の脆弱性を検出するために、以下のツールを導入する。CI でも実行されるが、ローカル開発時にも定期的にスキャンすることを推奨する。

| ツール | インストール | 用途 |
|-------|-------------|------|
| cargo-audit | `cargo install cargo-audit` | Rust の依存クレートに対する既知の脆弱性スキャン（RustSec Advisory DB 参照） |
| govulncheck | `go install golang.org/x/vuln/cmd/govulncheck@latest` | Go の依存モジュールおよびコード内で実際に呼び出される脆弱な関数の検出 |

```bash
# Rust プロジェクトの脆弱性スキャン
cargo audit

# Go プロジェクトの脆弱性スキャン
govulncheck ./...
```

### エディタ推奨

- VS Code + Dev Container 拡張（推奨）
- rust-analyzer, Go 拡張, ESLint, Prettier

## Dev Container による環境構築

Dev Container を利用することで、ツールのインストールを自動化し、チーム全体で統一された開発環境を即座に構築できる。

### 手順

1. VS Code で Dev Container 拡張をインストール
2. リポジトリをクローン後、VS Code でフォルダを開く
3. コマンドパレットから **"Dev Containers: Reopen in Container"** を選択
4. `.devcontainer/` の設定に基づき、全ツールが自動インストールされる

Dev Container には以下が事前設定されている。

- Rust toolchain（stable + clippy + rustfmt）
- Go toolchain
- Node.js + pnpm
- Flutter SDK
- buf, protoc, sqlx-cli, grpcurl
- Docker-in-Docker（コンテナ内からコンテナビルド可能）
- kubectl, helm, terraform

## リポジトリのクローンと sparse-checkout

k1s0 はモノリポのため、全体をクローンするとサイズが大きい。k1s0 CLI の sparse-checkout 機能を利用して、必要な部分のみ取得することを推奨する。

### 全体クローン

```bash
git clone <repository-url> k1s0
cd k1s0
```

### k1s0 CLI による sparse-checkout

```bash
# k1s0 CLI をインストール（初回のみ）
cargo install --path CLI/

# system tier のみをチェックアウト
k1s0 checkout --tier system

# 特定サーバーのみをチェックアウト
k1s0 checkout --tier system --component server/auth
```

sparse-checkout では以下が自動的に含まれる。

- 選択した tier/コンポーネント
- `docs/` 配下の関連設計書
- `proto/` 配下の共通 Proto 定義
- `.devcontainer/`, `docker-compose*.yml` 等の環境設定ファイル

## docker-compose によるローカル環境起動

system tier の開発には複数のミドルウェアが必要となる。docker-compose で一括起動する。

### インフラサービス

```bash
# 全インフラサービスの起動
docker compose -f docker-compose-infra.yml up -d

# 起動されるサービス:
# - PostgreSQL 17    (port: 5432)
# - Keycloak 26.0    (port: 8080)
# - Kafka + ZooKeeper (port: 9092)
# - Redis            (port: 6379)
# - Vault            (port: 8200)
# - Jaeger           (port: 16686)
# - Prometheus       (port: 9090)
# - Grafana          (port: 3000)
```

### 可観測性サービス

```bash
# 可観測性スタックのみ起動
docker compose -f docker-compose-observability.yml up -d

# - Jaeger (分散トレーシング)
# - Prometheus (メトリクス収集)
# - Grafana (ダッシュボード)
# - Loki (ログ集約)
```

### 起動確認

```bash
# 全サービスの状態確認
docker compose -f docker-compose-infra.yml ps

# Keycloak 管理コンソール: http://localhost:8080
# Grafana ダッシュボード:  http://localhost:3000
# Jaeger UI:              http://localhost:16686
# Prometheus:             http://localhost:9090
```

### DB マイグレーション

```bash
# 環境変数設定
export DATABASE_URL="postgresql://k1s0:k1s0@localhost:5432/k1s0"

# マイグレーション実行
sqlx migrate run --source regions/system/database/migrations/
```

## k1s0 CLI の基本操作

k1s0 CLI は対話式インターフェース（dialoguer）を提供し、日常的な開発タスクを効率化する。

### 主要コマンド

```bash
# プロジェクト構成の確認
k1s0 status

# 新規サーバーの作成（対話式）
k1s0 create server

# 新規ライブラリの作成（対話式）
k1s0 create library

# コード生成（Proto → Go/Rust）
k1s0 generate proto

# テスト実行
k1s0 test --tier system

# ビルド
k1s0 build --tier system

# デプロイ（ローカル K8s）
k1s0 deploy --env local --tier system
```

### 対話式フロー

`k1s0 create server` を実行すると、以下の情報を対話形式で入力する。

1. サーバー名
2. 実装言語（Rust / Go）
3. 使用するミドルウェア（DB, Kafka, Redis 等）
4. gRPC サービス定義の有無
5. デプロイ先の Namespace

入力後、テンプレートに基づいてプロジェクト構成、Dockerfile, Helm Chart, CI 設定等が自動生成される。
