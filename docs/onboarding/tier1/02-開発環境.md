# 開発環境セットアップ

## 必要なツール一覧

system tier の開発には以下のツールが必要である。

### 言語・ランタイム

| ツール | バージョン | 用途 |
|-------|-----------|------|
| Rust (rustup) | 1.93+ | サーバー・CLI・ライブラリ開発 |
| Go | 1.24+ | BFF サーバー・ライブラリ開発 |
| Node.js | 22+ | TypeScript ライブラリ・React クライアント |
| Flutter | 3.24+ | Dart ライブラリ・Flutter クライアント |

### 開発ツール

| ツール | 用途 |
|-------|------|
| Docker / Docker Compose | ローカル環境構築・コンテナビルド |
| buf | Protocol Buffers コンパイル・lint・breaking change 検出 |
| sqlx-cli | DB マイグレーション管理（`cargo install sqlx-cli`） |
| protoc | Protocol Buffers コンパイラ |
| grpcurl | gRPC エンドポイントのテスト |
| kubectl | Kubernetes クラスタ操作 |
| helm | Helm Chart 管理 |
| terraform | インフラプロビジョニング |
| k1s0 CLI | プロジェクト構築・コード生成・運用操作 |

### k1s0 CLI のインストール

```bash
# リポジトリルートで実行
cargo install --path CLI/crates/k1s0-cli

# インストール確認
k1s0 --version
```

devcontainer 利用者はコンテナ起動時に自動インストールされるため不要。

### セキュリティ脆弱性スキャンツール

依存関係やコードに含まれる既知の脆弱性を検出するために、以下のツールを導入する。CI でも実行されるが、ローカル開発時にも定期的にスキャンすることを推奨する。

| ツール | インストール | 用途 |
|-------|-------------|------|
| cargo-audit | `cargo install cargo-audit` | Rust の依存クレートに対する既知の脆弱性スキャン（RustSec Advisory DB 参照） |
| govulncheck | `go install golang.org/x/vuln/cmd/govulncheck@latest` | Go の依存モジュールおよびコード内で実際に呼び出される脆弱な関数の検出 |

```bash
# Rust プロジェクトの脆弱性スキャン
cargo audit

# Go プロジェクトの脆弱性スキャン
govulncheck ./...
```

### エディタ推奨

- VS Code + Dev Container 拡張（推奨）
- rust-analyzer, Go 拡張, ESLint, Prettier

## Dev Container による環境構築

Dev Container を利用することで、ツールのインストールを自動化し、チーム全体で統一された開発環境を即座に構築できる。

### 手順

1. VS Code で Dev Container 拡張をインストール
2. リポジトリをクローン後、VS Code でフォルダを開く
3. コマンドパレットから **"Dev Containers: Reopen in Container"** を選択
4. `.devcontainer/` の設定に基づき、全ツールが自動インストールされる

Dev Container には以下が事前設定されている。

- Rust toolchain（stable + clippy + rustfmt）
- Go toolchain
- Node.js + pnpm
- Flutter SDK
- buf, protoc, sqlx-cli, grpcurl
- Docker-in-Docker（コンテナ内からコンテナビルド可能）
- kubectl, helm, terraform

## リポジトリのクローンと sparse-checkout

k1s0 はモノリポのため、全体をクローンするとサイズが大きい。k1s0 CLI の sparse-checkout 機能を利用して、必要な部分のみ取得することを推奨する。

### 全体クローン

```bash
git clone <repository-url> k1s0
cd k1s0
```

### k1s0 CLI による sparse-checkout

```bash
# k1s0 CLI をインストール後、対話式ウィザードで初期化
k1s0 init
# または引数なしで起動して「プロジェクト管理 > プロジェクト初期化」を選択
k1s0
```

ウィザードで以下を選択する:
1. プロジェクト名
2. sparse-checkout の有無
3. チェックアウトする Tier（system / business / service）

sparse-checkout では選択した tier に加え、`docs/`・環境設定ファイルが自動的に含まれる。

## docker-compose によるローカル環境起動

system tier の開発には複数のミドルウェアが必要となる。docker-compose で一括起動する。

### インフラサービス

```bash
# インフラサービスの起動（PostgreSQL, Redis, Kafka, Keycloak, Vault）
just local-up
# または認証バイパス付き（ローカル開発推奨）
just local-up-dev

# 起動されるサービス:
# - PostgreSQL 17    (port: 5432)
# - Keycloak 26.0    (port: 8180)
# - Kafka            (port: 9092)
# - Redis            (port: 6379)
# - Vault            (port: 8200)
```

### 可観測性サービス

```bash
# 可観測性スタック（Jaeger, Prometheus, Grafana, Loki）を追加起動
just observability-up

# Jaeger UI:    http://localhost:16686
# Prometheus:   http://localhost:9090
# Grafana:      http://localhost:3200
```

### 起動確認

```bash
# 全サービスの状態確認
docker compose ps

# Keycloak 管理コンソール: http://localhost:8180 (admin/dev)
```

### DB マイグレーション

```bash
# 環境変数設定
export DATABASE_URL="postgresql://k1s0:k1s0@localhost:5432/k1s0"

# マイグレーション実行
sqlx migrate run --source regions/system/database/migrations/
```

## k1s0 CLI の基本操作

k1s0 CLI は対話式インターフェース（dialoguer）を提供し、日常的な開発タスクを効率化する。引数なしで起動するとメニューが表示される。

### 起動

```bash
k1s0
```

メニューカテゴリ:
- **よく使う操作** — プロジェクト初期化・ひな形生成・ビルド・テスト・ローカル開発
- **コード生成** — 設定スキーマ型生成・ナビゲーション型生成・イベントコード生成
- **プロジェクト管理** — バリデーション・依存関係マップ・テンプレートマイグレーション
- **運用** — デプロイ・マイグレーション管理

### 日常操作（just コマンド）

```bash
# テスト実行（全言語）
just test

# 言語指定テスト
just test-rust
just test-go

# 単一サービスのテスト（例: auth サーバー）
just test-service regions/system/server/rust/auth

# ビルド
just build-rust

# lint
just lint

# ログ確認
just logs <service名>
```
