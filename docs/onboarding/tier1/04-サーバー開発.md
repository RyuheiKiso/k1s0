# サーバー開発

## Rust サーバー実装パターン

system tier の大半のサーバーは Rust で実装される。主要な技術スタックは以下の通り。

| ライブラリ | 用途 |
|-----------|------|
| axum | HTTP サーバー（REST エンドポイント、ヘルスチェック） |
| tonic | gRPC サーバー・クライアント |
| sqlx | PostgreSQL 接続（型安全なクエリ、マイグレーション） |
| tokio | 非同期ランタイム |
| serde | シリアライズ/デシリアライズ |
| tracing | 構造化ログ、分散トレーシング |
| k1s0-server-common | 認証、設定、DB、gRPC、Kafka、RBAC、トレース等の共通機能 |

### エントリポイント（main.rs）

`K1s0App` を使用してサーバーを初期化する。これにより、設定読み込み、DB 接続、トレーシング初期化等のボイラープレートが削減される。

```rust
use k1s0_server_common::app::K1s0App;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let app = K1s0App::builder()
        .with_config("config.yaml")
        .with_database()
        .with_tracing()
        .with_auth()
        .build()
        .await?;

    // リポジトリ・ユースケースの構築
    let repo = PgTenantRepository::new(app.db_pool());
    let usecase = CreateTenantUseCase::new(repo);

    // gRPC サービスの構築と起動
    let grpc_service = TenantGrpcService::new(usecase);
    app.serve_grpc(grpc_service).await?;

    Ok(())
}
```

### gRPC ハンドラ実装

```rust
use tonic::{Request, Response, Status};

pub struct TenantGrpcService<U: CreateTenantUseCase> {
    usecase: U,
}

#[tonic::async_trait]
impl<U: CreateTenantUseCase> TenantService for TenantGrpcService<U> {
    async fn create_tenant(
        &self,
        request: Request<CreateTenantRequest>,
    ) -> Result<Response<CreateTenantResponse>, Status> {
        // 認証情報の抽出
        let claims = request.extensions().get::<JwtClaims>()
            .ok_or_else(|| Status::unauthenticated("missing auth"))?;

        // リクエストの変換とユースケース実行
        let input = request.into_inner().try_into()?;
        let tenant = self.usecase.execute(input).await?;

        Ok(Response::new(tenant.into()))
    }
}
```

### sqlx によるリポジトリ実装

```rust
use sqlx::PgPool;

pub struct PgTenantRepository {
    pool: PgPool,
}

impl PgTenantRepository {
    pub fn new(pool: PgPool) -> Self {
        Self { pool }
    }
}

#[async_trait]
impl TenantRepository for PgTenantRepository {
    async fn find_by_id(&self, id: &TenantId) -> Result<Option<Tenant>, DomainError> {
        let row = sqlx::query_as!(
            TenantRow,
            r#"SELECT id, name, status as "status: TenantStatus",
                      created_at, updated_at
               FROM tenants WHERE id = $1"#,
            id.as_uuid()
        )
        .fetch_optional(&self.pool)
        .await
        .map_err(|e| DomainError::Infrastructure(e.to_string()))?;

        Ok(row.map(|r| r.into()))
    }
}
```

## Go BFF 実装パターン

graphql-gateway と bff-proxy は Go で実装される。クライアントからのリクエストを受け、複数の gRPC サービスを集約する。

```go
// GraphQL リゾルバから gRPC サービスを呼び出す例
func (r *queryResolver) Tenant(ctx context.Context, id string) (*model.Tenant, error) {
    // gRPC クライアント経由でサービスを呼び出し
    resp, err := r.tenantClient.GetTenant(ctx, &pb.GetTenantRequest{
        Id: id,
    })
    if err != nil {
        return nil, err
    }

    return convertTenant(resp), nil
}
```

## config.yaml の構成と設定ルール

全サーバーは `config.yaml` によって設定される。構成は以下のセクションで標準化されている。

```yaml
# サーバー基本設定
server:
  host: "0.0.0.0"
  grpc_port: 50051           # gRPC ポート（必須）
  http_port: 8080            # HTTP ポート（ヘルスチェック、メトリクス）
  shutdown_timeout_secs: 30  # グレースフルシャットダウンのタイムアウト

# データベース設定
database:
  url: "${DATABASE_URL}"
  max_connections: 10
  min_connections: 2
  acquire_timeout_secs: 30
  idle_timeout_secs: 600

# 認証設定
auth:
  keycloak_url: "${KEYCLOAK_URL}"
  realm: "k1s0"
  client_id: "auth-service"
  client_secret: "${AUTH_CLIENT_SECRET}"

# Kafka 設定
kafka:
  brokers: "${KAFKA_BROKERS}"
  consumer_group: "service-name"
  topics:
    - name: "events.tenant"
      partitions: 6

# 可観測性設定
telemetry:
  service_name: "auth"
  otlp_endpoint: "${OTEL_EXPORTER_OTLP_ENDPOINT}"
  log_level: "info"

# RBAC 設定
rbac:
  policy_service_url: "${POLICY_SERVICE_URL}"
  cache_ttl_secs: 300
```

### 設定ルール

1. **環境変数**: シークレットや環境依存の値は `${ENV_VAR}` 形式で環境変数から注入する
2. **デフォルト値**: 合理的なデフォルト値を設定ファイルに記載する
3. **バリデーション**: サーバー起動時に全設定値のバリデーションを実行する
4. **型安全**: Rust では serde による型安全なデシリアライズを行う

## Proto 定義と buf によるコード生成

### Proto ファイル構成

```
regions/system/proto/
├── buf.yaml              # buf の設定
├── buf.gen.yaml          # コード生成設定
├── auth/
│   └── v1/
│       └── auth.proto    # Auth サービス定義
├── tenant/
│   └── v1/
│       └── tenant.proto  # Tenant サービス定義
└── common/
    └── v1/
        └── common.proto  # 共通メッセージ型
```

### Proto 定義の例

```protobuf
syntax = "proto3";

package k1s0.tenant.v1;

service TenantService {
  rpc CreateTenant(CreateTenantRequest) returns (CreateTenantResponse);
  rpc GetTenant(GetTenantRequest) returns (GetTenantResponse);
  rpc ListTenants(ListTenantsRequest) returns (ListTenantsResponse);
  rpc UpdateTenant(UpdateTenantRequest) returns (UpdateTenantResponse);
  rpc DeleteTenant(DeleteTenantRequest) returns (DeleteTenantResponse);
}

message CreateTenantRequest {
  string name = 1;
  map<string, string> metadata = 2;
}

message CreateTenantResponse {
  Tenant tenant = 1;
}

message Tenant {
  string id = 1;
  string name = 2;
  string status = 3;
  google.protobuf.Timestamp created_at = 4;
  google.protobuf.Timestamp updated_at = 5;
}
```

### buf によるコード生成

```bash
# lint 実行
buf lint

# breaking change 検出（main ブランチとの比較）
buf breaking --against '.git#branch=main'

# コード生成（Rust + Go）
buf generate

# 特定の言語のみ生成
buf generate --template buf.gen.rust.yaml
```

## 共通ライブラリの使い方

### k1s0-server-common

全 Rust サーバーが依存する共通ライブラリ。以下の機能を提供する。

```toml
# Cargo.toml
[dependencies]
k1s0-server-common = { path = "../../library/rust/k1s0-server-common" }
```

| モジュール | 機能 |
|-----------|------|
| `app` | K1s0App（サーバー初期化のボイラープレート削減） |
| `auth` | JWT 検証、Keycloak 連携 |
| `config` | config.yaml 読み込み、環境変数展開 |
| `database` | DB コネクションプール管理 |
| `grpc` | gRPC サーバー共通設定、インターセプター |
| `kafka` | Kafka プロデューサ/コンシューマ |
| `rbac` | RBAC ミドルウェア |
| `shutdown` | グレースフルシャットダウン |
| `response` | 統一レスポンス型 |
| `error` | エラー分類とステータスコード変換 |
| `trace` | OpenTelemetry トレーシング統合 |

### 使用例

```rust
use k1s0_server_common::{
    auth::JwtValidator,
    config::AppConfig,
    database::create_pool,
    grpc::GrpcServer,
    trace::init_tracing,
};
```

## 新規サーバー追加の手順

### 1. k1s0 CLI でスキャフォールド

```bash
k1s0 create server
# 対話式で以下を入力:
# - サーバー名: my-service
# - 言語: Rust
# - 使用ミドルウェア: PostgreSQL, Kafka
```

### 2. 生成されるファイル

```
regions/system/server/rust/my-service/
├── Cargo.toml
├── config.yaml
├── src/
│   ├── main.rs
│   ├── domain/
│   ├── usecase/
│   ├── adapter/
│   └── infrastructure/
├── migrations/
├── tests/
├── Dockerfile
└── helm/
    └── Chart.yaml
```

### 3. Proto 定義の追加

```bash
# Proto ファイルを作成
mkdir -p regions/system/proto/my-service/v1/
# my-service.proto を記述

# コード生成
buf generate
```

### 4. DB マイグレーション

```bash
# マイグレーションファイル作成
sqlx migrate add -r create_my_service_tables \
  --source regions/system/server/rust/my-service/migrations/

# マイグレーション実行
sqlx migrate run --source regions/system/server/rust/my-service/migrations/
```

### 5. 実装とテスト

1. domain 層のエンティティ・値オブジェクト・リポジトリ trait を定義
2. ユニットテストを記述（TDD）
3. usecase 層のビジネスロジックを実装
4. adapter 層の gRPC ハンドラ・リポジトリ実装を追加
5. 統合テストを追加
6. infrastructure 層で全体を結合

### 6. CI/CD とデプロイ

```bash
# ローカルでビルド・テスト
cargo build -p my-service
cargo test -p my-service

# Docker イメージビルド
docker build -t my-service:latest -f regions/system/server/rust/my-service/Dockerfile .

# Helm でデプロイ
helm install my-service regions/system/server/rust/my-service/helm/ \
  --namespace k1s0-system
```
