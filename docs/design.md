# デザイン

---

## 概要

システムの全体のアーキテクチャを定義する

---

## キーワード

- **BFF (Backend For Frontend)**  
    フロントエンドごとに最適化されたバックエンドを提供するアーキテクチャ。
- **マイクロサービス**  
    各機能を独立したサービスとして分割し、スケーラビリティと柔軟性を向上。
- **クリーンアーキテクチャ**  
    依存関係を明確にし、保守性を高める設計手法。

---

## 採用技術

システムに採用する技術を定義する。

### Frontend

フロントエンドに採用する技術を定義する。

#### NativeApp

| 言語   | プラットフォーム | 用途                                                |
| ------ | ---------------- | --------------------------------------------------- |
| Swift  | iOS              | iOS向けのネイティブアプリケーション開発に使用       |
| Kotlin | Android          | Android向けのネイティブアプリケーション開発に使用   |
| .NET   | Windows          | Windows向けのデスクトップアプリケーション開発に使用 |

#### Web

| 言語  | プラットフォーム | 用途                          |
| ----- | ---------------- | ----------------------------- |
| React | Web(SPA)         | Webアプリケーション開発に使用 |

#### Database

| 技術       | 用途                           |
| ---------- | ------------------------------ |
| PostgreSQL | データベース管理システムに使用 |

#### Gateway

| 技術 | 用途                      |
| ---- | ------------------------- |
| Kong | APIゲートウェイとして使用 |

### Backend

| 言語 | 用途                                           |
| ---- | ---------------------------------------------- |
| Go   | 高パフォーマンスなバックエンド開発に使用       |
| Rust | 安全性と効率性を重視したバックエンド開発に使用 |

### Observability

| 技術          | 用途                                                                                                                                        |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
| Prometheus    | メトリクスの収集と監視。<br/>プル型でサービスのメトリクスを収集し、アラーティングや可視化に使用                                             |
| Jaeger        | 分散トレーシング。<br/>リクエストの遅延解析やサービス間の依存関係の可視化に使用                                                             |
| OpenTelemetry | アプリケーションのインストルメンテーションと収集基盤。<br/>メトリクス・トレース・ログの統合収集とエクスポーター経由で各種バックエンドへ送信 |

これらを組み合わせて、メトリクス収集（Prometheus）、トレース（Jaeger）、およびアプリケーション内での計測（OpenTelemetry）を実現する。

ロードバランシング、認証、レート制限、モニタリングなどの機能を提供し、マイクロサービス間の通信を効率化

## ディレクトリ構成

ディレクトリ構成の基本方針を定義する。
reactのdictにような細部は記載しない。

```bash
k1s0
├─database
│  ├─backups                 # データベースのバックアップファイルを保存
│  ├─docker                  # データベース関連のDocker設定
│  ├─docs                    # データベースに関するドキュメント
│  ├─functions               # データベース操作用の関数群
│  ├─init                    # 初期化スクリプト
│  ├─migrations              # データベーススキーマのマイグレーションファイル
│  ├─schemas                 # データベーススキーマ定義
│  ├─seed                    # 初期データ投入用スクリプト
│  ├─tests                   # データベース関連のテスト
│  └─tools                   # データベース管理ツール
├─docs                        # プロジェクト全体のドキュメント
├─img                         # プロジェクトで使用する画像ファイル
├─infra                       # インフラ関連の設定
│  ├─gateway                 # APIゲートウェイ関連
│  │  └─kong                # Kongの設定ファイル
│  └─k8s                     # Kubernetes関連の設定
├─observability               # 可観測性ツールの設定
│  ├─alertmanager            # アラート管理ツールの設定
│  ├─grafana                 # ダッシュボードツールの設定
│  ├─jaeger                  # 分散トレーシングツールの設定
│  ├─loki                    # ログ収集ツールの設定
│  ├─opentelemetry           # メトリクス・トレース・ログの統合収集設定
│  └─prometheus              # メトリクス収集ツールの設定
├─scripts                     # 各種スクリプト
└─source                      # ソースコード
    ├─backend                 # バックエンド関連
    │  ├─bff                 # BFF (Backend For Frontend)
    │  │  ├─native-bff      # ネイティブアプリ向けBFF
    │  │  │  ├─adapters    # プラットフォームごとのアダプタ
    │  │  │  │  ├─android # Android向けアダプタ
    │  │  │  │  ├─ios     # iOS向けアダプタ
    │  │  │  │  └─windows # Windows向けアダプタ
    │  │  │  └─core        # BFFのコアロジック
    │  │  └─web-bff         # Webアプリ向けBFF
    │  ├─features            # 各機能の実装
    │  │  ├─go              # Go言語で実装された機能
    │  │  └─rust            # Rust言語で実装された機能
    │  └─framework           # フレームワーク関連
    │      ├─go              # Go言語のフレームワーク
    │      └─rust            # Rust言語のフレームワーク
    ├─frontend                # フロントエンド関連
    │  ├─native              # ネイティブアプリ
    │  │  ├─android         # Androidアプリ
    │  │  ├─ios             # iOSアプリ
    │  │  └─windows         # Windowsアプリ
    │  └─web                 # Webアプリ
    └─proto                   # プロトコル定義
        ├─gen                 # 自動生成されたコード
        └─src                 # プロトコル定義ファイル
        
```
