# ADR-0003: テンプレート fingerprint 戦略

## ステータス

承認（Accepted）

## 日付

2026-01-25

## コンテキスト

`k1s0 upgrade` コマンドでテンプレートの更新を安全に適用するためには、以下が必要:

1. **テンプレートの出自の特定**: 生成時にどのテンプレートバージョンを使用したか
2. **変更の検知**: テンプレートが更新されたかどうかの判定
3. **差分の計算**: 旧テンプレート → 新テンプレートの変更内容

Git リビジョンだけでは以下の問題がある:
- リポジトリ全体のコミットハッシュはテンプレート単位の変更を反映しない
- ローカル開発中の未コミット変更を検知できない
- 外部レジストリからのテンプレートでは Git 情報がない場合がある

## 決定

### fingerprint の定義

テンプレートの fingerprint は、テンプレートディレクトリ内のファイル内容から算出される **SHA-256 ハッシュ** とする。

```
fingerprint = SHA-256(
  for each file in sorted(template_files):
    relative_path + "\0" + file_content + "\0"
)
```

### 算出対象（含める/除外するファイル）

#### 含めるファイル

テンプレートディレクトリ配下のすべてのファイル（除外対象を除く）

#### 除外するパターン

| パターン | 理由 |
|---------|------|
| `.git/` | Git メタデータ |
| `target/` | Rust ビルド成果物 |
| `node_modules/` | Node.js 依存 |
| `dist/` | ビルド成果物 |
| `build/` | ビルド成果物 |
| `.DS_Store` | macOS メタデータ |
| `Thumbs.db` | Windows メタデータ |
| `*.pyc` | Python バイトコード |
| `__pycache__/` | Python キャッシュ |
| `.env` | 環境変数（ローカル設定） |
| `.env.local` | 環境変数（ローカル設定） |
| `*.log` | ログファイル |
| `.k1s0/` | k1s0 メタデータ（生成後のもの） |

#### 除外パターンの拡張

テンプレートごとに `.k1s0-ignore` ファイルで追加の除外パターンを指定できる（将来拡張）。

### 決定論的な算出

fingerprint は常に同じ結果になる必要がある:

1. **ファイル順序**: 相対パスでソート（辞書順）
2. **パス区切り**: `/` で統一（OS 非依存）
3. **改行コード**: そのまま（変換しない）
4. **エンコーディング**: バイナリとして扱う

### manifest への保存

`.k1s0/manifest.json` に以下の形式で保存:

```json
{
  "template": {
    "name": "backend-rust",
    "version": "0.1.0",
    "path": "CLI/templates/backend-rust/feature",
    "fingerprint": "e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855"
  }
}
```

### upgrade 時の差分計算フロー

```
1. manifest から旧 fingerprint を取得
2. 新テンプレートの fingerprint を算出
3. fingerprint が一致 → 変更なし、終了
4. fingerprint が不一致 → 差分計算へ
   4.1. 旧テンプレートのファイル一覧（template_snapshot から）
   4.2. 新テンプレートのファイル一覧
   4.3. ファイル単位のチェックサム比較
   4.4. 追加/変更/削除を検出
```

### 選択肢の比較

| 選択肢 | 長所 | 短所 |
|--------|------|------|
| A. Git コミットハッシュのみ | シンプル、既存ツール活用 | テンプレート単位の変更を反映しない |
| B. ファイル内容のハッシュ（採用） | テンプレート変更を正確に検知 | 算出コストがある |
| C. タイムスタンプ | 算出が高速 | 信頼性が低い |

### 採用理由

- テンプレートの変更を正確に検知できる
- Git に依存しない（レジストリ対応）
- 決定論的で再現可能

## 帰結

### 正の帰結

- `k1s0 upgrade` が安全にテンプレート更新を提案できる
- 不要な更新（変更なし）をスキップできる
- テンプレートの変更履歴が manifest で追跡可能

### 負の帰結

- 大きなテンプレートでは算出に時間がかかる可能性
- ファイル数が多いと IO 負荷が増える

### リスクと軽減策

| リスク | 軽減策 |
|--------|--------|
| 算出パフォーマンス | 並列読み込み、キャッシュの導入 |
| ハッシュ衝突 | SHA-256 を使用（衝突確率は無視できる） |
| 除外パターンの漏れ | CI でテンプレート変更時に fingerprint 差分を確認 |
