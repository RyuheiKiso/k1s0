| ID プレフィックス | 意味 | 所在 |

| `DS-SW-COMP-*` | ソフトウェアコンポーネント方式設計 | `04_概要設計/20_ソフトウェア方式設計/01_コンポーネント方式設計/` |

- `05_実装/00_ディレクトリ設計/` — ルートレイアウト / tier1-3 / infra / operation / 共通資産 / スパースチェックアウト運用 / トレーサビリティ

## 情報を探す時の推奨順序

1. **ID が既に分かっている場合** — 本索引で ID プレフィックスから所在ディレクトリを特定し、該当ディレクトリ内を Glob する。
2. **トピックだけ分かっている場合** — 本索引のディレクトリ一覧で近い章を特定し、その章の README や `00_方針` / `00_設計方針` を先に読む。
3. **横断的な調査が必要な場合** — `Agent(subagent_type=Explore)` に投げ、生データをサブエージェント内に隔離する。親コンテキストには要約だけ返す。
