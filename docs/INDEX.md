# docs 索引

`docs/` 配下は 600 ファイル超・53,000 行超の大規模ドキュメント群である。特定の情報を探す時は Glob で総当たりせず、まず本索引で該当ディレクトリと ID 体系を特定し、続いて該当ディレクトリの `README.md` をサブ索引として辿る、という 2 段階の階層ナビゲーションを徹底する。全 md を親コンテキストに読み込むとトークンを急速に消費するため、**横断調査は `docs-explorer` サブエージェント（[定義](../.claude/agents/docs-explorer.md)、Haiku ベース）に委譲し、親コンテキストには要約のみを返す**運用が基本。

本索引と各 `README.md` の役割分担は以下:

- **本索引 (`docs/INDEX.md`)** — 主要ディレクトリ一覧・ID 体系・ADR 系列・探索パターンの俯瞰。部外者・サブエージェントが最初に読む。
- **各ディレクトリの `README.md`** — 配下のファイル構成・読み順・責務境界を示すサブ索引。本索引から個別ファイルへの遷移点として機能する。
- **トレーサビリティ索引群** — 要件 ID / 設計 ID / ADR の網羅的な突合せに特化した既存索引。特定 ID の所在を一覧で引く時に使う。

## 階層ナビゲーション（最短動線）

各主要ディレクトリの `README.md` をサブ索引として使う。本索引 → サブ索引 → 目的のファイル、の順で辿ること。サブ索引には配下の章構成・読み順・格納予定ファイルが網羅されているため、ディレクトリを盲目的に Glob する必要はない。

## 情報を探す時の推奨順序

トークン消費を抑える観点から、**階層を降りるたびに 1 ファイルずつ開く**ことを推奨する。複数ファイルを同時に Read する前に、次に示すサブエージェント活用を先に検討する。

1. **ID が既に分かっている場合** — 本索引の ID 体系表で所在ディレクトリを特定し、その `README.md` を開く。多くの場合、目的ファイルが 1 Read で特定できる。
2. **トピックだけ分かっている場合** — 本索引の「階層ナビゲーション」表で近い章を特定し、その `README.md` を開く。章内の読み順と配下ファイル構成が書かれているため、1 ファイル特定まで到達可能。
3. **複数ファイルにまたがる調査が必要な場合（横断調査）** — `Agent(subagent_type=docs-explorer)` に委譲する（定義: [.claude/agents/docs-explorer.md](../.claude/agents/docs-explorer.md)、Haiku ベースで本索引の ID 体系と探索動線を予め学習済み）。親からは依頼趣旨を渡すだけでよく、サブエージェント側が INDEX → サブ README → 目的ファイルの動線を自走する。生データはサブエージェント内に隔離され、親コンテキストには要約のみが返るため、横断調査で発生する数千行の md 読み込みを親から切り離せる。

### サブエージェント委譲の目安

- 2 ファイル以上を読む必要が見込まれた時点で `docs-explorer` に委譲する（1 ファイル直読と 2 ファイル以上の横断は原則別扱い）。
- 調査対象が複数の主要ディレクトリにまたがる場合は必ず `docs-explorer` に委譲する。
- 特定の 1 ファイルを Read するだけなら親で直接読む方が起動コストより安い（`docs-explorer` 側も 1 ファイル直読を依頼された場合は冒頭で親に差し戻す設計）。

### 典型パターン

動線判断のベースライン。迷ったら本節と照合する。

- **「セキュリティ関連を網羅的に見たい」→ `docs-explorer` 必須**
  対象: 要件定義 `E_セキュリティ.md` + 概要設計 `50_非機能/05_セキュリティ方式設計.md` + ADR-SEC-* 3 件 + アーキテクチャ `03_セキュリティ/` 複数。合計 2,000 行超になるため親で直接読むのは不可。
- **「NFR-E-SIR-003 を確認したい」→ 親で直読（2 Read）**
  動線: [`03_要件定義/30_非機能要件/README.md`](03_要件定義/30_非機能要件/README.md) → `E_セキュリティ.md` の該当節。
- **「ADR-DEP-001 の影響章を知りたい」→ 親で直読（3 Read）**
  動線: [`02_構想設計/adr/README.md`](02_構想設計/adr/README.md) → 該当 ADR → [`04_概要設計/90_付録/02_ADR索引.md`](04_概要設計/90_付録/02_ADR索引.md) の該当行で影響章を確認。

### 自動ガード（hook）

ハーネス側の hook (`.claude/hooks/docs-read-guard.py`) が `docs/` 配下の Read 回数を 1 セッション内で累積監視する。3 件目以降は `systemMessage` で警告しつつ Read を許可、5 件目以降はブロックして `Agent(subagent_type=docs-explorer)` への委譲を強制する。警告が出た時点で本節の委譲目安に従い、以降は `docs-explorer` に切り替えること。しきい値は `.claude/hooks/docs-read-guard.py` 冒頭の `WARN_AT` / `BLOCK_AT` で調整可能。
