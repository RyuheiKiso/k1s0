# 10_アーキテクチャ: 構造要件

本カテゴリは、k1s0 プラットフォームの**構造**に関する要件を定義する。tier 境界、通信契約、ランタイム選定、内部基盤の採用判断が対象である。企画で宣言した「tier 構造」と構想設計で裏付けた「Dapr ファサード + 自作 Rust」の方針を、実装者が参照可能な粒度の要件に落とし込む。

本カテゴリが守るべき一貫性は 2 点ある。第一に、**tier2/tier3 から内部言語は不可視** という tier 境界のルール。これを守らないと、tier1 の内部リファクタが外部契約の破壊に波及する。第二に、**tier1 内部は Protobuf gRPC 必須** という通信ルール。これを緩めると、Rust と Go のファサード境界で型不整合が発生し、契約管理が破綻する。

## 読み方

`00_infra.md` が全 tier の基盤前提を定義し、`01_tier1.md` 〜 `03_tier3.md` が tier 階層ごとの要件を積み上げる。続いて `04_integration.md` で外部境界を、`05_api_sdk_contract.md` で内外の契約管理を、`06_rule_engine.md` と `07_eventing.md` で横断的な制御メカニズムを扱う。
