# 60_事業_契約: 事業性と契約

本カテゴリは、k1s0 プラットフォームの**事業性・契約・ガバナンス** に関する要件を定義する。UX、国際化、テナントオンボーディング、トレーニング、コスト、ライセンス、課金メータリング、契約、サプライチェーン、ガバナンス、AI ガバナンス、アナリティクス、サステナビリティを対象とする。

本カテゴリは技術要件より**事業判断が支配的** な領域である。したがって要件の根拠には企画の定量試算 ([`../../01_企画/04_定量試算/`](../../01_企画/04_定量試算/)) と法務サマリ ([`../../01_企画/05_法務サマリ/`](../../01_企画/05_法務サマリ/)) の引用が頻出する。要件化の目的は、事業判断を「契約・運用・監査で確認可能な条項」に翻訳することにある。

## 優先度の傾向

本カテゴリは**全体として SHOULD / MAY が多め** である。MUST は「k1s0 自体のライセンス選定（AGPL 内部利用の明示）」「テナント契約の免責条項」など、法務・契約上の不可逆判断に集中する。日常運用に属する要件は SHOULD にとどめ、事業成長に応じて MAY を昇格させる運用を想定する。
