# 30_セキュリティ_データ: 安全性と法令適合

本カテゴリは、k1s0 プラットフォームの**セキュリティ要件・データ管理要件・法令適合要件** を定義する。JTC 環境で稟議通過および監査応答の肝となる領域であり、要件の粒度と根拠の精度が直接的に事業リスクに連動する。

本カテゴリの要件は、他カテゴリと比べて**未達時の可逆性が著しく低い** 点に特徴がある。パフォーマンス要件は運用改善で事後に取り戻せるが、監査ログの改ざんや PII の漏洩は事後復旧が不可能である。したがって全要件で受け入れ基準を「検知可能性 / 防御可能性 / 復旧可能性」の 3 軸で検証する。
