# QUA-PRF: パフォーマンス要件

tier1 API のレイテンシ、スループット、リソース効率に関する数値要件を定義する。企画書で宣言した「p99 < 500ms」を受け入れ基準と検証手段の両面で具体化する。

## 要件本体

> 本ファイルは骨格のみ。本文は後続タスクで記述する。
>
> 想定要件 (draft):
>
> - `QUA-PRF-001` tier1 外向け API の p99 レイテンシ ≤ 500ms
> - `QUA-PRF-002` tier1 内部 gRPC の p99 レイテンシ ≤ 50ms
> - `QUA-PRF-003` ルール評価のスループット要件
> - `QUA-PRF-004` Kafka イベント処理の遅延上限
> - `QUA-PRF-005` 負荷試験の必須化（MVP-0 時点）
