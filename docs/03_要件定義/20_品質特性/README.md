# 20_品質特性: 非機能要件

本カテゴリは、k1s0 プラットフォームの**非機能要件** を数値で定義する。パフォーマンス / 可用性 / 可観測性 / SLA-SLO / テナント分離 / DR-BCP / アクセシビリティを対象とし、いずれも「測定可能な閾値」として記述する。

企画書で宣言した「tier1 API p99 < 500ms」「可用性 99.9%」といった目標値は、構想設計段階では見込み値だった。本カテゴリでは、これらを**合意された閾値** として採番し、未達時のリリース差し戻しルール、測定手段、達成 Phase を明示する。前回の技術査読で High リスクに挙がった「p99 500ms の実測未検証」問題は、本カテゴリの `00_performance.md` で受け入れ基準と検証手段を明文化することで解消する。

## 数値の扱い

全要件で数値は「**測定単位・測定地点・測定期間・許容違反頻度**」の 4 点を明記する。たとえば「p99 500ms」ではなく「**tier1 外向き API の p99 レイテンシが直近 30 日の 5 分間ウィンドウで 500ms 以下であること。Dapr サイドカーを含む。許容違反は月次 0.1% 以内**」のように記述する。曖昧な数値目標は要件として扱わず、MAY または SHOULD に降格させる。
