# 00_要件定義方針

本カテゴリは、k1s0 要件定義書の「どのように書くか・どのように合意するか」を規定するメタ文書群である。要件そのものは 10_業務要件 以降で記述し、本カテゴリはその書き方・プロセス・共通語彙を提供する。

## 本カテゴリの位置付け

要件定義段階の初期に関係者が共通認識を持たないまま執筆を始めると、数週間後に「この語はそういう意味だったのか」「この要件の優先度判定はそのロジックだったのか」「あのステークホルダーは凍結後に巻き込むのか」の揉め直しが発生する。本カテゴリ 5 文書を先に固定化し、以降の全要件文が本カテゴリの用語・プロセス・前提を引用する形で記述することで、揉め直しを リリース時点 の早期に集約する。

## ファイル構成

- [01_要件定義プロセス.md](01_要件定義プロセス.md): IPA 共通フレーム 2013 に準拠した 6 ステップのプロセスと改訂ルール
- [02_ステークホルダー.md](02_ステークホルダー.md): 12 ロールの関心事とレビュー関与点
- [03_前提と制約.md](03_前提と制約.md): 事業・技術・組織・法令・規程・インフラ・既存資産・ベンダー・時間の 9 軸の前提と制約
- [04_用語集.md](04_用語集.md): プラットフォーム層構造、採用 OSS、k1s0 固有語、IPA 標準用語、セキュリティ用語、業務プロセス用語
- [05_要件トレーサビリティ方針.md](05_要件トレーサビリティ方針.md): 要件 ID 体系と双方向トレース運用ルール
- [06_エラータクソノミ.md](06_エラータクソノミ.md): エラーコード体系、分類、再試行ポリシー、応答フォーマット
- [07_リスクレジスタ.md](07_リスクレジスタ.md): プロジェクト全域のリスク一元管理、対応戦略と責任者
- [08_ADR索引.md](08_ADR索引.md): 構想設計 ADR の索引と要件トレーサビリティ
- [09_データ辞書.md](09_データ辞書.md): 標準識別子、命名規則、データ型、機密度分類

## 読み順

初見の読者は 01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09 の番号順で読むことで、プロセス → 誰が関わるか → 何を所与とするか → 共通語 → ID 体系 → エラー規約 → リスク → 技術判断 → データ規約と、メタ情報が段階的に具体化する。

既に要件文を読んでいる途中で参照する場合は、04_用語集・06_エラータクソノミ・09_データ辞書 を辞書として使い、03 の前提 ID（P-*）・制約 ID（C-*）から要件へ逆引きする場合は 05 のトレース方針に従う。ADR と要件の相互参照には 08 を使う。

## 改訂

本カテゴリの文書はすべて「要件の書き方の契約文書」として扱う。改訂は 01_要件定義プロセス.md の改訂フローに従い、中規模以上の改訂は構想設計リードと運用リードのレビューを必須とする。
