# COM-ERR: エラー分類

k1s0 プラットフォーム内のエラーを**分類体系** として定義し、tier 間での伝播ルール、ログ記録、ユーザー通知の扱いを統一する。

エラー分類の目的は 2 つある。第一に、運用時のアラート閾値・インシデント重大度判定を機械化できるように、エラーに一貫したメタ情報を持たせる。第二に、tier3 アプリから tier1 / tier2 に通じる呼び出し経路で、エラーが tier 境界を越える際の情報損失を最小化する。

## 要件本体

### COM-ERR-001: エラー分類の 5 カテゴリ

- 優先度: MUST
- Phase: Phase 1a
- 関連: `ARC-API-*`, `OPS-INC-*`

エラーは以下 5 カテゴリで分類する。

- **ClientError**: tier 上位層の誤用。入力不正、認証不足、認可不足など。HTTP 4xx 相当。
- **ServerError**: tier 内部のバグ。NPE、未処理例外、前提条件違反など。HTTP 5xx 相当。
- **DependencyError**: 依存 OSS / 外部サービスの障害。DB 接続失敗、Kafka 到達不能、Keycloak 応答タイムアウトなど。
- **ResourceExhaustion**: クォータ超過、レート制限、ディスク枯渇、接続プール枯渇。
- **TransientError**: リトライで回復する見込みのある一時的な失敗。ネットワーク瞬断など。

5 カテゴリで覆えない「その他」を作らない。新種のエラーが出た場合は 5 カテゴリのいずれかに分類し、必要なら副分類を追加する。

**受け入れ基準**

- Protobuf スキーマでエラーコードの enum が 5 カテゴリに分類されている
- 分類不明のエラーは CI で警告

### COM-ERR-002: 伝播ルール

- 優先度: MUST
- Phase: Phase 1a
- 関連: `ARC-API-*`

tier1 → tier2 → tier3 の呼び出し経路で、エラーは以下のルールで伝播する。

- **ClientError**: そのまま伝播。tier3 アプリは画面上でユーザーに具体的な理由を提示する。
- **ServerError**: tier 境界で詳細をマスクし、呼び出し元には「内部エラー」として伝える。詳細は監査ログにのみ残す。
- **DependencyError**: 呼び出し元には「一時的に利用できない」として伝え、リトライ推奨を明示する。
- **ResourceExhaustion**: Retry-After ヘッダ相当のメタ情報を付与する。
- **TransientError**: 呼び出し元はリトライを試みることを想定する。

**受け入れ基準**

- tier1 クライアントライブラリがエラーカテゴリに応じた型を提供する
- tier2 / tier3 での誤った伝播（ServerError の内部詳細漏洩など）は CI / 静的解析で検出

---

### COM-ERR-003: ログ記録粒度

- 優先度: MUST
- Phase: Phase 1a
- 関連: `QUA-OBS-*`, `SEC-AUD-*`

各エラーカテゴリのログ記録粒度を定義する。

- **ClientError**: INFO レベルで記録。監査ログ対象（「誰が何にアクセスしようとして拒否されたか」）。
- **ServerError**: ERROR レベルで記録。スタックトレース含む。アラート対象。
- **DependencyError**: WARN レベルで記録。依存別のメトリクス化。
- **ResourceExhaustion**: WARN レベルで記録。テナント別の集計が必須。
- **TransientError**: DEBUG レベルで記録。過剰なログ出力を避ける。

**受け入れ基準**

- ロギングライブラリがカテゴリ別の推奨レベルを自動適用する
- 誤ったレベル設定は CI 警告

---

### COM-ERR-004: ユーザー向けメッセージ規約

- 優先度: SHOULD
- Phase: Phase 1b
- 関連: `BIZ-UX-*`, `BIZ-I18-*`

tier3 アプリのエンドユーザーに表示するエラーメッセージは、**技術用語を含まない** ことを原則とする。エラーカテゴリに応じてテンプレート化し、翻訳リソースとして多言語対応する。

**受け入れ基準**

- ユーザー向けメッセージテンプレートが日本語・英語の 2 言語で整備されている
- 技術詳細はトラッキング ID を通じて運用担当が参照できる
