# 20_機能要件

本カテゴリは、k1s0 が提供する機能要件を定義する。IPA 共通フレーム 2013 の「システム機能要件」に対応し、tier1 が公開する 11 種類の API と、外部システム連携、情報モデルを要件化する。業務要件（10_業務要件）で特定した利用シナリオを、実装可能な粒度の機能仕様として固定化する層である。

## 本カテゴリの位置付け

tier1 は Dapr ファサード層と自作 Rust 領域のハイブリッド構成で、tier2/tier3 には 11 種類の公開 API として提供される。本カテゴリでは内部実装を原則意識しない（tier2/tier3 から見える振る舞いのみ記述する）が、非機能要件との関係で内部構造が判断に必要な場合は、構想設計（[../../02_構想設計/](../../02_構想設計/)）を参照する形で補足する。

機能要件は「何ができるか」の記述に徹し、「どう作るか」は構想設計・基本設計で決定する。要件が曖昧だと、構想設計で「これは要件に書いてないから自由裁量」と判断された結果、tier2/tier3 利用者の期待と乖離した実装になる事故が起きる。

## 要件 ID 体系

機能要件は `FR-<カテゴリ>-<サブカテゴリ>-<通番>` で採番する。カテゴリは `T1`（tier1 API）、`EXT`（外部連携）、`INFO`（情報要件）。ID 体系の詳細は [../00_要件定義方針/05_要件トレーサビリティ方針.md](../00_要件定義方針/05_要件トレーサビリティ方針.md) を参照。

## 機能要件の記述方針

各要件は「現状 → 要件達成後 → 崩れた時」の 3 段落構造で散文記述する。機能要件では「現状」は「素の言語・素の Dapr SDK を使った場合の開発者体験」とし、「要件達成後」は「k1s0 公開 API を使った場合の開発者体験」の差分で記述する。崩れた時の影響は「tier2/tier3 開発者に何が起きるか」「既存資産とどう衝突するか」を具体化する。

表は章末のサマリのみ使用する。本文の受け入れ基準は箇条書きで列挙する。優先度・数値・出典には必ず根拠を併記する（[../00_要件定義方針/05_要件トレーサビリティ方針.md](../00_要件定義方針/05_要件トレーサビリティ方針.md) の受け入れ基準の書き方を参照）。

## tier1 API の一覧

tier1 が公開する 11 API の責務を以下に箇条書きで示す。詳細は各 API ファイルを参照。

1. **Service Invoke**: サービス間同期呼び出し（gRPC / HTTP/1.1 互換プロキシ）
2. **State**: 状態管理・キャッシュ（Valkey バックエンド）
3. **PubSub**: イベント配信（Kafka バックエンド、at-least-once 保証）
4. **Secrets**: 秘密情報管理（OpenBao バックエンド、動的シークレット）
5. **Binding**: 外部入出力バインディング（MinIO / SMTP / HTTP 等）
6. **Workflow**: ワークフロー・Saga（Dapr Workflow + Temporal 長期実行補完）
7. **Log**: 構造化ログ（W3C Trace Context 自動注入）
8. **Telemetry**: 分散トレース・メトリクス（OpenTelemetry ラッパ）
9. **Decision**: ビジネスルール決定表評価（ZEN Engine）
10. **Audit / Pii**: 監査改ざん防止ログ、個人情報マスキング（Rust 自作領域）
11. **Feature**: Feature Flag 段階ロールアウト（flagd バックエンド）

## 非機能要件との接続

機能要件には非機能特性（性能、可用性、セキュリティ）が紐付く。本カテゴリでは機能の振る舞いのみ記述し、具体的な数値水準は 30_非機能要件/ を参照する。例: Service Invoke API の機能要件（FR-T1-INVOKE-001）は「同期呼び出しができる」ことを記述し、「p99 < 500ms」は NFR-B-PERF-002 で記述する。

## 変更管理

機能要件の変更は [../00_要件定義方針/01_要件定義プロセス.md](../00_要件定義方針/01_要件定義プロセス.md) の改訂プロセスに従う。tier1 API の互換破壊変更（signature 変更、セマンティクス変更）は重大改訂として扱い、下流影響（tier2/tier3 利用コードへの影響）を 80_トレーサビリティ/03_構想設計マトリクス.md で確認する。
