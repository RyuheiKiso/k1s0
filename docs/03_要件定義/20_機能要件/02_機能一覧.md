# 機能一覧

本書は、機能要件カテゴリ（20_機能要件/）に存在するすべての要件 ID のインデックスである。散文主体の要件詳細は各 API ファイル（10_tier1_API要件/ 配下）・20_外部連携要件.md・30_情報要件.md で記述し、本書は一覧性のみを担う。

## 使い方

稟議説明資料・監査エビデンス・構想設計着手時の機能範囲確認で本書を参照する。要件 ID から詳細ファイルへ辿ることを目的とした索引であり、要件の意味は各詳細ファイル側で正とする。本書と詳細ファイルが乖離した場合は詳細ファイル側を優先する。

## 優先度分布

本カテゴリの要件総数と優先度分布をサマリする。新規要件追加時は本表を更新する。

| Phase | MUST | SHOULD | COULD | 計 |
|---|---|---|---|---|
| 1a | 13 | 0 | 0 | 13 |
| 1b | 13 | 11 | 2 | 26 |
| 1c | 2 | 3 | 2 | 7 |
| 2+ | 0 | 0 | 1 | 1 |
| **合計** | **28** | **14** | **5** | **47** |

## Phase 到達度

Phase ごとの累計要件数（新規 + 前 Phase 累計）を示す。Phase 1a は 13 要件、Phase 1b は 13+26=39 要件、Phase 1c は 39+7=46 要件、Phase 2 以降で 46+1=47 要件となる。実装完了判定はこの累計に対する達成率で Phase ゲートを評価する（95% 以上で通過）。

## 不足要件の検出

本書のインデックスに登場しないが、業務要件（10_業務要件）や構想設計で言及されている機能は、要件の記述漏れである可能性がある。四半期ごとの整合レビューで、業務要件 → 機能要件のリンクが切れている BR-* を検出し、本書への追加を検討する。過渡期はステークホルダーからの指摘ベースで収集する。
