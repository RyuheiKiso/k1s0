# 50_開発者体験: 開発者の生産性

本カテゴリは、k1s0 を利用する**内部チーム（tier1 開発者）と顧客チーム（tier2/tier3 開発者）の開発者体験** を要件化する。テスト戦略、日常的な開発ワークフロー、フィーチャーフラグ運用を対象とする。

開発者体験は稟議資料で直接的な数値目標に落ちにくいが、**tier3 オンボーディング速度** と **tier1 レグレッション検知率** という 2 軸に集約すると測定可能になる。前者は「雛形 CLI で生成したサンプルアプリが 30 分以内に動作するか」、後者は「PR がマージされた時点でどの範囲の回帰を検知できるか」という具体的指標に落とす。

## 測定指標

本カテゴリの要件は、**定量的な開発者体験指標** と紐付ける。代表例は以下。

- tier3 新規開発者のオンボーディングリードタイム（環境構築〜初回デプロイまでの時間）
- PR 開始からマージまでの中央値リードタイム
- ローカル開発環境での E2E テスト成功率
- 雛形 CLI 生成直後に CI が通る確率

これらの指標は `20_品質特性/02_observability.md` の可観測性要件と連動し、計測自体を自動化することで要件の達成度を運用時に追跡可能にする。
