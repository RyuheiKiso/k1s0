# 03. 参考文献

本書は k1s0 要件定義書を策定するにあたり参照した外部文献を集約する。IPA 文献、CNCF 仕様、法令、業界標準、書籍、論文、主要ベンダー文献を網羅し、要件の根拠となる原典へ辿れる状態を維持する。

## 本書の位置付け

要件定義で「業界標準に則っている」と主張する場合、参照元の明示は必須である。稟議・外部監査・契約交渉の場で「どの標準のどこを参照したのか」を即座に示せなければ主張は説得力を失う。本書は要件定義書本編・ADR・構想設計から参照される全外部文献の索引として機能する。

ステータスの意味は以下のとおり。

- **✓ 最新版**: 策定時点の版が最新であることを確認済み
- **△ 改訂検討中**: 改訂版がドラフトで進行中、動向監視
- **⚠ 動向監視**: 規制環境・業界動向の変化が影響する可能性あり
- **✗ 陳腐化**: 参照元が廃止・置換された（発見時は即時是正）

### 鮮度チェック自動化（将来構想）

参照文献が改訂・廃止されていないかを機械的にチェックする仕組みを Phase 1c 以降で整備する。GitHub Actions で以下を月次実行する想定。

```bash
# 参照 URL の到達性チェック
for url in $(grep -rohE 'https?://[^\s)]+' docs/03_要件定義/); do
  curl -sI -o /dev/null -w "%{http_code} $url\n" "$url"

# 発行年が古い文献の抽出（5 年超）
awk -v y=$(date +%Y) '{ if (match($0, /[0-9]{4}/, a) && y - a[0] > 5) print }' \
  docs/03_要件定義/90_付録/03_参考文献.md
```

- **非機能要求グレード 2018 年改訂版**: IPA 刊。非機能要件の共通グレード体系。[../90_付録/02_非機能要求グレード判定.md](02_非機能要求グレード判定.md) で判定

## 法令・ガイドライン

日本国内で k1s0 が遵守すべき法令群。

- **個人情報保護法（平成 15 年法律第 57 号、2022 年改正施行）**: 個人情報取扱事業者義務、漏えい通知義務。NFR-G-PRV-003、NFR-H-COMP-001、BC-LGL-002 で参照
- **マイナンバー法（行政手続における特定の個人を識別するための番号の利用等に関する法律）**: 特定個人情報の安全管理措置
- **金融商品取引法（J-SOX 内部統制）**: 財務報告に関わるシステムの統制評価。NFR-H-COMP-002、BC-LGL-006 で参照
- **サイバーセキュリティ基本法**: 重要インフラ該当時の対応義務
- **電気通信事業法**: 通信の秘密、電気通信事業者該当の判定
- **経済安全保障推進法**: 重要物資サプライチェーン管理（該当時）
- **個人情報保護委員会ガイドライン**: 委員会公表の逐条解説・事例集

## 業界標準・認証基準

プラットフォームとして参照する国際・業界標準。

- **ISO/IEC 27001:2022（ISMS）**: 情報セキュリティマネジメント。NFR-H-COMP-003 で参照
- **ISO/IEC 27017:2015（クラウドサービス情報セキュリティ）**: クラウド固有の管理策。NFR-H-COMP-003 で参照
- **ISO/IEC 27018:2019（クラウド上の個人情報保護）**: PII 処理者向け管理策
- **ISO/IEC 25010:2023（システム品質モデル）**: 品質特性モデル
- **NIST Cybersecurity Framework 2.0**: 組織セキュリティ成熟度
- **NIST SP 800-88 Rev.1（Media Sanitization）**: メディアサニタイズ。BC-SUS-005、NFR-G-LIF-002 で参照
- **NIST SP 800-207（Zero Trust Architecture）**: ゼロトラスト原則
- **WCAG 2.1 Level AA**: Web アクセシビリティ。BC-UX-002 で参照
- **OWASP ASVS 4.0**: アプリセキュリティ検証標準
- **OWASP Top 10:2021**: Web 脆弱性カテゴリ
- **PCI DSS 4.0**: 該当時（現時点で k1s0 スコープ外）
- **Payment Services Directive 2（PSD2）**: 金融 API 標準（将来拡張時）

## メンテナンス

外部文献は改訂頻度が高い。法令改正（個人情報保護法、AI Act 等）、ISO 改訂、CNCF 卒業/アーカイブ、OSS ライセンス変更があれば即時反映する。四半期ごとに Product Council で参照文献の陳腐化レビューを実施、古くなった参照は新版に置換または削除する。
