# ドキュメント・コードレビュー チェックリスト

本チェックリストは PR レビュー時に必須確認する観点をまとめる。レビュアーは該当するセクションを必ず通読し、不備があれば PR をブロックする。

## 全般（すべての PR に適用）

- [ ] PR タイトルが変更概要を 70 文字以内で正確に表現している
- [ ] PR 本文に「背景」「変更内容」「影響範囲」「動作確認方法」が記載されている
- [ ] 関連する要件 ID / ADR / Issue がリンクされている
- [ ] 破壊変更（Breaking Change）は明示されている

## ドキュメント変更

- [ ] アスキー図や罫線による図表が含まれていない
- [ ] 新規図は drawio で作成、img/ に配置、SVG で埋め込まれている
- [ ] 表のみで要件説明していない（散文が伴う）
- [ ] 要件 ID 体系（`BR-`, `FR-`, `NFR-*`, `OR-`, `DX-`, `BC-`, `RISK-`）に準拠
- [ ] 相対リンク（`../`, `./`）が正しく解決される
- [ ] 300 行超の大きい md は分割または README と分冊が検討されている
- [ ] CLAUDE.md の記述ポリシー（散文 + 図 + 具体値セル）に適合

## drawio 変更

- [ ] 背景は白矩形を最初の要素として配置している
- [ ] アプリ層 / ネットワーク層 / インフラ層 / データ層 の 4 レイヤ色分離を遵守
- [ ] 矢印がボックスやテキストに重なっていない
- [ ] ラベルが矢印を覆い隠していない
- [ ] ラベル付き矢印の接続元と接続先の間隔がラベル幅の 1.5 倍以上
- [ ] SVG が同フォルダにエクスポートされている
- [ ] ダークテーマでも視認可能（白矩形 + 色コントラスト確認）

## コード変更（Rust / Go / TypeScript）

- [ ] 各行の 1 行上に日本語コメントが記載されている
- [ ] ファイル先頭に日本語ファイル説明コメントが記載されている
- [ ] 1 ファイル 300 行以内を遵守（超過は分割）
- [ ] Lint / Format がグリーン
- [ ] 単体テストが追加・更新されている
- [ ] 契約テスト（Pact）に影響がある場合、Consumer / Provider 両方で検証済み
- [ ] カバレッジが基準値以上（tier1 コア 80%、致命的 95%、tier2/tier3 70%）

## tier1 API 変更

- [ ] Protobuf IDL（40_tier1_API契約IDL/ 配下の該当 API ファイル）が更新されている
- [ ] 破壊変更なら Version 増加、後方互換維持策が明示
- [ ] tier2/tier3 クライアント SDK への影響評価
- [ ] Audit イベント追加/変更の検討
- [ ] SLO への影響評価
- [ ] シーケンス図（50_tier1_APIシーケンス.md）の更新検討

## セキュリティ影響

- [ ] 新規 OSS 採用ならライセンス確認（AGPL / BUSL / SSPL 回避）
- [ ] Secret を環境変数・コードに埋め込んでいない
- [ ] PII を扱う場合 Pii API Masking 経由
- [ ] 新規 API は認証・認可を付与
- [ ] ネットワーク境界変更は脅威モデル更新
- [ ] Sigstore 署名、SBOM 生成を満たしている

## 性能・可用性影響

- [ ] SLO 計測点の変更有無を明記
- [ ] 負荷試験シナリオへの影響評価
- [ ] メトリクス・ログ・トレースの追加・変更
- [ ] FMEA 該当項目の更新検討

## 依存関係変更

- [ ] OSS バージョン変更は CHANGELOG 確認、破壊変更有無を記載
- [ ] 新規依存は License Policy 適合
- [ ] Renovate が自動生成した PR か、手動追加か明記
- [ ] SBOM が自動更新される設定になっている

## インフラ変更

- [ ] Argo CD 管理下の Helm Chart / Kustomize overlay で変更
- [ ] Kyverno ポリシー違反なし
- [ ] dev / staging での検証実績
- [ ] 本番適用手順の Runbook 有無

## ADR 変更

- [ ] コンテキスト・決定・検討肢・帰結・実装タスクの 5 段構成
- [ ] 検討した選択肢を複数列挙（最低 3 件）
- [ ] ポジティブ / ネガティブ帰結を両方記載
- [ ] 参考文献 / 関連 ADR を記載
- [ ] ADR 索引（[00_要件定義方針/08_ADR索引.md](../03_要件定義/00_要件定義方針/08_ADR索引.md)）への追加

## リリース判定（prod 昇格時）

- [ ] Critical / High 欠陥ゼロ
- [ ] SLO 達成状況確認
- [ ] 関連 Runbook 整備済み
- [ ] 監査・法務・セキュリティの承認
- [ ] Canary / Rollback 計画の明示
