# ADR-0001: Rust Edition 2024 の採用

- ステータス: Accepted
- 起票日: 2026-04-14
- 決定日: 2026-04-14
- 起票者: kiso ryuhei
- 関係者: インフラチーム / 起案者 / 決裁者

## コンテキスト

k1s0 は tier1 の自作領域 (ZEN Engine 統合 / JTC 固有機能 / 雛形生成 CLI) の実装言語として Rust を採用する方針を [`../../01_企画/03_tier1設計/02_内部言語ハイブリッド.md`](../../01_企画/03_tier1設計/02_内部言語ハイブリッド.md) で確定済みである ([ADR-0002](./ADR-0002-tier1-language-hybrid.md))。
一方、Rust の Edition (言語仕様の世代) を 2021 とするか 2024 とするかは未決定であった。

本プロジェクトは長期運用 (Phase 1 〜 Phase 5) を前提とし、初期セットアップ時点で Edition を確定する必要がある。
Edition を後から移行する場合、`cargo fix --edition` などの自動移行ツールはあるものの、依存クレートの対応状況確認・CI 再設定・コードレビューの追加コストが発生する。

## 決定

Rust Edition 2024 を採用する。
`Cargo.toml` の `edition = "2024"` を必須とし、CI で利用する Rust toolchain を Edition 2024 対応版に固定する。

## 検討した選択肢

### 選択肢 A: Edition 2021

- 概要: 長期間運用されており安定している Edition。
- メリット: 実績が豊富、周辺クレートとの互換性検証が完了している。
- デメリット: 2024 で導入される言語機能・改善 (例: `let-else` の更なる整理、`async fn in trait` の安定化進展、ライフタイム省略規則の改善 等) が利用できない。後年 Edition 2024 へ移行する作業が発生する。

### 選択肢 B: Edition 2024

- 概要: 2024 年版の最新 Edition。
- メリット: 新しい言語機能・人間工学改善を初期から活用できる。Edition 移行作業が当面不要。
- デメリット: 一部の古いクレートが Edition 2024 環境下での挙動を保証していない可能性がある。

## 決定理由

本プロジェクトは新規立ち上げかつ長期運用前提であり、Edition の移行コストを将来的に支払うよりも、立ち上げ時点で最新版を採用する方が合理的である。
具体的には次の点を重視した。

- **移行コストの先送りを避ける**: Edition 2021 を採用しても、いずれは 2024 への移行が必要になる。立ち上げ時点で移行を済ませておけば、稼働中サービスを抱えた状態での移行作業が不要となる。
- **依存クレートのリスクは限定的**: k1s0 の自作領域で利用する主要クレート (ZEN Engine 関連、tonic、tokio など) は活発にメンテナンスされており、Edition 2024 への対応は十分に進んでいる見込み。
- **コード規約との親和性**: 本プロジェクトの規約 (1 ファイル 300 行以内 / 日本語コメント必須) は新規プロジェクトであることを前提に設計されており、Edition 移行を避ける理由 (既存資産との互換性) は存在しない。

## 影響

### ポジティブな影響

- 新しい言語機能・人間工学改善を初期から活用でき、コード品質と開発体験が向上する。
- Edition 移行作業 (将来発生しうる工数) が不要となる。
- 新規参画者が学ぶ Rust が常に最新 Edition と整合する。

### ネガティブな影響 / リスク

- 一部のクレートが Edition 2024 に未対応である場合、回避策の検討が必要になる。
  - 緩和策: 採用するクレートは事前に Edition 2024 対応状況を確認する。問題が発生した場合は当該クレートを別バージョンに固定するか代替を検討する。
- Rust toolchain のバージョン要件が新しくなり、開発者の環境構築に最新版インストールが必要となる。
  - 緩和策: `rust-toolchain.toml` でバージョンを固定し、`rustup` で自動取得させる。

### 移行・対応事項

- すべての Rust クレートで `Cargo.toml` に `edition = "2024"` を明記する。
- リポジトリ直下に `rust-toolchain.toml` を配置し、Edition 2024 に対応した安定版バージョンに固定する。
- CI (GitHub Actions) で利用する Rust toolchain も同バージョンに固定する。
- 雛形生成 CLI が出力する Rust プロジェクトのテンプレートを Edition 2024 ベースとする。
