# 02_infra — インフラ設計と決定記録

本ディレクトリは k1s0 の **インフラ実装フェーズ** で参照する設計資料・決定記録 (ADR)・運用手順を格納する。

`docs/01_企画/` が「Proposed 状態の企画資料」であるのに対し、本ディレクトリは **Accepted 状態の確定情報** を扱う。
企画段階で合意された方針を、実装に着手できる粒度まで落とし込む。

---

## ディレクトリ構成

| パス | 内容 |
|---|---|
| [`00_ADR/`](./00_ADR/) | Architecture Decision Records。確定した技術決定の不変記録 |

> 以下は Phase 1a (MVP-0) 着手に伴って順次追加する想定。
>
> | パス | 内容 |
> |---|---|
> | `01_MVP-0/` | MVP-0 実装計画 (WBS / DoD / デモシナリオ) |
> | `02_要件/` | インフラ要件定義 (VM スペック / ネットワーク / IP 設計 / DNS / 内部 CA / バックアップ要件) |
> | `03_CLI/` | 雛形生成 CLI 仕様 |

---

## 読み方

1. [`00_ADR/`](./00_ADR/) を一読して、本プロジェクトで確定済みの技術決定とその根拠を把握する。
2. 個別の実装作業に着手する際は、関連する ADR と、(整備後の) `01_MVP-0/` `02_要件/` を参照する。
