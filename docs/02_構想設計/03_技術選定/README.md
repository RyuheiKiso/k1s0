# 03_技術選定

02・03 章で定めた構造を、実際に動く OSS にマッピングする章。選定方針は「オンプレ前提・OSI 承認済みライセンス・CNCF 推奨・運用実現性」の 4 軸に固定されており、本章はその軸で 40+ OSS を分類・採否判定している。[`01_俯瞰/00_選定方針.md`](./01_俯瞰/00_選定方針.md) が章内ナビの役割を兼ねるため、本 README はそれを補完する短い索引である。

## 想定読者

技術リード（調達・ライセンス・スキル獲得の判断）、経営層（採用検討根拠として「なぜこれか」を確認）、将来の差し替え判断者（長期戦略を確認する者）。

## 前章と後章

- 前章: [`../01_アーキテクチャ/`](../01_アーキテクチャ/) / [`../02_tier1設計/`](../02_tier1設計/) — 本章で構造を OSS にマッピングする。
- 後章: [`../04_CICDと配信/`](../04_CICDと配信/) — 選定した OSS を CI/CD に組み込む。

## 03 章との責務境界

本章は「03 章で決まった設計判断を、どの OSS でどう実装するか」を書く。**設計判断そのもの（何を隠蔽するか・どの責務を tier1 が引き受けるか・API 契約をどう切るか）は [`../02_tier1設計/`](../02_tier1設計/) の責務であり、本章では扱わない**。例えば「Go + Rust ハイブリッド」という設計意図は [`../02_tier1設計/01_設計の核/02_内部言語ハイブリッド.md`](../02_tier1設計/01_設計の核/02_内部言語ハイブリッド.md) で展開し、その決定を支える工数・性能の定量根拠は本章の [`01_俯瞰/18_言語選定の定量分析.md`](./01_俯瞰/18_言語選定の定量分析.md) に置く、という分業設計になっている。申請の場で「なぜその OSS か」と問われたら本章、「なぜそう設計したか」と問われたら 03 章を開く運用を徹底する。

## 本章の構成

本章は「俯瞰 / 中核 OSS / 周辺 OSS」の 3 つのサブディレクトリに分かれている。旧構成は 20 本の md が同一階層に並び、俯瞰的な方針ドキュメントと個別 OSS の詳細が視覚的に区別できなかった。申請の場面では前者を、実装落とし込みの場面では後者を開くという読み方の差をディレクトリで分離することで、どの粒度で参照しているかを URL から即判別できるようにした。

| サブディレクトリ | 責務 | 収録内容 |
|---|---|---|
| [`01_俯瞰/`](./01_俯瞰/) | 章全体の判断軸・マトリクス・タイムライン・長期戦略 | 選定方針 / 選定一覧（責務マトリクス） / OSS 長期戦略 / 言語選定の定量分析 / 採用段階別採用タイムライン |
| [`02_中核OSS/`](./02_中核OSS/) | 実行基盤の骨格に組み込まれる OSS（入替が最も重い層） | 実行基盤中核 OSS (k8s / Istio / Envoy Gateway / Kafka / Dapr / Go / Rust) / ルールエンジン (ZEN Engine) |
| [`03_周辺OSS/`](./03_周辺OSS/) | 中核を補う周辺 OSS（責務ドメインごとに細分化） | 周辺 OSS 総覧 / IaC / イベントスキーマレジストリ / ストレージと運用補助 / シークレット管理 / ネットワークとテレメトリ基盤 / Protobuf とロールアウト / トレーシングと DB 補助 / マニフェストとワークフロー / 運用 UI とシークレット同期 / オートスケーラー / サプライチェーンセキュリティ / プロファイリング |

## 読み順

本章は **俯瞰 → タイムライン → 中核 → 周辺** の順で読むことを推奨する。個別 OSS の根拠ファイルが多数に分岐しているため、先に俯瞰レイヤで全体像を取らないと迷子になる。

1. [`01_俯瞰/00_選定方針.md`](./01_俯瞰/00_選定方針.md) — 判断軸（**必ず最初に読む**）
2. [`01_俯瞰/04_選定一覧.md`](./01_俯瞰/04_選定一覧.md) — **責務マトリクス + 採用 40+ OSS のインデックス表 (俯瞰の起点)**
3. [`01_俯瞰/19_採用段階別採用タイムライン.md`](./01_俯瞰/19_採用段階別採用タイムライン.md) — **採用初期 (デモ / パイロット / 運用体制確立) / 採用後の運用拡大時 / 採用側のマルチクラスタ移行時 × OSS の時系列マトリクス**
4. [`02_中核OSS/01_実行基盤中核OSS.md`](./02_中核OSS/01_実行基盤中核OSS.md) — k8s / Istio / Envoy Gateway / Kafka / Dapr / Go / Rust の詳細
5. [`01_俯瞰/18_言語選定の定量分析.md`](./01_俯瞰/18_言語選定の定量分析.md) — Go+Rust ハイブリッドの定量根拠
6. [`01_俯瞰/17_OSS長期戦略.md`](./01_俯瞰/17_OSS長期戦略.md) — 継続性リスクと差し替え・自作移行シナリオ
