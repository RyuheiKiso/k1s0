# src コーディング規約

本ファイルは `src/` 配下のソースコードに適用する規約を定める。ルート `CLAUDE.md` の補完として、コード作業時にのみ読まれることを想定する。

## コメント

- コードの各行の 1 行上に、必ず日本語でコメントを記載すること。例外はない。
- コードファイルの先頭には、必ず日本語でファイルの説明コメントを記載すること。例外はない。

## ファイル構成

- 1 ファイルあたりの行数は 500 行以内とすること。いかなる例外も認めない。
- 500 行を超える場合は、適切にファイルを分割すること。

## 言語別の基本

- Rust Edition 2024（自作領域: ZEN Engine 統合 / crypto / 雛形 CLI / JTC 固有機能）
- Go（Dapr ファサード層: stable Dapr Go SDK を使用）
- tier1 内部のサービス間通信は Protobuf gRPC 必須。
- tier2/tier3 から内部言語（Rust / Go）は不可視。クライアントライブラリと gRPC エンドポイントのみ公開する。

## 依存方向

`tier3 → tier2 → (sdk ← contracts) → tier1 → infra` の一方向。逆向きは禁止。
