# PII 漏えい規制報告・本人通知 Runbook

> **severity**: SEV1
> **owner**: legal / security-sre
> **deadline**: 72h（個人情報保護委員会速報）/ 30日（確報）
> **last_updated**: 2026-04-28

## 1. 検出 (Detection)

本 Runbook は `pii-leak-detection.md` の **Step 4（影響範囲確定）** が完了し、
個人情報の漏えいが確定した時点で起動する。

個人情報保護法（令和 4 年施行）第 26 条に基づき、以下の要件を満たす場合は
**報告義務** が発生する。

- 要配慮個人情報が含まれる場合
- 不正アクセスに起因する場合
- 漏えい件数が 1,000 件を超える場合
- 財産的被害が生じるおそれがある場合
- 上記に該当しない場合も、100 件以上は自主報告を推奨する

**漏えい確定の起算点**: CPO（個人情報保護責任者）が漏えいを認識した時刻。

## 2. 初動 (Immediate Action)

### タイムライン管理

```
T+0h   漏えい確定（CPO 認識）
T+24h  [社内] 情シスマネージャ・監査担当・法務への通知完了
T+72h  [法定] 個人情報保護委員会への速報提出（締切）
T+30d  [法定] 個人情報保護委員会への確報提出（締切）
       [法定] 影響本人への通知（速やかに、確報前まで）
```

### Step 1: 報告義務の判定（〜1 時間）

1. 法務担当（`legal@k1s0.io`）に漏えい内容の詳細を連絡する。
2. 以下のチェックリストで報告義務を確認する。

   | チェック項目 | Yes/No | 法的根拠 |
   |---|---|---|
   | 要配慮個人情報（健康・信条・犯罪歴等）が含まれる | → Yes で即報告義務 | 個人情報保護法 26 条 |
   | 不正アクセスが原因と疑われる | → Yes で即報告義務 | 同上 |
   | 漏えい件数 ≥ 1,000 件 | → Yes で即報告義務 | 同上 |
   | 財産的被害（なりすまし詐欺等）の可能性 | → Yes で即報告義務 | 同上 |
   | GDPR 対象の EU 居住者データが含まれる | → 72h 以内に監督機関へ | GDPR 第 33 条 |

### Step 2: 個人情報保護委員会 速報（T+72h 以内）

3. 報告フォームにアクセスする: `https://www.ppc.go.jp/` → 漏えい等報告
4. 速報に必要な最低限の情報を記入する。

   ```
   【速報 記載項目（確定していない項目は「調査中」で可）】
   1. 事業者名・連絡先
   2. 漏えい等が発生した個人情報の種類・件数（概算可）
   3. 漏えい等が発生した原因（概要）
   4. 二次被害の可能性・有無
   5. 本人への通知の実施状況
   6. 公表の実施状況
   7. 再発防止措置の検討状況
   ```

5. 提出後、受付番号を記録し、法務と CPO に共有する。

### Step 3: 影響本人への通知

6. 通知対象者リストを確定する（`/tmp/affected_users.txt` を参照）。
7. 通知文書テンプレートを使用する。

   ```
   件名: 個人情報漏えいに関するお知らせ

   [氏名] 様

   この度、弊社のサービスにおいて、お客様の個人情報が漏えいした
   可能性があることが判明いたしました。

   【漏えいした情報の種類】
   <種類・件数>

   【漏えいの経緯】
   <概要>

   【弊社の対応】
   <封じ込め措置・再発防止策>

   【お客様へのお願い】
   <不審なメール・連絡への注意喚起等>

   【お問い合わせ先】
   privacy@k1s0.io（要置換）
   ```

8. 通知手段: メール（確認可能な場合）/ 書面（重大な場合） / 公表（個別通知困難な場合）。

## 3. 復旧 (Recovery)

1. 個人情報保護委員会との連絡窓口は **法務担当** が一元管理する。
2. 追加質問・調査依頼には **72 時間以内** に回答する。
3. 影響テナントの採用組織担当者（`customer-security@<adopter>.example`）にも個別連絡する。
4. GDPR 対象データが含まれる場合は各国 DPA（Data Protection Authority）への報告を確認する。

## 4. 原因調査 (Root Cause Analysis)

- 確報（30 日確報）作成のため、以下を根拠資料として準備する。
  - 漏えいデータの最終確定件数・種別
  - 漏えい経路・原因の技術詳細
  - 二次被害の発生有無・対応履歴
  - 本人通知の実施記録（通知日時・手段・通知数）
  - 再発防止措置の実施計画・実施結果

## 5. 事後処理 (Post-incident)

- 個人情報保護委員会への確報提出（T+30 日以内）
- 確報に基づくポストモーテム最終版の作成
- 個人情報保護委員会の改善指導がある場合は対応計画を策定し、期限内に報告する
- プライバシーポリシーの改訂が必要な場合は法務と協議し、公開する
- 四半期 PII スキャン・検知精度の向上
