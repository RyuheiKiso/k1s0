# 構想

---

## Goal

- 高速な開発サイクルを実現するframeworkの構築
- 各機能をモジュール化し、再利用性を高める
- チームごとに独立した開発環境を提供し、開発効率を向上させる
- 最新技術を取り入れ、将来の拡張性を確保する

---

## 開発環境

### OS

- windows

### エディタ

- VSCode

---

## チーム構成

### 開発基盤チーム

frameworkの開発をメインとする

作業ディレクトリ
- `framework\backend\go`
- `framework\backend\rust`
- `framework\frontend\flutter`
- `framework\frontend\react`

### 個別機能チーム

各機能の開発をメインとする

作業ディレクトリ
- `feature\backend\go\{feature_name}`
- `feature\backend\rust\{feature_name}`
- `feature\frontend\flutter\{feature_name}`
- `feature\frontend\react\{feature_name}`

---

## アーキテクチャ

- マイクロサービス(API First)
- クリーンアーキテクチャ

マイクロサービスの粒度は機能単位とする、複数機能をまとめることはない
例：ログイン機能、データベース機能、ログ機能

---

## framework提供機能

- 認証・認可
- ロギング(otel)
- エラーハンドリング
- データベース接続管理
- キャッシュ管理
- APIドキュメント生成
- モニタリング・アラート設定支援
- REST APIおよびGraphQLサポート
- gRPCサポート
- QUICサポート
- WebSocketサポート

---

## CLI

- プロジェクトの初期化
- 新規機能の雛形生成
- frameworkのバージョンアップ支援
- frameworkの機能を利用した新規機能の雛形を自動生成する
- frameworkのバージョンアップに伴うコード修正を自動化する
- 開発環境のセットアップ支援
- テストの自動化支援

---

## 流れ

1. 開発基盤チームがframeworkの基本機能を実装
2. 個別機能チームがCLIを利用してプロジェクトを初期化しframeworkを導入
3. 個別機能チームがCLIを利用して新規機能の雛形を生成し、開発を開始
4. 開発基盤チームがframeworkの機能を拡張・改善
5. 個別機能チームがCLIを利用してframeworkのバージョンアップを実施

## 技術スタック

- バックエンド: Go, Rust(axum + tokio)
- フロントエンド: Flutter, React
- CLI: Rust(clap)
- データベース: PostgreSQL
- キャッシュ: Redis
- コンテナオーケストレーション: Kubernetes
- モニタリング: Prometheus, Grafana
- ロギング: OpenTelemetry
- APIドキュメント: Swagger, GraphQL Playground
- 認証・認可: OAuth2, JWT
- メッセージング: RabbitMQ, Kafka
- バージョン管理: Git, GitHub
- テストフレームワーク: Jest, GoTest, Rust's built-in test framework
- コード品質ツール: ESLint, Prettier, GoLint, Clippy
- ドキュメンテーション: MkDocs, Docusaurus
- コンテナ化: Docker
- APIゲートウェイ: Kong
- サービスメッシュ: Istio
- 分散トレーシング: Jaeger, Zipkin
- 国際化・多言語対応: i18next, Lokalise
- APIテストツール: Postman
- ドキュメント生成ツール: Swagger Codegen, GraphQL Code Generator

## ディレクトリ構成図

### 方針

- **contracts/**: OpenAPI / protobuf / GraphQL schema など「サービス間の約束事」を集約（実装言語と独立）
- **framework/**: 各言語/フレームワークの横断機能（認証、otel、DB、エラハン等）をモジュール/パッケージとして提供
- **feature/**: 機能単位（=マイクロサービス単位）で完結する実装とデプロイ資材を配置
- **infra/**: 共有基盤（K8s/Istio/Kong/監視）など環境ごとの定義を集約
- **CLI/**: 雛形生成のテンプレートを同梱し、`feature/` と `framework/` を更新できるようにする
