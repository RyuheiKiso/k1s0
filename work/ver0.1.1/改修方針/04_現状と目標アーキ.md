## 現状分析

### 現行ディレクトリ構造

```
k1s0/
├── CLI/                          # CLI ツール
│   ├── crates/
│   │   ├── k1s0-cli/            # メイン CLI
│   │   ├── k1s0-generator/      # テンプレートエンジン
│   │   └── k1s0-lsp/            # LSP サーバー
│   └── templates/               # サービステンプレート
│       ├── backend-rust/
│       ├── backend-go/
│       ├── frontend-react/
│       └── frontend-flutter/
│
├── framework/                    # 共通基盤
│   ├── backend/
│   │   ├── rust/crates/         # Rust 共通クレート（11個）
│   │   ├── rust/services/       # 共通マイクロサービス
│   │   └── go/pkg/              # Go 共通パッケージ
│   └── frontend/
│       ├── react/packages/      # React 共通パッケージ（8個）
│       └── flutter/packages/    # Flutter 共通パッケージ
│
├── feature/                      # 個別業務機能
│   ├── backend/rust/
│   ├── backend/go/
│   ├── frontend/react/
│   └── frontend/flutter/
│
└── bff/                          # Backend For Frontend
```

### 現行の依存関係

```
feature/* → framework/*  （許可）
feature/* → feature/*    （禁止、ただし強制なし）
framework/* → feature/*  （禁止）
```

### 現行の Lint ルール

| ID        | 説明                          | 備考                                |
| --------- | ----------------------------- | ----------------------------------- |
| K001-K003 | manifest.json 検証            | -                                   |
| K010-K011 | ディレクトリ/ファイル存在検証 | -                                   |
| K020-K022 | コード規約検証                | K022 は Clean Architecture 依存方向 |
| K030-K032 | gRPC リトライ設定検証         | -                                   |

### 課題

1. **domain 層の不在**: 業務ドメイン共通機能の置き場所がない
2. **feature 間依存の制御なし**: 異なる feature 間の依存を検出できない
3. **チーム境界の不明確さ**: どのチームが何を管理するか曖昧

```
                            ▼

```

### 依存関係ルール詳細

| 依存元    | 依存先         | 許可/禁止 | 理由                                 |
| --------- | -------------- | --------- | ------------------------------------ |
| framework | 外部ライブラリ | ✓ 許可   | 基盤は外部ライブラリを使用可         |
| framework | domain         | ✗ 禁止   | 基盤が業務に依存すると再利用性が低下 |
| framework | feature        | ✗ 禁止   | 同上                                 |
| domain    | framework      | ✓ 許可   | 共通基盤の機能を使用可               |
| domain    | 外部ライブラリ | ✓ 許可   | 必要に応じて使用可                   |
| domain    | 他の domain    | ✗ 禁止   | ドメイン間の密結合を防止             |
| domain    | feature        | ✗ 禁止   | 上位層が下位層に依存してはならない   |
| feature   | framework      | ✓ 許可   | 共通基盤の機能を使用可               |
| feature   | 所属 domain    | ✓ 許可   | 自身が属するドメインの機能を使用可   |
| feature   | 他の domain    | ✗ 禁止   | ドメイン境界を超えた依存を防止       |
| feature   | 他の feature   | ✗ 禁止   | feature 間の密結合を防止             |
