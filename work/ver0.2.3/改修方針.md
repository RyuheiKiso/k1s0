# k1s0 v0.2.3 改修方針

## 概要

Docker と docker-compose をファーストクラスでサポートし、4つの機能を実装する。

- **全テンプレートに Dockerfile 追加**: backend-rust、backend-go、frontend-react に Dockerfile を新規追加。C#・Python は HEALTHCHECK 追加で改善
- **docker-compose.yml テンプレート追加**: 全バックエンド + frontend-react にローカル開発用 compose ファイルを追加
- **CLI docker コマンド追加**: `k1s0 docker build` / `k1s0 docker compose` ラッパー
- **CI/CD Docker ワークフロー追加**: Docker イメージのビルド・テストを自動化

## 機能目的

- `k1s0 new-feature` 後すぐに `docker compose up` でサービスが起動できるようにする
- 全テンプレートで一貫したコンテナ化パターンを提供する
- ローカル開発に必要な依存サービス（PostgreSQL、Redis）をワンコマンドで構築する

## 技術選定

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| Rust コンテナ | rust:1.85-slim + debian:bookworm-slim | 公式イメージ、マルチステージで軽量化 |
| Go コンテナ | golang:1.22-alpine + alpine:3.19 | 静的バイナリ、最小ランタイム |
| React コンテナ | node:20-slim + nginx:alpine | SPA ビルド + 静的配信 |
| Compose | Docker Compose v2 | Docker Desktop 標準、v1 は非推奨 |
| DB | PostgreSQL 16 Alpine | k1s0 標準 DB |
| Cache | Redis 7 Alpine | k1s0 標準キャッシュ |
| CLI コマンド | clap 4.5（既存） | 既存 CLI フレームワーク |
| テンプレート | Tera（既存） | 既存テンプレートエンジン |

## 機能 1: Dockerfile テンプレート追加

### 1.1 設計方針

- 全 Dockerfile はマルチステージビルド（build + runtime）
- ランタイムイメージは最小構成（slim/alpine）
- 非 root ユーザー（appuser）で実行
- `HEALTHCHECK` 命令を全テンプレートに追加
- 依存ファイルを先にコピーしてレイヤーキャッシュを最適化
- `config/` ディレクトリは COPY でイメージに含める（ローカル開発時は volume でオーバーライド）
- ENTRYPOINT + CMD 分離: バイナリを ENTRYPOINT、デフォルト引数（`--env default --config /app/config`）を CMD に指定
- プロキシ対応: 全 Dockerfile に `ARG HTTP_PROXY` / `ARG HTTPS_PROXY` / `ARG NO_PROXY` を宣言
- 中間ファイルの temp 配置: CLI docker コマンドの作業ファイルはシステム temp ディレクトリを使用

### 1.2 backend-rust Dockerfile.tera

```
CLI/templates/backend-rust/feature/Dockerfile.tera
```

```dockerfile
# {{ feature_name }} - Multi-stage Docker build
# Rust 1.85

# Stage 1: Build
FROM rust:1.85-slim AS build
WORKDIR /app

# Proxy support
ARG HTTP_PROXY
ARG HTTPS_PROXY
ARG NO_PROXY

# Install build dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    pkg-config libssl-dev \
    && rm -rf /var/lib/apt/lists/*

# Cache dependencies
COPY Cargo.toml Cargo.lock ./
RUN mkdir src && echo "fn main() {}" > src/main.rs \
    && cargo build --release \
    && rm -rf src

# Build application
COPY src/ src/
{% if with_grpc %}
COPY proto/ proto/
COPY buf.yaml buf.gen.yaml ./
{% endif %}
RUN touch src/main.rs && cargo build --release

# Stage 2: Runtime
FROM debian:bookworm-slim AS runtime
WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    ca-certificates curl \
    && rm -rf /var/lib/apt/lists/*

RUN groupadd -r appuser && useradd -r -g appuser -s /sbin/nologin appuser

COPY --from=build /app/target/release/{{ feature_name }} /app/{{ feature_name }}
COPY config/ /app/config/

EXPOSE 8080
{% if with_grpc %}
EXPOSE 50051
{% endif %}

HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD curl -f http://localhost:8080/healthz || exit 1

USER appuser

ENTRYPOINT ["/app/{{ feature_name }}"]
CMD ["--env", "default", "--config", "/app/config"]
```

```dockerfile
# {{ feature_name }} - Multi-stage Docker build
# Go 1.22

# Stage 1: Build
FROM golang:1.22-alpine AS build
WORKDIR /app

# Proxy support
ARG HTTP_PROXY
ARG HTTPS_PROXY
ARG NO_PROXY

# Cache dependencies
COPY go.mod go.sum ./
RUN go mod download

# Build application
COPY . .
{% if with_grpc %}
COPY proto/ proto/
{% endif %}
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o /app/server ./cmd/

# Stage 2: Runtime

WORKDIR /app

RUN addgroup -S appuser && adduser -S appuser -G appuser

COPY --from=build /app/server /app/server
COPY config/ /app/config/

EXPOSE 8080
{% if with_grpc %}
EXPOSE 50051
{% endif %}

HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD curl -f http://localhost:8080/healthz || exit 1

USER appuser

ENTRYPOINT ["/app/server"]
CMD ["--env", "default", "--config", "/app/config"]
```

### 1.4 frontend-react Dockerfile.tera

```
CLI/templates/frontend-react/feature/Dockerfile.tera
```

```dockerfile
# {{ feature_name }} - Multi-stage Docker build
# React (Node 20 + Nginx)

# Stage 1: Build
FROM node:20-slim AS build
WORKDIR /app

# Proxy support
ARG HTTP_PROXY
ARG HTTPS_PROXY
ARG NO_PROXY

RUN corepack enable pnpm

COPY package.json pnpm-lock.yaml ./
RUN pnpm install --frozen-lockfile

COPY . .
RUN pnpm build

# Stage 2: Runtime
FROM nginx:alpine AS runtime

COPY --from=build /app/dist /usr/share/nginx/html
COPY deploy/nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
    CMD wget -q --spider http://localhost:80/ || exit 1
```

## 機能 2: docker-compose.yml テンプレート追加

### 2.1 設計方針

- 全バックエンドテンプレートに docker-compose.yml.tera を追加
- 条件分岐: `with_db` で PostgreSQL、`with_cache` で Redis を追加
- シークレットは Docker secrets を使用（`secrets/` ディレクトリ）
- ヘルスチェック付き `depends_on` で起動順序を保証
- frontend-react には app + nginx のみ（DB/Redis なし）

### 2.2 バックエンド docker-compose.yml.tera

```yaml
# {{ feature_name }} - Docker Compose (ローカル開発用)
services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
    command: ["--env", "dev", "--config", "/app/config"]
    ports:
      - "8080:8080"
{% if with_grpc %}
      - "50051:50051"
{% endif %}
    volumes:
      - ./config:/app/config:ro
{% if with_db or with_cache %}
    depends_on:
{% endif %}
{% if with_db %}
      db:
        condition: service_healthy
{% endif %}
{% if with_cache %}
      redis:
        condition: service_healthy
{% endif %}
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8080/healthz"]
      interval: 10s
      timeout: 3s
      retries: 5

{% if with_db %}
  db:
    image: postgres:16-alpine
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: {{ feature_name_snake }}
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD_FILE: /run/secrets/db_password
    volumes:
      - db-data:/var/lib/postgresql/data
      - ./migrations:/docker-entrypoint-initdb.d
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 3s
      retries: 5
    secrets:
      - db_password
{% endif %}

{% if with_cache %}
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5
{% endif %}

{% if with_db %}
volumes:
  db-data:

secrets:
  db_password:
    file: ./secrets/db_password.txt
{% endif %}
```

## 機能 3: CLI docker コマンド追加

### 3.1 設計方針

- `k1s0 docker` サブコマンドグループを追加
- 内部で `docker` / `docker compose` コマンドを呼び出すラッパー
- manifest.json からサービス名を読み取り、タグを自動付与
- doctor チェックで Docker / Docker Compose の存在を確認

### 3.2 コマンド定義

```rust
#[derive(Subcommand)]
pub enum DockerCommand {
    /// Docker イメージをビルド
    Build {
        #[arg(short, long, help = "イメージタグ")]
        tag: Option<String>,
        #[arg(long, help = "ビルドキャッシュを使用しない")]
        no_cache: bool,
        #[arg(long, help = "HTTP プロキシ")]
        http_proxy: Option<String>,
        #[arg(long, help = "HTTPS プロキシ")]
        https_proxy: Option<String>,
    },
    /// Docker Compose 操作
    Compose {
        #[command(subcommand)]
        action: ComposeAction,
    },
    /// コンテナ状態表示
    Status,
}

#[derive(Subcommand)]
pub enum ComposeAction {
    /// サービスを起動
    Up {
        #[arg(short, long, help = "バックグラウンドで起動")]
        detach: bool,
        #[arg(long, help = "イメージを再ビルド")]
        build: bool,
    },
    /// サービスを停止・削除
    Down {
        #[arg(short, long, help = "ボリュームも削除")]
        volumes: bool,
    },
    /// ログ表示
    Logs {
        #[arg(short, long, help = "フォロー")]
        follow: bool,
        #[arg(help = "サービス名")]
        service: Option<String>,
    },
}
```

### 3.3 タグ命名規則

```
{feature_name}:{version}
```

- `version`: manifest.json の `template_version` から取得
- `--tag` 指定時はそちらを優先

### 3.4 doctor への Docker チェック追加

```rust
// doctor/requirements.rs に追加
ToolCheck::new("docker", "24.0", true),
ToolCheck::new("docker compose", "2.0", false),  // docker compose v2
```

## 機能 4: CI/CD Docker ワークフロー追加

### 4.1 docker.yml ワークフロー

```yaml
# .github/workflows/docker.yml
name: Docker Build Test
on:
  push:
    branches: [main, develop]
    paths:
      - 'CLI/templates/**/Dockerfile.tera'
      - 'CLI/templates/**/docker-compose.yml.tera'
      - 'feature/**/Dockerfile'
      - 'feature/**/docker-compose.yml'
  pull_request:
    paths:
      - 'CLI/templates/**/Dockerfile.tera'
      - 'CLI/templates/**/docker-compose.yml.tera'

jobs:
  build-test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        template: [backend-rust, backend-go, backend-csharp, backend-python, frontend-react]
    steps:
      - uses: actions/checkout@v4
      - name: Build Docker image
        run: docker build -t test-${{ matrix.template }} -f feature/backend/...
      - name: Verify container starts
        run: docker run --rm -d --name test test-${{ matrix.template }} && sleep 5 && docker stop test
```

## エラーハンドリング

### Docker 関連エラー

| エラー | 対応 |
|--------|------|
| Docker 未インストール | `k1s0 doctor` で検知、インストール手順を表示 |
| Dockerfile 不在 | `k1s0 docker build` 実行時にリカバリ提案（`k1s0 upgrade` でテンプレート更新） |
| ポート競合 | エラーメッセージに使用中ポートと解決方法を表示 |
| compose ファイル不在 | `k1s0 docker compose` 実行時にリカバリ提案 |
| ビルド失敗 | Docker ビルドログをそのまま表示 |
