# k1s0 v0.2.3 作業チェックリスト

## 概要

Docker と docker-compose をファーストクラスでサポートする。

- **全テンプレートに Dockerfile 追加**: backend-rust、backend-go、frontend-react に新規追加。C#・Python は HEALTHCHECK 追加
- **docker-compose.yml テンプレート追加**: 全バックエンド + frontend-react にローカル開発用 compose を追加
- **CLI docker コマンド追加**: `k1s0 docker build` / `k1s0 docker compose` ラッパー
- **CI/CD Docker ワークフロー追加**: Docker ビルドテストの自動化

## 受け入れ基準

### 機能要件（Dockerfile）

- [ ] `k1s0 new-feature --type backend-rust` で Dockerfile + .dockerignore が生成される
- [ ] `k1s0 new-feature --type backend-go` で Dockerfile + .dockerignore が生成される
- [ ] `k1s0 new-feature --type frontend-react` で Dockerfile + .dockerignore + nginx.conf が生成される
- [ ] 全 Dockerfile がマルチステージビルド構成である
- [ ] 全コンテナが非 root ユーザー（appuser）で実行される
- [ ] 全 Dockerfile に HEALTHCHECK 命令が含まれる
- [ ] `{% if with_grpc %}` で gRPC ポート（50051）が条件分岐される

### 機能要件（docker-compose）

- [ ] 全バックエンドテンプレートで docker-compose.yml が生成される
- [ ] `--with-db` 指定時に PostgreSQL サービスが含まれる
- [ ] `--with-cache` 指定時に Redis サービスが含まれる
- [ ] 全サービスにヘルスチェックが設定されている
- [ ] `depends_on: condition: service_healthy` で起動順序が保証される
- [ ] `--with-db` 指定時に secrets/db_password.txt が生成される
- [ ] frontend-react で docker-compose.yml が生成される（app のみ）

### 機能要件（CLI docker コマンド）

- [ ] `k1s0 docker build` でイメージがビルドされる
- [ ] `k1s0 docker compose up` でサービスが起動する
- [ ] `k1s0 docker compose down` でサービスが停止する
- [ ] `k1s0 docker status` でコンテナ状態が表示される
- [ ] Docker 未インストール時にエラーメッセージとリカバリ提案が表示される

### 機能要件（CI/CD）

- [ ] docker.yml ワークフローが全テンプレートのビルドをテストする

### 非機能要件

- [ ] `cargo check` パス
- [ ] `cargo clippy --all-targets --all-features -- -D warnings` パス
- [ ] `cargo test --all` 全テストパス
- [ ] 既存テンプレートの後方互換性が維持される（Docker ファイルは追加のみ）
- [ ] 環境変数ルール（K020）に違反しない（docker-compose の environment はDB初期化のみ）
