### 10.3 Dockerfile ビルドテスト（Docker 環境必須）

> **v0.2.3 スコープ外 — 前提条件未充足**
>
> 生成されたテンプレートの Cargo.toml は `path = "../../../../framework/..."` で
> モノレポ内の framework crate を参照している。Docker ビルドコンテキストは feature
> ディレクトリ単体のため、以下のいずれかが必要:
>
> 1. Docker ビルドコンテキストをモノレポルートに変更し Dockerfile のパスを調整
> 2. framework crate をレジストリ（crates.io / npm / NuGet / PyPI）に公開
> 3. `cargo vendor` 等で依存をベンダリング
>
> これは v0.3.0 以降で framework 公開と合わせて対応する。
> テンプレートの Dockerfile 構文・構造の正しさは Phase 10.1 の Tera レンダリングテストで検証済み。

## 受け入れ基準

### 機能要件（Dockerfile）

- [x] `k1s0 new-feature --type backend-rust` で Dockerfile + .dockerignore が生成される
- [x] `k1s0 new-feature --type backend-go` で Dockerfile + .dockerignore が生成される
- [x] `k1s0 new-feature --type frontend-react` で Dockerfile + .dockerignore + nginx.conf が生成される
- [x] 全 Dockerfile がマルチステージビルド構成である
- [x] 全コンテナが非 root ユーザー（appuser）で実行される
- [x] 全 Dockerfile に HEALTHCHECK 命令が含まれる
- [x] `{% if with_grpc %}` で gRPC ポート（50051）が条件分岐される

### 機能要件（docker-compose）

- [x] 全バックエンドテンプレートで docker-compose.yml が生成される
- [x] `--with-db` 指定時に PostgreSQL サービスが含まれる
- [x] `--with-cache` 指定時に Redis サービスが含まれる
- [x] 全サービスにヘルスチェックが設定されている
- [x] `depends_on: condition: service_healthy` で起動順序が保証される
- [x] `--with-db` 指定時に secrets/db_password.txt が生成される
- [x] frontend-react で docker-compose.yml が生成される（app のみ）

### 機能要件（CLI docker コマンド）

- [x] `k1s0 docker build` でイメージがビルドされる
- [x] `k1s0 docker compose up` でサービスが起動する
- [x] `k1s0 docker compose down` でサービスが停止する
- [x] `k1s0 docker status` でコンテナ状態が表示される
- [x] Docker 未インストール時にエラーメッセージとリカバリ提案が表示される

### 機能要件（CI/CD）

- [x] docker.yml ワークフローが全テンプレートのビルドをテストする

### 非機能要件

- [x] `cargo check` パス
- [x] `cargo clippy --all-targets --all-features -- -D warnings` パス
- [x] `cargo test --all` 全テストパス（511 テスト通過）
- [x] 既存テンプレートの後方互換性が維持される（Docker ファイルは追加のみ）
- [x] 環境変数ルール（K020）に違反しない（docker-compose の environment はDB初期化のみ）
