# k1s0 v0.2.2 改修方針

## 概要

CLI のユーザー体験（UX）を5つの機能で向上させる。

- **実行前プレビュー＆確認画面**: 変更を伴うコマンドで実行前にドライラン結果を表示
- **リアルタイム進捗フィードバック**: 長時間操作に progress_bar / spinner を追加
- **doctor 自動実行統合**: `init`/`new-feature` 時に必要ツールを自動チェック
- **ヘルプテキスト拡充**: 全コマンドに使用例と影響範囲を追加
- **エラー時自動リカバリ提案**: コピー可能な解決コマンドを提示

## 機能目的

- ユーザーが「何が起きるか」を事前に把握でき、安心して操作できるようにする
- 長時間操作中の「動いているか分からない」問題を解消する
- 初回セットアップの摩擦を軽減し、エラー時の自己解決率を向上させる

## 技術選定

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| 確認プロンプト | inquire（既存） | 既存の対話モードで使用中 |
| 進捗表示 | indicatif（既存） | output.rs に progress_bar/spinner API 実装済み |
| カラー出力 | console（既存） | 既存の output.rs スタイリング基盤 |
| ヘルプテキスト | clap 4.5 after_help | 既存 CLI フレームワーク標準機能 |
| TTY 検出 | atty / std::io::IsTerminal | 既存の is_interactive() 判定 |

## 機能 1: 実行前プレビュー＆確認画面

### 1.1 設計方針

- テンプレート生成エンジンにドライランモードを追加し、実際のファイル書き込みなしに生成予定ファイル一覧を取得
- `upgrade` コマンドには既存の `--check` フラグがあるため、その結果を確認画面に転用
- TTY環境でのみ確認プロンプトを表示。`--yes`/`-y` フラグで確認スキップ可能

### 1.2 対象コマンドと表示内容

| コマンド | プレビュー内容 |
|---------|--------------|
| `new-feature` | 生成ファイル/ディレクトリ一覧（ファイル数・ディレクトリ数のサマリ付き） |
| `new-domain` | 生成ファイル/ディレクトリ一覧 |
| `new-screen` | 生成ファイル一覧 |
| `upgrade` | 変更/追加/削除されるファイル一覧 |

### 1.3 CLIオプション追加

```rust
// 全生成コマンド共通

// upgrade コマンド（既存の --check を活用）
// --check: プレビューのみで実行しない（既存）
// --yes: プレビュー表示後に自動続行（新規）
```

### 1.4 ジェネレータ拡張

## 機能 2: リアルタイム進捗フィードバック

### 2.1 設計方針

- 既存の `output.rs` の `progress_bar()` / `spinner()` API を活用
- ジェネレータに進捗コールバックを追加し、ファイル単位で進捗を報告
- lint エンジンにもファイルスキャン進捗を追加

### 2.2 コールバックインターフェース

```rust
// k1s0-generator に追加
pub trait ProgressCallback: Send + Sync {
    fn on_file_start(&self, path: &Path);
    fn on_file_complete(&self, path: &Path);
    fn on_total_known(&self, total: usize);
}

// CLI 側の実装
pub struct CliProgressCallback {
    bar: ProgressBar,
}

impl ProgressCallback for CliProgressCallback {
    fn on_total_known(&self, total: usize) {
        self.bar.set_length(total as u64);
    }
    fn on_file_complete(&self, path: &Path) {
        self.bar.set_message(path.display().to_string());
        self.bar.inc(1);
    }
    // ...
}
```

### 2.3 対象操作

| 操作 | 表示形式 | 実装箇所 |
|------|---------|---------|
| テンプレート生成 | プログレスバー（ファイル数） | generator/template_engine.rs |
| lint 実行 | スピナー + チェック済みファイル数 | generator/lint/linter.rs |
| upgrade 差分計算 | スピナー | generator/upgrade.rs |
| upgrade ファイル適用 | プログレスバー | generator/upgrade.rs |

### 2.4 抑制ロジック

```rust
// Output に条件分岐を追加
impl Output {
    fn should_show_progress(&self) -> bool {
        !self.json_mode && !self.no_color && self.is_tty
    }
}
```

## 機能 3: doctor 自動実行統合

### 3.1 設計方針

- 既存の `doctor` コマンドのチェックロジックを再利用可能なモジュールとして分離
- `new-feature` / `init` コマンドから呼び出し、必要なツールのみをチェック
- チェック結果に応じて警告表示＋続行確認

### 3.2 doctor モジュール分離

```rust
// 既存: CLI/crates/k1s0-cli/src/commands/doctor.rs （コマンドハンドラ + チェックロジック）
// 変更後:
//   CLI/crates/k1s0-cli/src/doctor/mod.rs        （チェックロジック）
//   CLI/crates/k1s0-cli/src/doctor/checks.rs     （個別ツールチェック）
//   CLI/crates/k1s0-cli/src/doctor/report.rs     （結果表示）
//   CLI/crates/k1s0-cli/src/commands/doctor.rs   （コマンドハンドラ、上記を呼び出し）
```

### 3.3 テンプレートタイプ → 必要ツールマッピング


## 機能 4: ヘルプテキスト拡充

### 4.1 設計方針

- clap の `after_help` / `after_long_help` 属性を使用
- 短いヘルプ（`-h`）には Examples のみ
- 長いヘルプ（`--help`）には Examples + 生成されるもの/影響範囲

### 4.2 対象コマンドと追加内容

| コマンド | Examples | 影響範囲 |
|---------|----------|---------|
| `new-feature` | 3例（基本、オプション付き、対話モード） | 生成されるディレクトリ構造概要 |
| `new-domain` | 3例（基本、イベント付き、対話モード） | 生成されるパッケージ構造概要 |
| `new-screen` | 2例（基本、対話モード） | 生成されるファイル概要 |
| `lint` | 3例（基本、ルール指定、自動修復） | チェック対象・出力形式 |
| `upgrade` | 3例（チェック、実行、強制） | 変更対象・衝突処理 |
| `doctor` | 2例（基本、JSON出力） | チェック対象ツール |
| `init` | 2例（基本、対話モード） | 生成されるディレクトリ |
| `domain list` | 1例 | 出力形式 |
| `domain version` | 2例（表示、更新） | バージョン管理動作 |
| `domain dependents` | 1例 | 出力形式 |
| `domain impact` | 1例 | 影響分析内容 |
| `domain-catalog` | 1例 | 出力形式 |
| `domain-graph` | 2例（Mermaid、DOT） | 出力形式 |

## 機能 5: エラー時自動リカバリ提案

### 5.1 設計方針

- 既存の `CliError` の `hint` フィールドを活用（`String` → `Vec<String>` またはマルチライン対応）
- エラー種別ごとにリカバリコマンドを定義
- コマンドはコピー＆ペースト可能な形式で出力

### 5.2 CliError hint 拡張

```rust
// 現在
pub struct CliError {
    pub kind: CliErrorKind,
    pub message: String,
    pub target: Option<String>,
    pub hint: Option<String>,   // 単一行
    pub source: Option<Box<dyn std::error::Error>>,
}

// 変更後
pub struct CliError {
    pub kind: CliErrorKind,
    pub message: String,
    pub target: Option<String>,
    pub hint: Option<String>,              // 説明テキスト
    pub recovery_commands: Vec<Recovery>,   // リカバリコマンド一覧（新規）
    pub source: Option<Box<dyn std::error::Error>>,
}

pub struct Recovery {
    pub command: String,      // "k1s0 upgrade --force"
    pub description: String,  // "テンプレートの内容で上書き"
}
```

### 5.3 Output 表示更新

```rust
impl Output {
    pub fn error(&self, err: &CliError) {
        // 既存のエラー表示
        eprintln!("✗ {}: {}", err.kind, err.message);
        if let Some(target) = &err.target {
            eprintln!("  対象: {}", target);
        }
        if let Some(hint) = &err.hint {
            eprintln!("  ヒント: {}", hint);
        }
        // 新規: リカバリコマンド表示
        if !err.recovery_commands.is_empty() {
            eprintln!("  解決方法:");
            for (i, recovery) in err.recovery_commands.iter().enumerate() {
                eprintln!("    {}. {}    # {}", i + 1, recovery.command, recovery.description);
            }
        }
    }
}
```

### 5.4 エラー種別ごとのリカバリ定義

| エラー状況 | リカバリコマンド |
|-----------|---------------|
| ファイル衝突（upgrade） | `k1s0 upgrade --force` / `k1s0 upgrade --skip {file}` |
| manifest.json 不正 | `k1s0 init` |
| 必須ファイル不足（K010/K011） | `k1s0 lint --fix` |
| 出力先既存（new-feature） | `k1s0 new-feature ... --force` / 別名で再実行 |
| ツール不足（doctor） | 各ツールのインストールコマンド |
| 設定ファイル未検出 | `config/default.yaml を作成してください` |
| ドメイン未検出（K041） | `k1s0 new-domain --type {type} --name {name}` |
| バージョン不一致（K042） | `k1s0 domain version --name {name} --set {version}` |

## エラーハンドリング

### 確認プロンプト関連

| エラー | 対応 |
|--------|------|
| 非TTYで確認が必要 | `--yes` フラグの使用を提案 |
| ユーザーが "n" を選択 | exit code 0 で正常終了（中断はエラーではない） |

### doctor 自動チェック関連

| エラー | 対応 |
|--------|------|
| 必須ツール不足 | 警告表示 + 続行確認（強制停止しない） |
| チェック自体の失敗 | 警告としてスキップ（メイン処理は続行） |

## テスト方針

### ユニットテスト

| テスト対象 | テスト内容 |
|-----------|-----------|
| GeneratePreview | ドライランモードで正しいファイル一覧が返る |
| ProgressCallback | コールバックが正しいタイミングで呼ばれる |
| required_tools() | テンプレートタイプごとに正しいツールが返る |
| CliError recovery_commands | リカバリコマンドが正しく表示される |
| after_help テキスト | ヘルプ文字列が clap に正しく登録される |

### 統合テスト

| テスト対象 | テスト内容 |
|-----------|-----------|
| new-feature --yes | 確認スキップで正常生成 |
| new-feature（ドライラン） | プレビュー表示→中断→ファイル未生成 |
| lint（進捗表示） | TTY環境で進捗表示が出る |
| doctor 自動チェック | new-feature 時にツールチェックが実行される |
| --skip-doctor | 環境チェックがスキップされる |
| エラー + リカバリ | エラー時にリカバリコマンドが表示される |

### E2E テスト

| テスト対象 | テスト内容 |
|-----------|-----------|
| フルフロー | new-feature → 確認 → 生成 → lint → 全パス |
| JSON モード | --json で確認プロンプト・進捗なし、構造化出力 |
| 非TTY | パイプ環境で確認スキップ、進捗なし |
