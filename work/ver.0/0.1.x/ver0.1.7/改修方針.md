# k1s0 v0.1.7 改修方針

## 概要

ドメイン層の開発基盤（ドメインレジストリ）を構築する。

- **`k1s0 new-domain` コマンド強化**: 4言語対応のドメインスキャフォールドテンプレート
- **`k1s0 domain catalog` コマンド新設**: ドメイン一覧と依存関係の可視化
- **`k1s0 domain graph` コマンド新設**: 依存グラフの Mermaid / DOT 出力、循環依存検出

## 機能目的

- 空のドメイン層を実用的なものにし、3層アーキテクチャを実体化する
- ドメイン駆動設計のボイラープレートを自動生成し、導入障壁を下げる
- ドメイン間・ドメイン→feature 間の依存関係を可視化・検証する
- 既存 lint ルール（K040-K047）が即座に機能する状態を作る

## 技術選定

### グラフ解析

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| 依存グラフ構築 | petgraph 0.6 | Rust 標準的なグラフライブラリ、DFS/BFS/トポロジカルソート対応 |
| 循環依存検出 | petgraph::algo::is_cyclic_directed | 検証済みアルゴリズム |
| Mermaid 出力 | 自前フォーマッタ | 外部依存不要、テンプレート形式で十分 |
| DOT 出力 | 自前フォーマッタ | Graphviz 標準形式、シンプルなテキスト生成 |

### テンプレート生成

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| テンプレートレンダリング | Tera 1.19 | 既存テンプレートエンジンと統一 |
| CLI 引数 | clap 4.5 | 既存サブコマンド体系との整合 |
| マニフェスト解析 | serde_json + walkdir | 既存 manifest パーサー活用 |
| 名前変換 | heck（既存依存） | kebab → snake → PascalCase 変換 |

## 機能 2: `k1s0 domain catalog`

### 2.1 設計方針

- プロジェクトルートから `domain/` 配下を走査
- 各ドメインの `.k1s0/manifest.json` を読み取り
- feature 層の manifest から `domain.dependencies` を逆引きして dependents を算出
- テーブル / JSON 形式で出力

### 2.2 実装構成

```
k1s0-generator/src/
├── domain/
│   ├── mod.rs              # ドメインモジュールルート
│   ├── scanner.rs          # ドメイン走査（manifest 収集）
│   ├── catalog.rs          # カタログ構築・表示
│   └── types.rs            # DomainInfo, DomainSummary 型
```

### 2.3 主要型

```rust
/// スキャン結果のドメイン情報
pub struct DomainInfo {
    pub name: String,
    pub service_type: String,      // "backend" / "frontend"
    pub language: String,          // "rust" / "go" / "react" / "flutter"
    pub version: String,
    pub path: PathBuf,
    pub deprecated: bool,
    pub min_framework_version: Option<String>,
    pub dependencies: Vec<DomainDependency>,
    pub dependents: Vec<String>,   // feature 名のリスト
    pub breaking_changes: Vec<BreakingChange>,
}

pub struct DomainDependency {
    pub name: String,
    pub version_constraint: String,  // ">=1.0.0" 等
}

pub struct BreakingChange {
    pub version: String,
    pub description: String,
}

pub struct DomainSummary {
    pub total: usize,
    pub active: usize,
    pub deprecated: usize,
}
```

## 機能 3: `k1s0 domain graph`

### 3.1 設計方針

- `petgraph::DiGraph` でドメイン間依存グラフを構築
- feature 層からの依存も含めた完全グラフを構築
- Mermaid / DOT テキスト出力（画像生成は行わない）
- `--detect-cycles` で `petgraph::algo::is_cyclic_directed` を使用

### 3.2 実装構成

```
k1s0-generator/src/
├── domain/
│   ├── graph.rs            # グラフ構築・循環検出
│   └── formatter.rs        # Mermaid / DOT フォーマッタ
```

### 3.3 グラフ構築

```rust
use petgraph::graph::DiGraph;
use petgraph::algo;

pub struct DomainGraph {
    graph: DiGraph<DomainNode, DependencyEdge>,
}

pub struct DomainNode {
    pub name: String,
    pub version: String,
    pub layer: Layer,           // Domain / Feature
    pub deprecated: bool,
}

pub struct DependencyEdge {
    pub version_constraint: Option<String>,
}

pub enum Layer {
    Domain,
    Feature,
}

impl DomainGraph {
    /// manifest 一覧からグラフを構築
    pub fn from_domains(
        domains: &[DomainInfo],
        features: &[FeatureInfo],
    ) -> Self;

    /// 循環依存を検出
    pub fn detect_cycles(&self) -> Vec<Vec<String>>;

    /// Mermaid 形式で出力
    pub fn to_mermaid(&self) -> String;

    /// DOT 形式で出力
    pub fn to_dot(&self) -> String;

    /// 特定ノードからの部分グラフ
    pub fn subgraph(&self, root: &str) -> Option<DomainGraph>;
}
```

### 3.4 CLI オプション

```rust
#[derive(Args)]
pub struct DomainGraphArgs {
    /// 出力形式
    #[arg(long, default_value = "mermaid")]
    pub format: GraphFormat,

    /// 起点ドメイン名（部分グラフ）
    #[arg(long)]
    pub root: Option<String>,

    /// 循環依存検出モード
    #[arg(long, default_value_t = false)]
    pub detect_cycles: bool,
}

#[derive(ValueEnum, Clone)]
pub enum GraphFormat {
    Mermaid,
    Dot,
}
```
