# 要望

CLIの操作時に引数で指定する項目を減らしたい。
viteのように入力欄や選択形式で操作できると嬉しい。

## 回答（実装方針）

### 概要

Viteのような対話式インターフェースを実装し、引数なしでコマンドを実行した場合にプロンプト形式で必要な情報を収集する。

### 採用クレート

**`inquire`** を採用する。

- 高機能で美しいUI
- Viteのような洗練された対話体験を実現可能
- カスタマイズ性が高い
- アクティブにメンテナンスされている

### 対象コマンドと対話フロー

#### 1. `k1s0 new-feature`

```
$ k1s0 new-feature

? テンプレートタイプを選択してください: (↑↓で移動、Enterで決定)
❯ backend-rust   - Rustバックエンドサービス
  backend-go     - Goバックエンドサービス
  frontend-react - Reactフロントエンド
  frontend-flutter - Flutterフロントエンド

? フィーチャー名を入力してください: (kebab-case)
❯ user-management

? 追加オプションを選択してください: (Spaceで選択、Enterで確定)
❯ [x] gRPCサポート (--with-grpc)
  [x] データベース接続 (--with-db)
  [ ] Redisキャッシュ (--with-cache)

✔ フィーチャー 'user-management' を作成しました
```

#### 2. `k1s0 new-domain`

```
$ k1s0 new-domain

? テンプレートタイプを選択してください:
❯ backend-rust
  backend-go
  frontend-react
  frontend-flutter

? ドメイン名を入力してください: (kebab-case)
❯ order

? 初期バージョンを指定してください: [0.1.0]

✔ ドメイン 'order' (v0.1.0) を作成しました
```

#### 3. `k1s0 new-screen`

```
$ k1s0 new-screen

? テンプレートタイプを選択してください:
❯ frontend-react
  frontend-flutter

? スクリーン名を入力してください:
❯ user-profile

? 対象フィーチャーを選択してください:
❯ user-management
  order-processing

✔ スクリーン 'user-profile' を 'user-management' に作成しました
```

### 実装方針

#### 1. 後方互換性の維持

```bash
# 従来の引数指定方式（CI/CDで使用）は維持
k1s0 new-feature --type backend-rust --name user-management --with-grpc

# 引数不足時に対話モードへ自動フォールバック
k1s0 new-feature
# → 対話モード起動
```

#### 2. 明示的な対話モード

```bash
# -i / --interactive フラグで強制的に対話モード
k1s0 new-feature -i
k1s0 new-feature --interactive
```

#### 3. 部分指定のサポート

```bash
# 一部引数を指定、不足分のみ対話で補完
k1s0 new-feature --type backend-rust
# → 名前とオプションのみ対話で入力
```

```toml
# CLI/crates/k1s0-cli/Cargo.toml

```

### 実装優先度

1. **Phase 1**: `new-feature` コマンドの対話化
2. **Phase 2**: `new-domain`, `new-screen` の対話化
3. **Phase 3**: `init` コマンドの対話化（プロジェクト初期設定）

### 注意事項

- **TTY検出**: パイプ入力やCI環境では対話モードを無効化
- **カラー対応**: `--no-color` オプションとの整合性
- **タイムアウト**: 長時間無応答時の処理
- **Ctrl+C**: 中断時のクリーンアップ
