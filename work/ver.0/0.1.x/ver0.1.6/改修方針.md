# k1s0 v0.1.6 改修方針

## 概要

バックエンドフレームワーク（Rust/Go）にドメインイベント基盤パッケージを追加する。

- **k1s0-domain-event（Rust crate）**: ドメインイベントの定義・発行・購読・Outbox
- **k1s0-domain-event（Go module）**: Go 版の同等実装

## 機能目的

- マイクロサービス間のイベント駆動連携の標準化
- Outbox パターンによるトランザクション保証付きイベント発行
- テスト用インメモリバスの提供
- フロントエンド `@k1s0/realtime` / `k1s0_realtime` との連携パス確立

## 技術選定

### Rust

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| イベント定義 | trait + serde | 型安全かつシリアライズ可能 |
| メタデータ | uuid v7 + chrono | 時系列ソート可能な ID |
| インメモリバス | tokio::broadcast | 非同期マルチ購読者対応 |
| Outbox | k1s0-db 連携（feature flag） | トランザクション同期 |
| 観測性 | k1s0-observability 連携 | trace_id 伝搬 |
| エラー処理 | k1s0-error 連携 | 統一エラー体系 |

### Go

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| イベント定義 | interface + Generics | Go 1.22 のジェネリクス活用 |
| メタデータ | google/uuid | 標準的な UUID 生成 |
| インメモリバス | channel + sync | Go 標準ライブラリ |
| Outbox | k1s0-db 連携 | トランザクション同期 |

## Tier 分類

**Tier 2** crate として配置する。

| 依存先 | 必須/任意 |
|--------|----------|
| k1s0-error（Tier 1） | 必須 |
| k1s0-observability（Tier 2） | 必須 |
| k1s0-db（Tier 2） | 任意（`outbox` feature flag） |

Tier 依存ルール（Tier 2 は Tier 1 のみ依存可能）に対して `k1s0-observability`（Tier 2）への依存がある。
これは trace_id 伝搬のために必要であり、Tier 2 同士の依存として許容する（同一 Tier 内依存は許可）。

## 機能 1: ドメインイベント定義

### 1.1 Rust 版

#### 設計方針

- **`DomainEvent` trait**: すべてのドメインイベントが実装する契約
- **`EventMetadata`**: イベント ID、タイムスタンプ、相関 ID を自動付与
- **`EventEnvelope`**: イベント + メタデータのペア

#### 型定義

```rust
/// ドメインイベント trait
pub trait DomainEvent: Serialize + Send + Sync + 'static {
    fn event_type(&self) -> &str;
    fn aggregate_id(&self) -> &str;
    fn aggregate_type(&self) -> &str;
    fn version(&self) -> u32 { 1 }
}

/// イベントメタデータ
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EventMetadata {
    pub event_id: String,
    pub timestamp: DateTime<Utc>,
    pub correlation_id: Option<String>,
    pub causation_id: Option<String>,
    pub source_service: String,
    pub trace_id: Option<String>,
}

impl EventMetadata {
    /// 現在のコンテキストからメタデータを自動生成
    pub fn from_context(source_service: &str) -> Self;

    /// 因果関係チェーンを構築
    pub fn caused_by(parent: &EventMetadata, source_service: &str) -> Self;
}

/// エンベロープ
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EventEnvelope<E: DomainEvent> {
    pub metadata: EventMetadata,
    pub payload: E,
}
```

#### 内部構成

```
src/
├── event.rs          # DomainEvent trait
├── envelope.rs       # EventEnvelope, EventMetadata
└── error.rs          # PublishError, SubscribeError, HandlerError, OutboxError
```

### 1.2 Go 版

#### 型定義

```go
type DomainEvent interface {
    EventType() string
    AggregateID() string
    AggregateType() string
    Version() uint32
}

type EventMetadata struct {
    EventID       string    `json:"event_id"`
    Timestamp     time.Time `json:"timestamp"`
    CorrelationID string    `json:"correlation_id,omitempty"`
    CausationID   string    `json:"causation_id,omitempty"`
    SourceService string    `json:"source_service"`
    TraceID       string    `json:"trace_id,omitempty"`
}

type EventEnvelope[E DomainEvent] struct {
    Metadata EventMetadata `json:"metadata"`
    Payload  E             `json:"payload"`
}
```

### 2.1 Rust 版

#### 設計方針

- **`EventPublisher` trait**: infrastructure 層で実装を差し替え可能

#### インターフェース

```rust
pub trait EventPublisher: Send + Sync {
    async fn publish<E: DomainEvent>(&self, event: E) -> Result<(), PublishError>;

}
```

```go
type EventPublisher interface {
```

## 機能 3: イベント購読（Subscriber）

### 3.1 Rust 版

#### 設計方針

- **`EventHandler` trait**: 購読時に型安全なハンドラを登録
- **`SubscriptionHandle`**: 購読解除に使用

#### インターフェース

```rust
#[async_trait]
pub trait EventHandler<E: DomainEvent>: Send + Sync {
    async fn handle(&self, envelope: EventEnvelope<E>) -> Result<(), HandlerError>;
}

#[async_trait]
pub trait EventSubscriber: Send + Sync {
    async fn subscribe<E, H>(
        &self,
        event_type: &str,
        handler: H,
    ) -> Result<SubscriptionHandle, SubscribeError>
    where
        E: DomainEvent + for<'de> Deserialize<'de>,
        H: EventHandler<E> + 'static;

    async fn unsubscribe(&self, handle: SubscriptionHandle) -> Result<(), SubscribeError>;
}
```

### 3.2 Go 版

```go
type EventHandler[E DomainEvent] interface {
    Handle(ctx context.Context, envelope EventEnvelope[E]) error
}

type EventSubscriber interface {
    Subscribe(ctx context.Context, eventType string, handler any) (SubscriptionHandle, error)
    Unsubscribe(ctx context.Context, handle SubscriptionHandle) error
}
```

## 機能 4: Outbox パターン

### 4.1 設計方針

- **`outbox` feature flag** で有効化（`k1s0-db` 依存）
- ビジネストランザクションと同一トランザクションで outbox テーブルに記録
- `OutboxRelay` がポーリングで未送信エントリをブローカーへ転送
- リトライ回数と dead letter 管理

### 4.2 Outbox テーブルスキーマ

```sql
CREATE TABLE outbox_events (
    id            VARCHAR(36) PRIMARY KEY,  -- UUID v7
    event_type    VARCHAR(255) NOT NULL,
    aggregate_id  VARCHAR(255) NOT NULL,
    aggregate_type VARCHAR(255) NOT NULL,
    payload       JSONB NOT NULL,
    metadata      JSONB NOT NULL,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    delivered_at  TIMESTAMPTZ,
    retry_count   INTEGER NOT NULL DEFAULT 0,
    max_retries   INTEGER NOT NULL DEFAULT 5,
    next_retry_at TIMESTAMPTZ,
    status        VARCHAR(20) NOT NULL DEFAULT 'pending'  -- pending, delivered, failed
);

CREATE INDEX idx_outbox_pending ON outbox_events (status, next_retry_at)
    WHERE status = 'pending';
CREATE INDEX idx_outbox_aggregate ON outbox_events (aggregate_type, aggregate_id);
```

### 4.3 Rust インターフェース

```rust
#[async_trait]
pub trait OutboxStore: Send + Sync {
    async fn store<E: DomainEvent>(
        &self,
        tx: &mut Transaction,
        event: E,
    ) -> Result<OutboxEntry, OutboxError>;

    async fn fetch_pending(&self, limit: usize) -> Result<Vec<OutboxEntry>, OutboxError>;

    async fn mark_delivered(&self, entry_id: &str) -> Result<(), OutboxError>;

    async fn mark_failed(&self, entry_id: &str, error: &str) -> Result<(), OutboxError>;
}

pub struct OutboxRelay {
    store: Arc<dyn OutboxStore>,
    publisher: Arc<dyn EventPublisher>,
    poll_interval: Duration,     // デフォルト: 1秒
    batch_size: usize,           // デフォルト: 100
    max_retries: u32,            // デフォルト: 5
}

impl OutboxRelay {
    pub fn builder() -> OutboxRelayBuilder;
    pub async fn start(&self, shutdown: watch::Receiver<bool>);
}
```

## エラー型

```rust
/// 発行エラー
pub enum PublishError {
    Serialization(String),
    Transport(String),
    Timeout,
}

/// 購読エラー
pub enum SubscribeError {
    AlreadySubscribed(String),
    Transport(String),
}

/// ハンドラエラー
pub enum HandlerError {
    Processing(String),
    Retryable(String),
    Fatal(String),
}

/// Outbox エラー
pub enum OutboxError {
    Store(String),
    Fetch(String),
    Delivery(String),
}
```

すべてのエラー型は `k1s0-error` の `K1s0Error` への `From` 実装を持つ。
