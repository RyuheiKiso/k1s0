# k1s0 v0.2.5 作業チェックリスト

## 概要

`k1s0 playground` コマンドの新設。Docker モード（docker compose）とローカルモード（直接プロセス起動）の2モード対応。start/stop/status/list サブコマンドの実装、playground 専用テンプレート、テストを含む。

## 現状分析

### 実装済み（関連コンポーネント）

- [x] `docker.rs` — Docker サブコマンドパターン、`check_docker()` / `check_docker_compose()`
- [x] `new_feature.rs` — テンプレートレンダリングフロー、Tera コンテキスト構築
- [x] `k1s0-generator::template::TemplateRenderer` — Tera レンダリングエンジン（`WriteResult::Overwritten` 対応済み）
- [x] `prompts/` — inquire ベースのインタラクティブモード
- [x] `CLI/templates/backend-rust/feature/` — ベーステンプレート（6種全て）

### v0.2.5 で実装

- [x] `playground.rs` — コマンド本体（Docker モード + ローカルモード）

## 受け入れ基準

### 機能要件（Docker モード）

- [x]`k1s0 playground start --type backend-rust --with-db` で Docker 環境が起動する
- [x]`curl http://localhost:8080/health` が 200 を返す
- [x]`curl http://localhost:8080/api/v1/items` がサンプルデータを返す
- [x]`k1s0 playground status` で mode: docker と表示される
- [x]`k1s0 playground stop` で Docker コンテナ停止 + ファイル削除される
- [x]`--port-offset 100` で REST:8180, DB:5532 で起動する

### 機能要件（ローカルモード）

- [x]`k1s0 playground start --type backend-rust --with-db --mode local` でローカルプロセスが起動する
- [x]`curl http://localhost:8080/health` が 200 を返す
- [x]`curl http://localhost:8080/api/v1/items` がサンプルデータを返す（SQLite）
- [x]`k1s0 playground status` で mode: local, PID が表示される
- [x]`k1s0 playground stop` でプロセス停止 + ファイル削除される
- [x]Docker 未インストール時に自動的にローカルモードにフォールバックする

### 機能要件（共通）

- [x]`k1s0 playground list` で全テンプレート + ローカルモード前提ツールが表示される
- [x]`k1s0 init` 未実行の空ディレクトリでも動作する
- [x]インタラクティブモードでテンプレート/モード/オプション選択ができる
- [x]`--port-offset` で複数 playground を並行起動できる

### テスト要件

- [x]`cargo test --all` が全テスト通過
- [x]ユニットテスト: detect_mode, resolve_ports, check_ports, metadata 等
- [x]テンプレートテスト: Docker/ローカル両モードでレンダリング検証
- [x]統合テスト（ローカル）: テンプレート生成 → ディレクトリ構造検証

### 非機能要件

- [x]サンプルコードが K020（環境変数禁止）ルールに違反しない
- [x]サンプルコードが Clean Architecture の依存方向を遵守
- [x]Docker 未インストール時に明確なメッセージが表示される
- [x]言語ツールチェーン未インストール時にインストール案内が表示される
- [x]Windows / macOS / Linux でプロセス管理が正しく動作する
- [x]ヘルスチェックタイムアウト時にログ確認方法が案内される
