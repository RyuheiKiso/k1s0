# k1s0 v0.2.6 作業チェックリスト

## 概要

`k1s0 migrate` コマンドの新設。既存プロジェクトの k1s0 準拠度を分析し、移行計画を生成し、段階的に k1s0 構造へ変換する。analyze/plan/apply/status の4サブコマンドを実装。分析エンジン（analyzer/）の新規追加、既存 lint エンジンの拡張適用を含む。

## 受け入れ基準

### 機能要件（analyze）

- [x] `k1s0 migrate analyze --path ./rust-project` で分析結果が表示される
- [x] プロジェクトタイプが正しく推定される
- [x] 総合スコアが 0-100 で算出される
- [x] ディレクトリ構造の過不足が検出される
- [x] K020, K021, K029, K050, K053, K060 違反が検出される
- [x] `--json` で JSON 形式出力が得られる
- [x] `--verbose` で具体的なファイル:行番号が表示される
- [x] `--type` で推定結果を上書きできる

### 機能要件（plan）

- [x] `k1s0 migrate plan --path ./rust-project --name order-service` で計画 JSON が生成される
- [x] 計画にバックアップ → 構造整理 → 管理ファイル生成 → 規約修正 → 検証の5フェーズが含まれる
- [x] `.env` の内容が `config/default.yaml` に変換される
- [x] 自動修正不可能な項目は `ManualAction` として含まれる
- [x] `--dry-run` でファイル書き出しなし

### 機能要件（apply）

- [x] `k1s0 migrate apply` で計画に基づいて移行が実行される
- [x] バックアップが `.k1s0/backup/{timestamp}/` に作成される
- [x] ディレクトリ構造が k1s0 準拠に変換される
- [x] `.k1s0/manifest.json` が生成される
- [x] `config/default.yaml` が生成される
- [x] `--dry-run` で非破壊的にプレビューできる
- [x] 2回目の `apply` で完了済みステップがスキップされる
- [x] 失敗時にロールバック案内が表示される

### 機能要件（status）

- [x] `k1s0 migrate status` で移行の進捗が表示される
- [x] 各フェーズの完了/残りステップ数が表示される
- [x] 残りの手動作業リストが表示される

### テスト要件

- [x] `cargo test --all` が全テスト通過（717件合格、0失敗）
- [x] 4言語のテストフィクスチャに対する分析結果が期待値と一致
- [x] `cargo clippy --all-targets -- -D warnings` 警告なし

### 非機能要件

- [x] analyze, plan は完全に非破壊的（ファイルシステムへの書き込みなし、plan は出力ファイルのみ）

- [x] Windows / macOS / Linux でパス処理が正しく動作（`std::path::Path` 使用）
