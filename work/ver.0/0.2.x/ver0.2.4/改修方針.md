# k1s0 v0.2.4 改修方針

## 概要

Python と C# のバックエンドフレームワークに4パッケージずつ（計8パッケージ）を新規実装する。

- **k1s0-domain-event / K1s0.DomainEvent**: ドメインイベント、Outbox パターン
- **k1s0-resilience / K1s0.Resilience**: サーキットブレーカー、タイムアウト、バルクヘッド
- **k1s0-cache / K1s0.Cache**: Redis キャッシュ、キャッシュパターン
- **k1s0-auth / K1s0.Auth**: JWT/OIDC 認証、ポリシーエンジン、ミドルウェア

## 機能目的

- Python/C# バックエンドサービスで Rust/Go と同等のフレームワーク機能を利用可能にする
- 全4言語の機能パリティを達成する

## 技術選定

### Python パッケージ

| パッケージ | 主要依存 | 理由 |
|-----------|---------|------|
| k1s0-domain-event | pydantic, uuid, asyncio | 既存パッケージと統一、型安全性 |
| k1s0-resilience | asyncio, time | 外部依存最小、asyncio ネイティブ |
| k1s0-cache | redis[hiredis], pydantic | redis-py 公式、hiredis で高速化 |
| k1s0-auth | PyJWT, cryptography, httpx | JWT 標準ライブラリ、OIDC 対応 |

### C# パッケージ

| パッケージ | 主要依存 | 理由 |
|-----------|---------|------|
| K1s0.DomainEvent | System.Text.Json | .NET 標準、外部依存なし |
| K1s0.Resilience | Microsoft.Extensions.DependencyInjection | DI 統合、Polly 非依存で軽量 |
| K1s0.Cache | StackExchange.Redis | .NET Redis 標準、高性能 |
| K1s0.Auth | Microsoft.AspNetCore.Authentication.JwtBearer, Microsoft.IdentityModel.Tokens | ASP.NET Core 標準認証基盤 |

## パッケージ 1: k1s0-domain-event / K1s0.DomainEvent

### 1.1 設計方針

- Rust 版の全機能を移植（DomainEvent, EventEnvelope, Publisher, Subscriber, Outbox）
- InMemory 実装をデフォルト提供
- Outbox パターンは optional（Python: asyncpg/SQLAlchemy、C#: EF Core）

### 1.2 公開 API

| 型 | Python | C# | 説明 |
|----|--------|-----|------|
| DomainEvent | `abc.ABC` | `interface IDomainEvent` | イベント基底（event_type, aggregate_id, aggregate_type） |
| EventEnvelope | `@dataclass` | `record EventEnvelope` | メタデータ付きイベントラッパー |
| EventMetadata | `@dataclass` | `record EventMetadata` | event_id, occurred_at, source, correlation_id, causation_id |
| EventPublisher | `abc.ABC` (async) | `interface IEventPublisher` | publish, publish_batch |
| EventHandler | `abc.ABC` (async) | `interface IEventHandler` | event_type, handle |
| EventSubscriber | `abc.ABC` (async) | `interface IEventSubscriber` | subscribe → cancellation |
| InMemoryEventBus | class | class | Publisher + Subscriber のインメモリ実装 |
| OutboxEntry | `@dataclass` | `record OutboxEntry` | id, event_type, payload, status, retry_count |
| OutboxStore | `abc.ABC` (async) | `interface IOutboxStore` | insert, fetch_pending, mark_published, mark_failed |
| OutboxRelay | class | class | Outbox → Publisher リレー（ポーリング + リトライ） |

### 1.3 Tier 依存

- Tier 2
- 必須依存: なし（Python: pydantic のみ、C#: System.Text.Json のみ）
- Optional: k1s0-db（Outbox の永続化）

## パッケージ 2: k1s0-resilience / K1s0.Resilience

### 2.1 設計方針

- Rust 版の全機能を移植（Timeout, ConcurrencyLimiter, Bulkhead, CircuitBreaker）
- Go 版の Retry（指数バックオフ + ジッター）も追加
- 外部依存最小（Python: asyncio 標準ライブラリ、C#: .NET 標準）

### 2.2 公開 API

| 型 | Python | C# | 説明 |
|----|--------|-----|------|
| TimeoutGuard | class | class | min/max 検証付きタイムアウト（100ms-300s） |
| TimeoutConfig | `@dataclass` | `record TimeoutConfig` | タイムアウト設定 |
| ConcurrencyLimiter | class (asyncio.Semaphore) | class (SemaphoreSlim) | 同時実行数制限 |
| ConcurrencyConfig | `@dataclass` | `record ConcurrencyConfig` | 同時実行数設定 |
| Bulkhead | class | class | サービス別リソース分離 |
| BulkheadConfig | `@dataclass` | `record BulkheadConfig` | バルクヘッド設定 |
| CircuitBreaker | class | class | 状態機械（Closed/Open/HalfOpen） |
| CircuitBreakerConfig | `@dataclass` | `record CircuitBreakerConfig` | failure_threshold, success_threshold, reset_timeout |
| CircuitState | Enum | enum | Closed, Open, HalfOpen |
| RetryExecutor | class | class | 指数バックオフ + ジッター付きリトライ |
| RetryConfig | `@dataclass` | `record RetryConfig` | max_attempts, initial_interval, max_interval, multiplier, jitter_factor |
| ResilienceError | Exception class | Exception class | Timeout, CircuitOpen, ConcurrencyLimit |

### 2.3 Tier 依存

- Tier 2
- 必須依存: なし
- Optional: なし

| SetOperations | `abc.ABC` (async) | `interface ISetOperations` | sadd, srem, smembers, sismember |

| WriteThrough | class | class | DB優先 → キャッシュ同期 |
| WriteBehind | class | class | 非同期バッチ書き込み |
| CacheMetrics | class | class | ヒット率、操作カウント |
| CacheHealthChecker | class | class | k1s0-health 統合 |

## パッケージ 4: k1s0-auth / K1s0.Auth

### 4.1 設計方針

- Rust 版のフル機能を移植（Go 版より広いスコープ）
- モジュール: JWT, OIDC, Policy, Middleware, Audit, Blacklist, Refresh
- フレームワーク統合: Python → FastAPI middleware、C# → ASP.NET Core middleware

### 4.2 公開 API

| 型 | Python | C# | 説明 |
|----|--------|-----|------|
| JwtVerifier | class | class | JWKS ローテーション対応 JWT 検証 |
| JwtVerifierConfig | `@dataclass` | `record JwtVerifierConfig` | issuer, jwks_uri, audience |
| Claims | `@dataclass` | `record Claims` | sub, roles, permissions, tenant_id 等 |
| OidcDiscovery | class | class | OIDC ディスカバリエンドポイント取得 |
| OidcJwtVerifier | class | class | OIDC + JWT 統合検証 |
| UserInfo / UserInfoClient | class | class | OIDC UserInfo エンドポイント |
| PolicyEvaluator | `abc.ABC` | `interface IPolicyEvaluator` | RBAC ポリシー評価 |
| PolicyBuilder | class | class | ポリシールール構築（admin, read, custom） |
| PolicyRepository | `abc.ABC` (async) | `interface IPolicyRepository` | ポリシー永続化 |
| InMemoryPolicyRepository | class | class | インメモリ実装 |
| CachedPolicyRepository | class | class | k1s0-cache 統合キャッシュ |
| AuditLogger | class | class | 認証/認可イベントの監査ログ |
| TokenBlacklist | `abc.ABC` (async) | `interface ITokenBlacklist` | トークン無効化 |
| InMemoryBlacklist | class | class | インメモリ実装 |
| RefreshTokenManager | class | class | リフレッシュトークンライフサイクル |
| RefreshTokenStore | `abc.ABC` (async) | `interface IRefreshTokenStore` | リフレッシュトークン永続化 |
| AuthMiddleware | FastAPI Middleware | ASP.NET Core Middleware | HTTP 認証ミドルウェア |
| GrpcAuthInterceptor | grpcio Interceptor | gRPC Interceptor | gRPC 認証インターセプター |

### 4.3 Tier 依存

- Tier 3
- 必須依存: なし
- Optional: k1s0-cache（CachedPolicyRepository）、k1s0-db（永続化）

## テスト方針

### ユニットテスト

各パッケージに以下のテストを実装:

| パッケージ | Python テスト | C# テスト |
|-----------|-------------|-----------|
| domain-event | EventEnvelope 生成、InMemoryEventBus の publish/subscribe、OutboxRelay のリトライ | 同左 |
| resilience | TimeoutGuard の min/max 検証、CircuitBreaker の状態遷移、RetryExecutor の指数バックオフ、Bulkhead の分離 | 同左 |
| cache | CacheClient の get/set/delete、CacheAside の get_or_load、WriteThrough/WriteBehind の動作、CacheMetrics のカウント | 同左 |
| auth | JWT 検証（正常/期限切れ/無効署名）、Claims パース、PolicyEvaluator の allow/deny、TokenBlacklist、RefreshTokenManager | 同左 |

### テスト基盤

- Python: pytest + pytest-asyncio + pytest-mock
- C#: xUnit + Moq + FluentAssertions

### テストカバレッジ目標

- 各パッケージ 80% 以上の行カバレッジ
- 全パブリック API にテストあり

## 注意事項

- **環境変数禁止**: 全設定は config/*.yaml 経由。os.environ / Environment.GetEnvironmentVariable を使用しない
- **Tier 依存ルール厳守**: Tier 2 は Tier 1 のみ依存可、Tier 3 は Tier 1/2 に依存可
- **既存パッケージとの一貫性**: 既存 Python/C# パッケージの命名規則・コード構造・テストパターンに従う
- **Rust 版とのAPI互換**: 概念レベルで同等の API を提供（言語固有のイディオムは許容）
