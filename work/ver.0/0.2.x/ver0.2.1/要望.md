# 要望

バックエンドの対応言語に Python（FastAPI）を追加したい。
現在 Rust、Go、C# の3言語でバックエンドサービスを生成・管理できるが、データサイエンス・ML 領域やスタートアップで広く使われている Python もサポートすることで、k1s0 プラットフォームの適用範囲をさらに拡大したい。

## 背景

- データサイエンス・ML チームが Python を主要言語として使用しており、API サーバーも Python で統一したいニーズがある
- FastAPI は高パフォーマンスかつ型安全な Python Web フレームワークとして急速に普及している
- Pydantic v2 による強力なバリデーション、非同期対応、OpenAPI 自動生成など、k1s0 の設計方針と親和性が高い
- k1s0 の Clean Architecture 強制・lint・テンプレート生成の恩恵を Python にも適用したい
- gRPC / REST / OpenAPI の API コントラクト管理は言語非依存のため、Python にも同様に適用可能

## 回答（実装方針）

### 概要

1. **`backend-python` テンプレート**: Python / FastAPI ベースのバックエンドスキャフォールド（feature + domain）
2. **CLI 拡張**: `ServiceType` / `DomainType` enum に `BackendPython` を追加
3. **フレームワークパッケージ**: Python 版の k1s0 共通ライブラリ（pip パッケージ）
4. **lint ルール対応**: K010/K011 必須ファイル、K020 環境変数禁止、K022 Clean Architecture 依存方向を Python に対応
5. **CI/CD ワークフロー**: Python 用の GitHub Actions ワークフロー追加

### 技術選定

| 機能 | 技術 | 理由 |
|------|------|------|
| Web フレームワーク | FastAPI 0.115+ | 高パフォーマンス、型安全、OpenAPI 自動生成 |
| ASGI サーバー | Uvicorn | FastAPI 推奨、非同期対応 |
| バリデーション | Pydantic v2 | FastAPI 標準、型ベースバリデーション |
| ORM | SQLAlchemy 2.0 + asyncpg | 非同期対応、成熟した ORM |
| gRPC | grpcio + grpcio-tools | Python 公式 gRPC 実装 |
| テスト | pytest + pytest-asyncio + httpx | Python 標準テストスタック |
| ログ/トレース | OpenTelemetry Python SDK | k1s0-observability 互換 |
| パッケージ管理 | uv (pyproject.toml) | 高速パッケージマネージャ、標準準拠 |
| フォーマット | Ruff | 高速フォーマッタ・リンター |
| 型チェック | mypy | Python 標準の静的型チェッカー |
| Python バージョン | 3.12+ | パフォーマンス改善、型ヒント強化 |

### 設計方針

- **既存パターン踏襲**: Rust / Go / C# テンプレートと同じ Clean Architecture ディレクトリ構造
- **k1s0 規約準拠**: 環境変数禁止（K020）、シークレットハードコード禁止（K021）、CA 依存方向（K022）
- **manifest.json 互換**: 既存のスキーマを拡張し `language: "python"` を追加
- **段階的導入**: v0.2.1 ではテンプレート + lint 対応まで、フレームワークパッケージは後続バージョン

## 機能 4: lint ルール Python 対応

### K010/K011: 必須ディレクトリ・ファイル

**必須ディレクトリ（feature）:**
- `src/`
- `src/{feature_name_snake}/`
- `src/{feature_name_snake}/domain/`
- `src/{feature_name_snake}/application/`
- `src/{feature_name_snake}/infrastructure/`
- `src/{feature_name_snake}/presentation/`
- `config/`
- `deploy/`

**必須ファイル（feature）:**
- `pyproject.toml`
- `src/{feature_name_snake}/__init__.py`
- `config/default.yaml`
- `.k1s0/manifest.json`

**必須ディレクトリ（domain）:**
- `src/{domain_name_snake}_domain/`
- `src/{domain_name_snake}_domain/entities/`
- `src/{domain_name_snake}_domain/value_objects/`
- `src/{domain_name_snake}_domain/repositories/`

**必須ファイル（domain）:**
- `pyproject.toml`
- `src/{domain_name_snake}_domain/__init__.py`
- `.k1s0/manifest.json`

### K020: 環境変数禁止

**Python 禁止パターン:**
- `os.environ`
- `os.getenv`
- `os.environ.get`
- `environ[`
- `dotenv` / `load_dotenv`

**正しい方法:** `config/*.yaml` + `k1s0-config` パッケージ

### K021: シークレットハードコード禁止

Python でも同様に YAML 内の `password`, `secret`, `api_key`, `token`, `credential`, `private_key` の直接値を禁止。

### K022: Clean Architecture 依存方向

Python の `import` 文を解析し、以下を禁止:
- `domain → application`
- `domain → presentation`
- `domain → infrastructure`
- `application → presentation`

## 機能 5: CI/CD ワークフロー

### python.yml

```yaml
name: Python Backend
on:
  push:
    branches: [main, develop]
    paths:
      - 'framework/backend/python/**'
      - 'feature/backend/python/**'
      - 'domain/backend/python/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v4
        with:
          version: "latest"
      - uses: actions/setup-python@v5
        with:
          python-version: '3.12'
      - run: uv sync
      - run: uv run ruff check .
      - run: uv run ruff format --check .
      - run: uv run mypy .
      - run: uv run pytest --tb=short -q
```

## 期待される効果

1. **対応言語拡大**: Rust / Go / C# に加え Python をサポートし、データサイエンス・ML チームも k1s0 プラットフォームを利用可能
2. **スタートアップ対応**: Python / FastAPI は迅速なプロトタイピングに適しており、小規模チームでの採用を促進
3. **統一ガバナンス**: 言語が異なっても同じ lint ルール・テンプレート・コントラクト管理が適用される
4. **Clean Architecture の強制**: Python の import 文を使った依存方向の検証が可能
5. **既存資産の活用**: gRPC / OpenAPI コントラクト、Kubernetes デプロイメント、設定管理は言語非依存で再利用
6. **モダンツールチェーン**: uv + Ruff + mypy による高速で型安全な開発体験

---

## 注意事項

- **Python バージョン**: 3.12+ を対象とする（パフォーマンス改善、型ヒント強化）
- **環境変数禁止**: `os.environ`, `os.getenv` は K020 違反。`config/*.yaml` を使用すること
- **プロジェクト構造**: Python は src レイアウト（`src/{package_name}/`）を標準とし、各 CA 層をサブパッケージとする
- **パッケージ管理**: uv + `pyproject.toml` を使用し、`requirements.txt` は使用しない
- **型ヒント**: 全コードで型ヒントを必須とし、mypy strict モードで検証
- **コードフォーマット**: Ruff を CI で強制
- **命名規約**: Python 標準の snake_case を使用（kebab-case のサービス名 → snake_case のパッケージ名に変換）
