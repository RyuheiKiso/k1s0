# k1s0 v0.2.1 改修方針

## 概要

バックエンドの対応言語に Python（FastAPI）を追加する。

- **`backend-python` テンプレート新設**: feature / domain 両方のスキャフォールド生成
- **CLI 拡張**: `ServiceType` / `DomainType` enum に `BackendPython` 追加
- **フレームワークパッケージ**: Python 版 k1s0 共通ライブラリ（pip）8パッケージ
- **lint ルール Python 対応**: K010/K011/K020/K021/K022 の Python パターン追加
- **CI/CD**: `python.yml` ワークフロー追加
- **manifest.schema.json 更新**: `language` enum に `"python"` 追加

## 機能目的

- Rust / Go / C# に並ぶ第4のバックエンド言語として Python をサポートし、データサイエンス・ML 開発者が k1s0 プラットフォームを利用可能にする
- 同一の Clean Architecture 規約・lint・テンプレート生成の仕組みを Python にも適用する
- スタートアップ・ML チームでの k1s0 採用を促進する

## 技術選定

### Python バックエンド

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| Web フレームワーク | FastAPI 0.115+ | 高パフォーマンス、型安全、OpenAPI 自動生成 |
| ASGI サーバー | Uvicorn | FastAPI 推奨、非同期対応 |
| バリデーション | Pydantic v2 | FastAPI 標準、型ベースバリデーション |
| ORM | SQLAlchemy 2.0 + asyncpg | 非同期対応、成熟した ORM |
| gRPC | grpcio + grpcio-tools | Python 公式 gRPC 実装 |
| DI | dependency-injector | Python 標準的な DI コンテナ |
| 設定管理 | PyYAML | k1s0 YAML 設定規約互換 |
| レジリエンス | tenacity | リトライ・バックオフ標準 |
| ログ/トレース | OpenTelemetry Python SDK | 既存 k1s0-observability 互換 |
| テスト | pytest + pytest-asyncio + httpx | Python 標準テストスタック |
| フォーマット/リント | Ruff | 高速フォーマッタ・リンター（Black + isort + flake8 統合） |
| 型チェック | mypy | Python 標準の静的型チェッカー |
| パッケージ管理 | uv + pyproject.toml | 高速、PEP 標準準拠 |

### CLI 拡張

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| テンプレートレンダリング | Tera 1.19 | 既存テンプレートエンジン |
| CLI 引数 | clap 4.5 | 既存サブコマンド体系 |
| 名前変換 | heck | kebab → snake_case 変換（Python 命名規約） |

## 機能 1: backend-python テンプレート（feature）

### 1.1 設計方針

- Python の標準的な src レイアウト（`src/{package_name}/`）を採用
- Clean Architecture の各層をサブパッケージとし、import 文で依存方向を制御
- Rust / Go / C# テンプレートと同等の Tera 変数（`feature_name`, `feature_name_snake`, `with_grpc`, `with_db` 等）を使用
- `config/*.yaml` による設定管理（環境変数禁止）

### 3.3 k1s0-error の主要型

```python
# src/k1s0_error/exception.py
"""k1s0 統一例外基底クラス"""
from __future__ import annotations


class K1s0Exception(Exception):
    """k1s0 統一例外基底クラス"""

    def __init__(
        self,
        error_code: str,
        message: str,
        http_status: int = 500,
        trace_id: str | None = None,
    ) -> None:
        super().__init__(message)
        self.error_code = error_code  # e.g., "auth.invalid_credentials"
        self.http_status = http_status  # e.g., 404
        self.trace_id = trace_id


class NotFoundException(K1s0Exception):
    """リソース未検出例外"""

    def __init__(self, error_code: str, message: str) -> None:
        super().__init__(error_code, message, http_status=404)


class ValidationException(K1s0Exception):
    """バリデーション例外"""
```

### 3.4 k1s0-config の主要型

```python
# src/k1s0_config/loader.py
"""k1s0 YAML 設定管理"""
from __future__ import annotations

from pathlib import Path
from typing import Any

import yaml


def load_config(
    env: str | None = None,
    config_dir: str = "config",
    secrets_dir: str | None = None,
) -> dict[str, Any]:
    """config/{env}.yaml を読み込む

    1. config/default.yaml を読み込み
    2. env 引数から環境名を取得
    3. config/{env}.yaml をオーバーレイ
    4. secrets_dir からシークレットファイルを読み込み
    """
    config_path = Path(config_dir)
    config: dict[str, Any] = {}

    # default.yaml を読み込み
    default_file = config_path / "default.yaml"
    if default_file.exists():
        with default_file.open() as f:
            config = yaml.safe_load(f) or {}

    # 環境別 YAML をオーバーレイ
    if env:
        env_file = config_path / f"{env}.yaml"
        if env_file.exists():
            with env_file.open() as f:
                env_config = yaml.safe_load(f) or {}
                _deep_merge(config, env_config)

    # シークレットファイル読み込み
    if secrets_dir:
        _load_secrets(config, Path(secrets_dir))

    return config
```
