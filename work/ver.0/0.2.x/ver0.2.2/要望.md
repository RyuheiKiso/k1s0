# 要望

CLIのユーザー体験（UX）を向上させたい。
現在のk1s0 CLIは機能的には充実しているが、操作中のフィードバックや誤操作防止、エラーリカバリの面でユーザーに不親切な部分がある。これらを改善し、初心者からパワーユーザーまでストレスなく利用できるCLIを実現したい。

## 背景

- `new-feature`や`upgrade`コマンドでファイル生成後に結果が表示されるが、実行前に何が起きるか確認できない
- 大量のファイル生成やlint実行中に進捗が表示されず、動いているか分からない
- `doctor`コマンドは単独実行のみで、`init`時や`new-feature`時にツール不足を自動検知しない
- `--help`のテキストが引数説明のみで、使用例や影響範囲が記載されていない
- エラー発生時にヒントは出るが、実行可能なリカバリコマンドが提示されない

## 回答（実装方針）

### 概要

5つのUX改善機能を実装する。

1. **実行前プレビュー＆確認画面**: 変更を伴うコマンド実行前にドライラン結果を表示し、ユーザー確認を求める
2. **リアルタイム進捗フィードバック**: 既存の`progress_bar`/`spinner` APIを活用し、長時間操作に進捗表示を追加
3. **doctor自動実行統合**: `init`/`new-feature`時に必要ツールの不足を自動検知し、インストール手順を提示
4. **ヘルプテキスト拡充**: 各コマンドに使用例（Examples）と影響範囲（生成されるもの）セクションを追加
5. **エラー時自動リカバリ提案**: エラー発生時にコピー可能な解決コマンドを提示

### 技術選定

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| 進捗表示 | indicatif（既存） | progress_bar/spinner API が実装済み |
| 確認プロンプト | inquire（既存） | 対話モードで使用中のプロンプトライブラリ |
| カラー出力 | console（既存） | 既存の output.rs スタイリング基盤 |
| ヘルプテキスト | clap 4.5（既存） | after_help / after_long_help 属性 |

### 設計方針

- **既存API最大活用**: `output.rs`の`progress_bar()`/`spinner()`、`error()`のhint機能をフル活用
- **非破壊的変更**: 既存のCLIインターフェースに変更なし。新機能はオプションまたは自動的に適用
- **CI互換**: `--json`モード・非TTY環境では確認プロンプトをスキップし、進捗表示も抑制
- **日本語ローカライズ**: 既存の日本語メッセージ体系に統一

## 機能 1: 実行前プレビュー＆確認画面

### 対象コマンド

| コマンド | プレビュー内容 |
|---------|--------------|
| `new-feature` | 生成されるファイル/ディレクトリ一覧 |
| `new-domain` | 生成されるファイル/ディレクトリ一覧 |
| `new-screen` | 生成されるファイル一覧 |
| `upgrade` | 変更/追加/削除されるファイル一覧（diff要約） |

### 動作仕様

```
$ k1s0 new-feature --type backend-rust --name order-processing

📋 以下のファイルを生成します:
  + feature/backend/rust/order-processing/Cargo.toml
  + feature/backend/rust/order-processing/src/main.rs
  + feature/backend/rust/order-processing/src/domain/mod.rs
  + feature/backend/rust/order-processing/config/default.yaml
  + feature/backend/rust/order-processing/.k1s0/manifest.json
  ... 他 9 ファイル（計 14 ファイル、5 ディレクトリ）

続行しますか？ [Y/n]
```

### スキップ条件

- `--yes`/`-y` フラグ: 確認をスキップ
- `--json` モード: 確認をスキップ（プレビュー内容はJSON出力に含める）
- 非TTY環境: 確認をスキップ

## 機能 2: リアルタイム進捗フィードバック

### 対象操作と表示形式

| 操作 | 表示形式 | 例 |
|------|---------|-----|
| ファイル生成（new-feature等） | プログレスバー | `⠋ ファイルを生成中... [████████░░░░] 8/14 src/domain/mod.rs` |
| lint チェック | スピナー + ファイル数 | `⠋ lint 実行中... 42 ファイルをチェック済み` |
| upgrade 差分計算 | スピナー | `⠋ 変更を計算中...` |
| upgrade ファイル適用 | プログレスバー | `⠋ 更新を適用中... [████░░░░░░░░] 3/12` |

### 抑制条件

- `--json` モード: 進捗表示なし
- `--no-color` モード: ANSI非対応の簡素表示
- 非TTY環境: 進捗表示なし

## 機能 3: doctor 自動実行統合

### 自動実行タイミング

| コマンド | チェック内容 |
|---------|-------------|
| `init` | 基本ツール（Rust/Node等プロジェクト設定に応じて） |
| `new-feature --type backend-rust` | cargo, rustfmt, clippy |
| `new-feature --type backend-go` | go, gofmt, golangci-lint |
| `new-feature --type backend-csharp` | dotnet |
| `new-feature --type backend-python` | python, uv, ruff, mypy |
| `new-feature --type frontend-react` | node, pnpm |
| `new-feature --type frontend-flutter` | flutter, dart |

### 動作仕様

```
$ k1s0 new-feature --type backend-python --name order-processing

⚠ 環境チェック:
  ✗ uv が見つかりません
    インストール: curl -LsSf https://astral.sh/uv/install.sh | sh
    詳細: https://docs.astral.sh/uv/getting-started/installation/
  ✗ ruff が見つかりません
    インストール: uv tool install ruff
  ✓ python 3.12.x

不足ツールがあります。続行しますか？ [Y/n]
```

### スキップ条件

- `--skip-doctor` フラグ: 環境チェックをスキップ
- `--json` モード: チェック結果をJSON出力に含めるが、プロンプトなし

## 機能 4: ヘルプテキスト拡充

### 追加セクション

各コマンドの `--help` に以下を追加:

1. **Examples**: 具体的な使用例（2-3個）
2. **生成されるもの / 影響範囲**: コマンドが何を行うかの概要

### 例: new-feature --help

```
使用例:
  # Rust バックエンドサービスを生成
  k1s0 new-feature --type backend-rust --name order-processing

  # gRPC + DB 付きの Python サービスを生成
  k1s0 new-feature --type backend-python --name user-auth --with-grpc --with-db

  # 対話モードで生成
  k1s0 new-feature -i

生成されるもの:
  feature/{backend|frontend}/{language}/{name}/ 配下に
  Clean Architecture 構成のスキャフォールドを生成します。
  （テンプレートにより 10-20 ファイル、5-8 ディレクトリ）
```

### 実装方法

clapの `after_help` / `after_long_help` 属性を使用。

## 機能 5: エラー時自動リカバリ提案

### 対象エラーとリカバリコマンド

| エラー状況 | 提案内容 |
|-----------|---------|
| ファイル衝突（upgrade） | `k1s0 upgrade --force` / `--skip {file}` |
| manifest.json 不正（lint） | `k1s0 init` で再初期化 |
| 必須ファイル不足（lint K010/K011） | `k1s0 lint --fix` で自動修復 |
| 出力先ディレクトリ既存（new-feature） | `--force` で上書き / 別名で再実行 |
| ツール不足（doctor/自動チェック） | インストールコマンドをコピー可能な形式で提示 |

### 動作仕様

```
✗ ファイル衝突: src/main.rs が変更されています
  ヒント: 以下のいずれかで解決できます
    1. k1s0 upgrade --force           # テンプレートの内容で上書き
    2. k1s0 upgrade --skip src/main.rs  # このファイルをスキップ
```

### 実装方法

既存の `CliError` の `hint` フィールドを拡張し、複数行のリカバリ提案を記述可能にする。

## 主要機能一覧

| 機能 | 対象 | 状態 |
|------|------|------|
| 実行前プレビュー＆確認画面 | new-feature, new-domain, new-screen, upgrade | 新規 |
| リアルタイム進捗フィードバック | new-feature, new-domain, lint, upgrade | 改修 |
| doctor 自動実行統合 | init, new-feature | 改修 |
| ヘルプテキスト拡充 | 全コマンド | 改修 |
| エラー時自動リカバリ提案 | upgrade, lint, new-feature | 改修 |

## 期待される効果

1. **誤操作防止**: 実行前プレビューにより、意図しないファイル生成・変更を事前に確認可能
2. **操作中の安心感**: 進捗表示により「動いているか分からない」問題を解消
3. **初期セットアップの摩擦軽減**: doctor自動実行により、ツール不足を即座に検知・解決手順を提示
4. **自己解決率の向上**: ヘルプ拡充とリカバリ提案により、ドキュメント参照や質問の必要性を低減
5. **CI互換性の維持**: `--json`/非TTY環境では自動的にプロンプト・進捗を抑制し、既存のCI/CDパイプラインに影響なし

---

## 注意事項

- **後方互換性**: 既存のCLIインターフェース（引数・出力形式）は変更しない
- **CI/CD互換**: `--json`モード・非TTY環境では確認プロンプト・進捗表示を自動スキップ
- **パフォーマンス**: doctor自動チェックは該当ツールのみに限定し、全ツールチェックは行わない
- **ローカライズ**: 全メッセージは既存の日本語体系に統一
