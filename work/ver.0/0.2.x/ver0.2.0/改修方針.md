# k1s0 v0.2.0 改修方針

## 概要

バックエンドの対応言語に C#（ASP.NET Core 8.0）を追加する。

- **`backend-csharp` テンプレート新設**: feature / domain 両方のスキャフォールド生成
- **CLI 拡張**: `ServiceType` / `DomainType` enum に `BackendCsharp` 追加
- **フレームワークパッケージ**: C# 版 k1s0 共通ライブラリ（NuGet）8パッケージ
- **lint ルール C# 対応**: K010/K011/K020/K021/K022 の C# パターン追加
- **CI/CD**: `csharp.yml` ワークフロー追加
- **manifest.schema.json 更新**: `language` enum に `"csharp"` 追加

## 機能目的

- Rust / Go に並ぶ第3のバックエンド言語として C# をサポートし、.NET 開発者が k1s0 プラットフォームを利用可能にする
- 同一の Clean Architecture 規約・lint・テンプレート生成の仕組みを C# にも適用する
- エンタープライズ領域での k1s0 採用を促進する

## 技術選定

### C# バックエンド

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| Web フレームワーク | ASP.NET Core 8.0 | LTS、高パフォーマンス、gRPC ネイティブ |
| DI | Microsoft.Extensions.DependencyInjection | ASP.NET Core 標準 |
| gRPC | Grpc.AspNetCore | 公式実装、HTTP/2 対応 |
| REST | ASP.NET Core Controllers | 標準的な REST 実装 |
| ORM | Entity Framework Core 8.0 | マイグレーション・LINQ 対応 |
| バリデーション | FluentValidation | Fluent API、Zod ライクな体験 |
| 設定管理 | YamlDotNet + IConfiguration | k1s0 YAML 設定規約互換 |
| レジリエンス | Polly v8 | リトライ・サーキットブレーカ標準 |
| ログ/トレース | OpenTelemetry .NET SDK | 既存 k1s0-observability 互換 |
| テスト | xUnit + Moq + FluentAssertions | .NET 標準テストスタック |
| フォーマット | dotnet format | CI 強制可能 |
| パッケージ管理 | NuGet + Central Package Management | バージョン統一 |

### CLI 拡張

| 機能 | 使用技術 | 理由 |
|------|---------|------|
| テンプレートレンダリング | Tera 1.19 | 既存テンプレートエンジン |
| CLI 引数 | clap 4.5 | 既存サブコマンド体系 |

## 機能 1: backend-csharp テンプレート（feature）

### 1.1 設計方針

- C# の標準的なプロジェクト構造（.sln + Multi-Project）を採用
- Clean Architecture の各層を独立した .csproj プロジェクトとし、`<ProjectReference>` で依存方向を制御
- Rust / Go テンプレートと同等の Tera 変数（`feature_name`, `feature_name_pascal`, `with_grpc`, `with_db` 等）を使用
- `config/*.yaml` による設定管理（環境変数禁止）

namespace K1s0.Error;

/// <summary>
/// k1s0 統一例外基底クラス

public class K1s0Exception : Exception
{
    public string ErrorCode { get; }     // e.g., "auth.invalid_credentials"
    public int HttpStatus { get; }       // e.g., 404
    public string? TraceId { get; }

    public K1s0Exception(string errorCode, string message, int httpStatus = 500)
        : base(message)
    {
        ErrorCode = errorCode;
        HttpStatus = httpStatus;
    }
}

/// <summary>
/// RFC 7807 Problem Details レスポンス
/// </summary>
public record ProblemDetails(
    int Status,
    string Title,
    string Detail,
    string ErrorCode,
    string? TraceId
);

## 機能 5: CI/CD ワークフロー

### 5.1 csharp.yml

```yaml
# .github/workflows/csharp.yml
name: C# Backend

on:
  push:
    branches: [main, develop]
    paths:
      - 'framework/backend/csharp/**'
      - 'feature/backend/csharp/**'
      - 'domain/backend/csharp/**'
  pull_request:
    paths:
      - 'framework/backend/csharp/**'
      - 'feature/backend/csharp/**'
      - 'domain/backend/csharp/**'

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Setup .NET
        uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '8.0.x'

      - name: Restore dependencies
        run: dotnet restore

      - name: Build
        run: dotnet build --no-restore --configuration Release

      - name: Test
        run: dotnet test --no-build --configuration Release --verbosity normal

      - name: Format check
        run: dotnet format --verify-no-changes
```
