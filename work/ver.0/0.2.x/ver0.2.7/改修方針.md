# k1s0 v0.2.7 改修方針

## 概要

Kotlin エコシステムを包括的にサポートする。以下の2つのテンプレートを新設する：

### backend-kotlin（バックエンド）
- `CLI/templates/backend-kotlin/` — feature / domain テンプレート
- `framework/backend/kotlin/` — Kotlin バックエンド用 framework パッケージ群（12パッケージ）

### frontend-android(Kotlin)（Android フロントエンド）
- `CLI/templates/frontend-android(Kotlin)/` — feature / domain / screen テンプレート
- `framework/frontend/android/` — Android 用 framework パッケージ群（8パッケージ）

### 共通変更
- CLI lint ルール拡張 — K020, K022, K029 等に Kotlin パターンを追加
- CI/CD ワークフロー — `kotlin.yml`, `frontend-android(Kotlin).yml` の新設
- `manifest.schema.json` — `language` enum に `"kotlin"` 追加
- `migrate analyze` — Kotlin / Android プロジェクトの種別推定

## 機能目的

1. JVM エコシステム（Kotlin バックエンド + Android フロントエンド）を包括的にカバー
2. 既存6テンプレートと同等の scaffold 生成・lint・upgrade・docker・playground 機能を提供
3. Kotlin の言語統一性を活かし、バックエンド（Ktor）とフロントエンド（Compose）で同一言語による開発
4. framework パッケージにより、Kotlin/Android チームが k1s0 規約に準拠した開発を行える基盤を提供

### Lint ルール拡張

#### K020: 環境変数使用禁止 — Kotlin パターン

```rust
pub fn kotlin() -> Self {
    Self {
        file_extensions: vec!["kt", "kts"],
        patterns: vec![
            EnvVarPattern { pattern: "System.getenv", hint: "..." },
            EnvVarPattern { pattern: "System.getProperty", hint: "..." },
            EnvVarPattern { pattern: "ProcessBuilder", hint: "..." },
            EnvVarPattern { pattern: "dotenv", hint: "..." },
            EnvVarPattern { pattern: "BuildConfig.", hint: "BuildConfig の使用は禁止されています。config/{env}.yaml を使用してください。" },
        ],
    }
}
```

**注**: `BuildConfig.` は Android 固有パターン。ビルド時の定数埋め込み（`BuildConfig.API_URL` 等）を検出。

#### K022: Clean Architecture 依存方向 — Kotlin 対応

バックエンド・Android 共通:
```
import dev.k1s0.feature.{name}.infrastructure.* // domain から参照 → 違反
import dev.k1s0.feature.{name}.presentation.*   // domain から参照 → 違反
```

Android 固有:
```
import dev.k1s0.app.{name}.presentation.*       // domain から参照 → 違反
import android.widget.*                          // domain 層での Android UI 依存 → 違反
```

#### K029: panic/unwrap/expect — Kotlin パターン

| パターン | 説明 |
|---------|------|
| `!!` | non-null assertion |
| `error(` | RuntimeException のスロー |
| `TODO(` | NotImplementedError のスロー |
| `throw RuntimeException` | 非チェック例外の直接スロー |

テストファイル（`*Test.kt`, `*Spec.kt`）とエントリポイント（`Main.kt`, `*Application.kt`）は除外。

#### K050: SQL インジェクション — Kotlin パターン

```kotlin
"SELECT * FROM users WHERE id = $id"          // 文字列テンプレート
"SELECT * FROM users WHERE id = " + id        // 文字列連結
```

#### K053: センシティブデータログ — Kotlin パターン

```kotlin
logger.info("password: $password")
log.info { "token=$token" }
Log.d(TAG, "secret: $secret")       // Android Log
```

### manifest.schema.json 変更

```json
"language": {
  "enum": ["rust", "go", "csharp", "python", "kotlin", "typescript", "dart"]
}
```

**注**: `"kotlin"` は backend-kotlin と frontend-android(Kotlin) の両方で使用。`service.type` で `"backend"` / `"frontend"` を区別。

### migrate analyze 拡張

#### バックエンド Kotlin

| 検出ファイル | 推定結果 | 確信度 |
|-------------|---------|:------:|
| `build.gradle.kts` + `src/main/kotlin/` (Android 非依存) | backend-kotlin | 95% |
| `build.gradle.kts` + Ktor 依存 | backend-kotlin (Ktor) | 90% |
| `build.gradle.kts` + Spring Boot 依存 | backend-kotlin (Spring Boot) | 90% |

#### Android

| 検出ファイル | 推定結果 | 確信度 |
|-------------|---------|:------:|
| `build.gradle.kts` + `AndroidManifest.xml` | frontend-android(Kotlin) | 95% |
| `build.gradle.kts` + AGP 依存 + Compose 依存 | frontend-android(Kotlin) (Compose) | 95% |
| `build.gradle.kts` + AGP 依存 (View ベース) | frontend-android(Kotlin) (View) | 80% |

判定ロジック: `com.android.application` プラグインまたは `android {}` ブロックの存在で Android を優先判定。

## Part 4: CI/CD ワークフロー

### kotlin.yml（バックエンド）

```yaml
name: Kotlin Backend
on:
  push:
    branches: [main, develop]
    paths:
      - 'framework/backend/kotlin/**'
      - 'feature/backend/kotlin/**'
      - 'domain/backend/kotlin/**'
  pull_request:
    paths: [same as above]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4

      - uses: gradle/actions/setup-gradle@v4
      - run: ./gradlew ktlintCheck
      - run: ./gradlew detekt
      - run: ./gradlew build
```

- **Tera/Kotlin `$` 競合**: `{% raw %}` ブロックで Kotlin/Gradle の `$` をエスケープ
- **バックエンドとフロントエンドで同一 `language: "kotlin"`**: `service.type` (`"backend"` / `"frontend"`) で区別。lint ルールは `type` も参照して Android 固有パターン（`BuildConfig.`, `Log.d`）を適用
- **Android は Docker 非対象**: frontend-android(Kotlin) テンプレートに Dockerfile は含まない（APK/AAB ビルド）
- **AGP (Android Gradle Plugin) バージョン**: Gradle 8.x と AGP 8.x の互換性を `libs.versions.toml` で管理
- **Android SDK**: `doctor` で `ANDROID_HOME` の存在を確認。環境変数禁止ルール K020 の除外対象として `JAVA_HOME`, `ANDROID_HOME`, `GRADLE_HOME` を追加
- **Gradle マルチモジュール**: Android テンプレートは `app/` モジュールを持つマルチモジュール構成（Android 標準）
- **Screen テンプレート**: Flutter の `screen/` テンプレートパターンに準拠。Screen + ViewModel + Navigation エントリの3ファイルを生成
