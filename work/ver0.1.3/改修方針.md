# k1s0 v0.1.3 改修方針

## 概要

開発環境の健全性をチェックする `k1s0 doctor` コマンドを実装する。

## 機能目的

- 初回セットアップ時の躓きを減らす
- チーム全員が同じ環境で開発できるよう支援
- CI/CDでの環境検証にも使用可能

## チェック対象

### 必須ツール

| ツール | 最小バージョン | チェックコマンド | 用途 |
|--------|---------------|-----------------|------|
| Rust | 1.85.0 | `rustc --version` | CLI本体、backend-rust |
| Node | 20.0.0 | `node --version` | frontend-react |
| pnpm | 9.15.4 | `pnpm --version` | Node パッケージ管理 |

### オプションツール

| ツール | 推奨バージョン | チェックコマンド | 関連テンプレート |
|--------|---------------|-----------------|-----------------|
| Go | 1.21.0 | `go version` | backend-go |
| golangci-lint | 1.55.0 | `golangci-lint --version` | backend-go |
| buf | 1.28.0 | `buf --version` | backend-rust, backend-go |
| Flutter | 3.16.0 | `flutter --version` | frontend-flutter |
| Dart | 3.2.0 | `dart --version` | frontend-flutter |

## コマンドインターフェース

```bash
# 基本実行
k1s0 doctor

# オプション
k1s0 doctor --verbose    # 詳細情報（パス、全バージョン）
k1s0 doctor --json       # JSON出力（CI向け）
k1s0 doctor --check <category>  # 特定カテゴリのみ
```

### カテゴリ

| カテゴリ | チェック対象 |
|---------|-------------|
| `rust` | Rust, Cargo |
| `go` | Go, golangci-lint |
| `node` | Node, pnpm |
| `flutter` | Flutter, Dart |
| `proto` | buf |
| `all` | 全て（デフォルト） |

## 終了コード

| コード | 意味 | 使用場面 |
|--------|------|---------|
| 0 | 成功 | 必須ツール全てOK |
| 1 | エラー | 必須ツールに問題あり |
| 2 | 警告 | オプションツールのみ問題（--strict時はエラー） |

## 設計方針

### 1. 非同期並列チェック

```rust
// 各ツールのチェックを並列実行
let checks = tokio::join!(
    check_rust(),
    check_node(),
    check_pnpm(),
    check_go(),
    // ...
);
```

### 2. バージョン比較

```rust
use semver::{Version, VersionReq};

fn check_version(actual: &str, required: &str) -> CheckResult {
    let actual = Version::parse(actual)?;
    let required = VersionReq::parse(&format!(">={}", required))?;
    if required.matches(&actual) {
        CheckResult::Ok
    } else {
        CheckResult::VersionMismatch { actual, required }
    }
}
```

### 3. プラットフォーム対応

| プラットフォーム | 考慮事項 |
|-----------------|---------|
| Windows | `.exe` 拡張子、パス区切り、PowerShell |
| macOS | Homebrew パス、arm64/x86_64 |
| Linux | 各ディストリビューションの差異 |

## 注意事項

- **タイムアウト**: 各コマンド実行は最大5秒でタイムアウト
- **エラー耐性**: 1つのチェック失敗で全体を止めない
- **ネットワーク不要**: 全てローカルチェック
- **キャッシュなし**: 常に最新状態を確認
