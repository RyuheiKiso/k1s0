# 要望

k1s0 CLIを使い始める際に、開発環境が正しくセットアップされているか確認したい。
必要なツールがインストールされているか、バージョンが適切か、一目で分かると嬉しい。

## 回答（実装方針）

### 概要

`k1s0 doctor` コマンドを実装し、開発環境の健全性をチェックする診断機能を提供する。

### 機能概要

- 必須/推奨ツールのインストール状況確認
- バージョン互換性チェック
- 問題発見時の推奨アクション提示
- カラー出力による視覚的なステータス表示

### 対話フロー

```
$ k1s0 doctor

k1s0 環境診断

k1s0 CLI
  k1s0: v0.1.3

必須ツール
  Rust:    1.85.0
  Node:    v20.10.0
  pnpm:    9.15.4

オプションツール
  Go:              not found (backend-go テンプレートに必要)
  golangci-lint:   not found
  buf:             1.28.1
  Flutter:         not found (frontend-flutter テンプレートに必要)

推奨アクション:
  1. Go をインストール: https://go.dev/dl/
  2. golangci-lint をインストール: brew install golangci-lint
  3. Flutter をインストール: https://flutter.dev/docs/get-started/install
```

### チェック項目

#### 必須ツール（k1s0 CLI動作に必要）
| ツール | 最小バージョン | 用途 |
|--------|---------------|------|
| Rust | 1.85.0 | CLI本体のビルド |
| Node | 20.x | フロントエンド開発 |
| pnpm | 9.15.4 | Node パッケージ管理 |

#### オプションツール（テンプレート別に必要）
| ツール | 関連テンプレート | 用途 |
|--------|-----------------|------|
| Go | backend-go | Go バックエンド開発 |
| golangci-lint | backend-go | Go lint |
| buf | backend-rust, backend-go | Protocol Buffers |
| Flutter | frontend-flutter | Flutter 開発 |
| Dart | frontend-flutter | Dart 言語 |

### 実装方針

#### 1. コマンド構造

```bash
# 基本実行
k1s0 doctor

# 詳細出力
k1s0 doctor --verbose

# JSON出力（CI向け）
k1s0 doctor --json

# 特定カテゴリのみ
k1s0 doctor --check rust
k1s0 doctor --check go
k1s0 doctor --check frontend
```

#### 2. 終了コード

| コード | 意味 |
|--------|------|
| 0 | 全チェック成功 |
| 1 | 必須ツールに問題あり |
| 2 | オプションツールのみ問題あり（警告） |

#### 3. 出力形式

- デフォルト: 人間可読なカラー出力
- `--json`: 機械可読なJSON出力
- `--verbose`: バージョン情報、パス、詳細診断を追加

### コード構成案

```
CLI/crates/k1s0-cli/src/
├── commands/
│   ├── doctor.rs        # 新規追加
│   └── ...
├── doctor/              # 新規追加
│   ├── mod.rs
│   ├── checker.rs       # 各ツールのチェックロジック
│   ├── requirements.rs  # 必須/推奨バージョン定義
│   └── output.rs        # 出力フォーマット
└── ...
```

### 注意事項

- **クロスプラットフォーム**: Windows/macOS/Linux で動作
- **タイムアウト**: 各コマンド実行に適切なタイムアウトを設定
- **キャッシュ**: 繰り返し実行時のパフォーマンス考慮（オプション）
- **ネットワーク不要**: オフラインでも動作
