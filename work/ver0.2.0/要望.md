# 要望

バックエンドの対応言語に C#（.NET）を追加したい。
現在 Rust と Go の2言語でバックエンドサービスを生成・管理できるが、エンタープライズ領域で広く使われている C# / .NET もサポートすることで、k1s0 プラットフォームの適用範囲を拡大したい。

## 背景

- エンタープライズ開発では C# / ASP.NET Core が主要な選択肢
- 既存の Rust / Go 開発者だけでなく、.NET 開発者もチームに多い
- k1s0 の Clean Architecture 強制・lint・テンプレート生成の恩恵を C# にも適用したい
- gRPC / REST / OpenAPI の API コントラクト管理は言語非依存のため、C# にも同様に適用可能

## 回答（実装方針）

### 概要

1. **`backend-csharp` テンプレート**: C# / ASP.NET Core ベースのバックエンドスキャフォールド（feature + domain）
2. **CLI 拡張**: `ServiceType` / `DomainType` enum に `BackendCsharp` を追加
3. **フレームワークパッケージ**: C# 版の k1s0 共通ライブラリ（NuGet パッケージ）
4. **lint ルール対応**: K010/K011 必須ファイル、K020 環境変数禁止、K022 Clean Architecture 依存方向を C# に対応
5. **CI/CD ワークフロー**: C# 用の GitHub Actions ワークフロー追加

### 技術選定

| 機能 | 技術 | 理由 |
|------|------|------|
| Web フレームワーク | ASP.NET Core 8.0 | LTS、高パフォーマンス、gRPC ネイティブ対応 |
| DI | Microsoft.Extensions.DependencyInjection | ASP.NET Core 標準 |
| gRPC | Grpc.AspNetCore | 公式 gRPC 実装 |
| REST | ASP.NET Core Minimal API / Controllers | 標準 REST 実装 |
| ORM | Entity Framework Core 8.0 | 標準的な ORM、マイグレーション対応 |
| バリデーション | FluentValidation | Zod ライクな fluent API |
| 設定管理 | Microsoft.Extensions.Configuration（YAML） | k1s0-config 互換 |
| テスト | xUnit + Moq + FluentAssertions | .NET 標準テストスタック |
| ログ/トレース | OpenTelemetry .NET SDK | k1s0-observability 互換 |
| パッケージ管理 | NuGet | .NET 標準 |
| ビルドツール | dotnet CLI (8.0+) | .NET SDK 標準 |

### 設計方針

- **既存パターン踏襲**: Rust / Go テンプレートと同じ Clean Architecture ディレクトリ構造
- **k1s0 規約準拠**: 環境変数禁止（K020）、シークレットハードコード禁止（K021）、CA 依存方向（K022）
- **manifest.json 互換**: 既存のスキーマを拡張し `language: "csharp"` を追加
- **段階的導入**: v0.2.0 ではテンプレート + lint 対応まで、フレームワークパッケージは後続バージョン

## 機能 4: lint ルール C# 対応

### K010/K011: 必須ディレクトリ・ファイル

**必須ディレクトリ（feature）:**
- `src/`
- `src/{Name}.Domain/`
- `src/{Name}.Application/`
- `src/{Name}.Infrastructure/`
- `src/{Name}.Presentation/`
- `config/`
- `deploy/`

**必須ファイル（feature）:**
- `*.sln`
- `src/{Name}.Presentation/{Name}.Presentation.csproj`
- `config/default.yaml`
- `.k1s0/manifest.json`

**必須ディレクトリ（domain）:**
- `{Name}.Domain/`
- `{Name}.Domain/Entities/`
- `{Name}.Domain/ValueObjects/`
- `{Name}.Domain/Repositories/`

**必須ファイル（domain）:**
- `{Name}.Domain/{Name}.Domain.csproj`
- `.k1s0/manifest.json`

### K020: 環境変数禁止

**C# 禁止パターン:**
- `Environment.GetEnvironmentVariable`
- `Environment.GetEnvironmentVariables`
- `Environment.ExpandEnvironmentVariables`
- `ConfigurationBuilder().AddEnvironmentVariables()`

**正しい方法:** `config/*.yaml` + `K1s0.Config` パッケージ

### K021: シークレットハードコード禁止

C# でも同様に YAML / `appsettings.json` 内の `password`, `secret`, `api_key`, `token`, `credential`, `private_key` の直接値を禁止。

### K022: Clean Architecture 依存方向

C# プロジェクト参照（`.csproj` の `<ProjectReference>`）を解析し、以下を禁止:
- `Domain → Application`
- `Domain → Presentation`
- `Domain → Infrastructure`
- `Application → Presentation`

## 機能 5: CI/CD ワークフロー

### csharp.yml

```yaml
name: C# Backend
on:
  push:
    branches: [main, develop]
    paths:
      - 'framework/backend/csharp/**'
      - 'feature/backend/csharp/**'
      - 'domain/backend/csharp/**'

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-dotnet@v4
        with:
          dotnet-version: '8.0.x'
      - run: dotnet restore
      - run: dotnet build --no-restore
      - run: dotnet test --no-build --verbosity normal
      - run: dotnet format --verify-no-changes
```

## 期待される効果

1. **対応言語拡大**: Rust / Go に加え C# をサポートし、.NET 開発者も k1s0 プラットフォームを利用可能
2. **エンタープライズ対応**: C# / ASP.NET Core は大規模エンタープライズで広く採用されており、導入先が拡大
3. **統一ガバナンス**: 言語が異なっても同じ lint ルール・テンプレート・コントラクト管理が適用される
4. **Clean Architecture の強制**: C# プロジェクト参照を使った依存方向の検証が可能
5. **既存資産の活用**: gRPC / OpenAPI コントラクト、Kubernetes デプロイメント、設定管理は言語非依存で再利用

## 注意事項

- **ターゲットフレームワーク**: .NET 8.0 LTS を対象とする（.NET 6.0 は EOL、.NET 9.0 は STS）
- **環境変数禁止**: `Environment.GetEnvironmentVariable()` は K020 違反。`config/*.yaml` を使用すること
- **プロジェクト構造**: C# は Multi-Project Solution（.sln + 複数 .csproj）が標準であり、各 CA 層を独立プロジェクトとする
- **NuGet パッケージ管理**: Central Package Management（`Directory.Packages.props`）を使用し、バージョン統一を図る
- **Nullable Reference Types**: 全プロジェクトで `<Nullable>enable</Nullable>` を必須とする
- **コードフォーマット**: `dotnet format` を CI で強制
