# k1s0 v0.2.4 作業チェックリスト

## 概要

Python と C# のバックエンドフレームワークに4パッケージずつ（計8パッケージ）を新規実装する。

- **k1s0-domain-event / K1s0.DomainEvent**: ドメインイベント、Outbox パターン
- **k1s0-resilience / K1s0.Resilience**: サーキットブレーカー、タイムアウト、バルクヘッド、リトライ
- **k1s0-cache / K1s0.Cache**: Redis キャッシュ、キャッシュパターン
- **k1s0-auth / K1s0.Auth**: JWT/OIDC 認証、ポリシーエンジン、ミドルウェア

## 受け入れ基準

### 機能要件

- [x] Python 4パッケージが全て `import` 可能
- [x] C# 4プロジェクトが全て `dotnet build` 成功
- [x] 全パッケージのパブリック API が Rust 版と概念レベルで一致
- [x] InMemory 実装がデフォルトで動作（外部サービス不要）
- [x] Optional 依存（Redis, DB）が未インストールでもコア機能は動作

### テスト要件

- [x] Python: 全パッケージで `pytest` パス
- [x] C#: 全プロジェクトで `dotnet test` パス（17 + 27 + 24 + 29 = 97 tests）
- [x] 各パッケージの主要パス（正常系 + エラー系）がテスト済み

### 非機能要件

- [x] Tier 依存ルール遵守（Tier 2 → Tier 1 のみ、Tier 3 → Tier 1/2 のみ）
- [x] 環境変数禁止ルール（K020）違反なし
- [x] 既存パッケージとの命名規則・構造の一貫性
- [x] CI ワークフローで全テストがパス
