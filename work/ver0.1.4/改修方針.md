# k1s0 v0.1.4 改修方針

## 概要

フロントエンドフレームワーク（React/Flutter）に DataTable コンポーネントと Form Generator を追加する。

## 機能目的

- ビジネスアプリケーション開発の効率化
- UI コンポーネントの一貫性確保
- ドメイン層との連携強化（スキーマ駆動開発）

## 技術選定

### React

| 機能 | 使用ライブラリ | 理由 |
|------|---------------|------|
| DataTable | **@mui/x-data-grid** | MUIエコシステム統合、高機能、TypeScript対応 |
| Form | **MUI + react-hook-form + Zod** | MUIコンポーネント活用、型安全なバリデーション |
| UI基盤 | **@mui/material v6** | 既存の@k1s0/uiと統一 |

### Flutter

| 機能 | 使用ライブラリ | 理由 |
|------|---------------|------|
| DataTable | **カスタム実装** | Material 3準拠、Riverpod統合 |
| Form | **カスタム実装** | Freezed親和性、型安全性 |

## 機能 2: Form Generator

### 2.1 React 版 (`@k1s0/ui`) - MUI コンポーネントベース

#### 設計方針

- **MUI コンポーネント**（TextField, Select, Checkbox, Switch, RadioGroup 等）を使用
- **react-hook-form** でフォーム状態管理
- **Zod** でスキーマ定義とバリデーション
- **@hookform/resolvers** で Zod と react-hook-form を統合

#### MUI コンポーネントマッピング

| Zod 型 | MUI コンポーネント | 備考 |
|--------|-------------------|------|
| `z.string()` | `TextField` | 通常のテキスト入力 |
| `z.string().email()` | `TextField` type="email" | メール入力 |
| `z.string()` (長文) | `TextField` multiline | テキストエリア |
| `z.number()` | `TextField` type="number" | 数値入力 |
| `z.boolean()` | `Switch` または `Checkbox` | 設定で切替可能 |
| `z.enum([...])` | `Select` または `RadioGroup` | 選択肢数で自動判定 |
| `z.date()` | `DatePicker` (@mui/x-date-pickers) | 日付選択 |
| `z.array()` | 動的フィールドリスト | 追加/削除可能 |

#### 型定義

  // カスタムレンダー

// 生成されるフォームコンポーネントのProps
interface GeneratedFormProps<T> {

  values?: T; // 制御モード
  onSubmit: (values: T) => void | Promise<void>;
  onCancel?: () => void;
  onChange?: (values: Partial<T>) => void;
  disabled?: boolean;
  loading?: boolean;
  readOnly?: boolean;
  // react-hook-form インスタンスを外部から渡す（高度な使い方）
  form?: UseFormReturn<T>;
}

    this.showCancel = false,
    this.showReset = false,

final userFormSchema = K1s0FormSchema<UserInput>(

    ),

      name: 'email',
      label: 'メールアドレス',
      required: true,
      type: K1s0FieldType.email,
      validator: EmailValidator().validate,
    ),
    K1s0FormFieldSchema(
      name: 'age',
      label: '年齢',
      type: K1s0FieldType.number,
    ),
    K1s0FormFieldSchema(
      name: 'role',
      label: '権限',
      required: true,
      type: K1s0FieldType.select,
      options: [
        K1s0FieldOption(label: '管理者', value: 'admin'),
        K1s0FieldOption(label: '一般', value: 'user'),
        K1s0FieldOption(label: 'ゲスト', value: 'guest'),
      ],
    ),
    K1s0FormFieldSchema(
      name: 'notifications',
      label: '通知を受け取る',
      type: K1s0FieldType.switchField,
      defaultValue: true,
    ),
    K1s0FormFieldSchema(
      name: 'bio',
      label: '自己紹介',
      type: K1s0FieldType.textarea,
      helperText: '500文字以内',
    ),
  ],
  fromMap: (map) => UserInput(
    name: map['name'] as String,
    email: map['email'] as String,
    age: map['age'] as int?,
    role: map['role'] as String,
    notifications: map['notifications'] as bool? ?? true,
    bio: map['bio'] as String?,
  ),
  toMap: (user) => {
    'name': user.name,
    'email': user.email,
    'age': user.age,
    'role': user.role,
    'notifications': user.notifications,
    'bio': user.bio,
  },
);

class CreateUserPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final isLoading = ref.watch(createUserLoadingProvider);

    return Scaffold(
      appBar: AppBar(title: const Text('ユーザー作成')),
      body: Padding(
        padding: const EdgeInsets.all(16),
        child: K1s0Form<UserInput>(
          schema: userFormSchema,
          initialValues: const UserInput(
            name: '',
            email: '',
            role: 'user',

          submitLabel: '作成',
          showCancel: true,
          cancelLabel: 'キャンセル',
          columns: 2,
          loading: isLoading,
        ),
      ),
    );
  }
