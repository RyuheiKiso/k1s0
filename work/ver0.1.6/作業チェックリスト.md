# k1s0 v0.1.6 作業チェックリスト

## 概要

バックエンドフレームワーク（Rust/Go）にドメインイベント基盤パッケージを追加する。

- **k1s0-domain-event（Rust crate）**: ドメインイベントの定義・発行・購読・Outbox
- **k1s0-domain-event（Go module）**: Go 版の同等実装

## 受け入れ基準

### 機能要件（イベント定義）

- [x] DomainEvent trait/interface を実装したイベント型が作成できる
- [x] EventMetadata が自動生成される（event_id, occurred_at）
- [x] EventEnvelope がシリアライズ/デシリアライズできる
- [x] correlation_id / causation_id で相関・因果関係を設定できる

### 機能要件（Publisher / Subscriber）

- [x] EventPublisher でイベントを発行できる
- [x] EventSubscriber でイベントタイプ別に購読できる
- [x] EventHandler がイベント受信時に呼ばれる
- [x] SubscriptionHandle で購読をキャンセルできる
- [x] バッチ発行が動作する（デフォルト実装）

### 機能要件（InMemoryEventBus）

- [x] インメモリでの publish → subscribe フローが動作する
- [x] イベントタイプによるフィルタリングが動作する
- [x] 購読キャンセル後はハンドラが呼ばれない

### 機能要件（Outbox）

- [x] OutboxEntry が作成できる（Pending 状態）
- [x] OutboxStore trait で永続化インターフェースが定義されている
- [x] OutboxRelay がポーリングで pending エントリを取得・発行する
- [x] 正常配信後に mark_published される
- [x] 配信失敗時に mark_failed される
- [x] max_retries 超過時にログ出力 + failed マーク
- [x] graceful shutdown が動作する（oneshot / context.Done）

### 非機能要件

- [x] Rust: `cargo check` パス
- [x] Rust: `cargo test` 全テストパス（5 テスト + 1 doctest）
- [x] Rust: unsafe コード不使用（workspace.lints で forbid）
- [x] Go: `go build ./...` パス
- [x] Go: `go test ./...` 全テストパス
- [x] Tier 2 依存ルールに準拠（outbox feature でのみ k1s0-db に依存）

### 実装方針の差分（チェックリスト原案との違い）

| 項目 | 原案 | 実装 | 理由 |
|------|------|------|------|
| edition | 2024 | 2021 | 既存 crate と統一 |
| rust-version | 未指定 | workspace から継承（1.75） | workspace.package 準拠 |
| workspace | CLI/Cargo.toml | framework/backend/rust/Cargo.toml | 実際の workspace 位置 |
| Go module path | github.com/k1s0/... | github.com/RyuheiKiso/k1s0/... | 既存パターン準拠 |
| UUID | v7 | v4 | 既存 crate（k1s0-db 等）と統一 |
| K1s0Error From 実装 | あり | なし | k1s0-error への依存を避け独立性を維持 |
| version() メソッド | DomainEvent に含む | 省略 | YAGNI、必要時に追加可能 |
| Go EventEnvelope | ジェネリクス | json.RawMessage | Go 1.22 互換性、シンプルさ |
| go.work | 未言及 | 追加 | Go workspace に登録必要 |
