# k1s0 v0.1.2 改修方針

## 概要

CLIの操作性を向上させるため、Viteのような対話式インターフェースを実装する。

## 採用クレート

**`inquire`** を採用する。

- 高機能で美しいUI
- Viteのような洗練された対話体験を実現可能
- カスタマイズ性が高い
- アクティブにメンテナンスされている

```toml
# CLI/crates/k1s0-cli/Cargo.toml
[dependencies]
inquire = "0.7"
```

## 改修対象コマンド

| コマンド | 対話項目 | 優先度 |
|---------|---------|--------|
| `new-feature` | テンプレートタイプ、名前、オプション | P1 |
| `new-domain` | テンプレートタイプ、名前、バージョン | P2 |
| `new-screen` | テンプレートタイプ、スクリーン名、対象フィーチャー | P2 |
| `init` | プロジェクト初期設定 | P3 |

## 対話フロー例

### `k1s0 new-feature`

```
$ k1s0 new-feature

? テンプレートタイプを選択してください: (↑↓で移動、Enterで決定)
❯ backend-rust   - Rustバックエンドサービス
  backend-go     - Goバックエンドサービス
  frontend-react - Reactフロントエンド
  frontend-flutter - Flutterフロントエンド

? フィーチャー名を入力してください: (kebab-case)
❯ user-management

? 追加オプションを選択してください: (Spaceで選択、Enterで確定)
❯ [x] gRPCサポート (--with-grpc)
  [x] データベース接続 (--with-db)
  [ ] Redisキャッシュ (--with-cache)

✔ フィーチャー 'user-management' を作成しました
```

## 設計方針

### 1. 後方互換性の維持

```bash
# 従来の引数指定方式（CI/CDで使用）は維持
k1s0 new-feature --type backend-rust --name user-management --with-grpc

# 引数不足時に対話モードへ自動フォールバック
k1s0 new-feature
# → 対話モード起動
```

### 2. 明示的な対話モード

```bash
# -i / --interactive フラグで強制的に対話モード
k1s0 new-feature -i
k1s0 new-feature --interactive
```

### 3. 部分指定のサポート

```bash
# 一部引数を指定、不足分のみ対話で補完
k1s0 new-feature --type backend-rust
# → 名前とオプションのみ対話で入力
```

## 注意事項

- **TTY検出**: パイプ入力やCI環境では対話モードを無効化
- **カラー対応**: `--no-color` オプションとの整合性
- **タイムアウト**: 長時間無応答時の処理
- **Ctrl+C**: 中断時のクリーンアップ
