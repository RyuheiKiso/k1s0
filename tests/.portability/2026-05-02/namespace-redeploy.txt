## 制約解析

本セッション host（WSL2 + devcontainer 相当環境）の制約:
  - multipass: nested virtualization 制約で起動不可（host OS 上の VM hypervisor 不可視）
  - kind 2nd cluster: /proc/sys/user/max_inotify_instances=128 が root 専有で枯渇、root 権限なし
  - 結果: run.sh / run-kind.sh はいずれも host kernel 環境で blocked

## 採用した代替経路

同 cluster 上の別 namespace に fresh helm install + 公開 e2e shell test の経路で
「chart の再現性」「manifest の cluster 状態非依存」を最小確認する。
