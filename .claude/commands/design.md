# /design — 技術説明書・設計書の新規作成

`05_実装/` / `04_概要設計/` / `02_構想設計/` 配下に技術説明書・設計書（タイプ D）を新規作成する。

## 引数

$ARGUMENTS

自然な日本語で「何を設計するか」を指定する。配置先を先頭に指定するとより正確。

例:
- `05_実装/60_観測性設計 に SLO-SLI 測定設計`
- `04_概要設計 に コンポーネント間通信方式`
- `STRIDE 脅威モデル拡張`
- `DORA 4keys メトリクス集計設計`

## 作成手順

### Step 0: 前提規約の読み込み

1. `.claude/skills/docs-delivery-principles/SKILL.md` を読む。
2. `.claude/skills/docs-design-spec/SKILL.md` を読み、タイプ D 書式規約を把握する。

### Step 1: 配置先の決定

引数から配置先を判別する。明示指定があればそれに従う。

| 判別基準 | 配置先 | 例 |
|----------|--------|-----|
| 実装段階の具体設計 | `05_実装/<章>/<節>/` | ビルド / CI/CD / 観測性 / リリース / サプライチェーン等 |
| システム/ソフトウェア方式レベル | `04_概要設計/<章>/<節>/` | システム方式 / コンポーネント方式 / IF / 非機能方式 |
| アーキテクチャ深掘り | `02_構想設計/<章>/<節>/` | アーキテクチャ / tier1 設計 / 技術選定 |

`05_実装/` の章一覧:
- 00_ディレクトリ設計 / 10_ビルド設計 / 20_コード生成設計 / 30_CI_CD設計 / 40_依存管理設計 / 50_開発者体験設計 / 60_観測性設計 / 70_リリース設計 / 80_サプライチェーン設計 / 85_Identity設計 / 90_ガバナンス設計 / 95_DXメトリクス / 99_索引

### Step 2: 既存節の確認

配置先ディレクトリを Glob し、既存の節番号を確認する。次番号（例: `40_新規節/`）を使うか、既存節に追記するかを判断する。

### Step 3: ファイル命名

設計書本体は `01_<主題>設計.md` を基本とする（既存パターン）。

例:
- `05_実装/60_観測性設計/40_SLO_SLI定義/01_tier1_公開11API_SLO_SLI.md`
- `05_実装/90_ガバナンス設計/40_脅威モデル_STRIDE/01_STRIDE脅威モデル設計.md`

### Step 4: 骨格の下書き

`docs-design-spec` Skill に従い、以下の骨格を生成する:

```markdown
# <タイトル>

<導入段落: 何を解決するか / 部外者が 1 段落で目的を理解できる水準>

## 1. 背景と目的

## 2. 全体構造

<concept / structure 図を drawio で作成、img/ に配置、svg 埋め込み>

## 3. 処理フロー

<sequence 図を drawio で作成>

## 4. データモデル

<ER 図、該当時のみ>

## 5. 設計判断の根拠

- 関連 ADR: [ADR-XXX-YYY](...)
- 関連要件: [FR-ZZZ / NFR-...](...)

## 6. トレーサビリティ

- 上流: <要件 ID / ADR>
- 下流: <実装 / 他設計 / Runbook>

## 7. 制約と今後の課題
```

### Step 5: 図の作成

- 「概念図 → 構造図 → シーケンス図 → ER 図」の順序で配置。
- 各図は `drawio-authoring` Skill を呼んで作成。
- 複数レイヤが登場する場合は `figure-layer-convention` Skill も併用。
- md と同階層に `img/` を作り、drawio と svg の両方を配置。

### Step 6: トレーサビリティ登録

`05_実装/99_索引/` 配下の該当対応表に ID を登録することをユーザーに案内する:

- `10_ADR対応表/` — 根拠 ADR がある場合
- `20_DS-SW-COMP対応表/` — 概要設計との対応
- `30_NFR対応表/` — 非機能要件との対応
- `40_Backstage_catalog対応/` — Backstage 登録対象の場合

### Step 7: 自己検証

`docs-design-spec` Skill の自己チェックリスト 9 項目を通す。特に「論理レベルで書かれ実装詳細は参照リンクに留められているか」を重点確認。

## 記述ルール

- 本文は日本語
- 論理レベルで記述、実装詳細（Helm values / CI step / 具体コード）は参照リンクで誘導
- 300 行超は分割
- アスキー図・罫線図禁止、drawio svg を使用
