# 図解レイヤ記法規約（ダイジェスト）

複数のレイヤが同一図内に登場する drawio は、読み手がレイヤ差異を誤認しないよう、色・線種・配置の 3 軸で責務レイヤを常に明示的に分離すること。過去に Dapr サイドカー（アプリ層）と Istio サイドカー（ネットワーク層）を同じビジュアルで描き、読み手が責務差異を誤認した事故がある。

## レイヤ基本色

| レイヤ | 塗色 | 枠色 | 性質 |
|--------|------|------|------|
| アプリ層 | `#fff2cc` | `#d79b00` | アプリから明示的に呼び出す抽象（Dapr、tier1-3 各層） |
| ネットワーク層 | `#dae8fc` | `#6c8ebf` | アプリから透過的に効く機構（Istio Ambient、Envoy） |
| インフラ層 | `#f5f5f5` | `#666666` | Kubernetes とノード自体 |
| データ層 | `#e1d5e7` | `#9673a6` | 状態保持コンポーネント（Postgres、Kafka、Valkey、MinIO） |

## 線種

| 線種 | 意味 |
|------|------|
| 実線太 (strokeWidth=2) | 明示的な同期呼び出し（gRPC / HTTP / SDK） |
| 点線細 (dashed) | 透過捕捉（iptables / eBPF / ztunnel） |
| 破線 (紫) | 非同期イベント（Kafka / PubSub） |
| 破線 (橙/緑) | CI/CD / GitOps |

## 凡例ブロック（必須）

全図に凡例ブロックを右下固定で配置する。使ったレイヤと線種のサンプルのみで構成し、全図で同一サイズ（幅 480 / 高さ 100〜180）。枠色 `#666666` / 塗色 `#f5f5f5`。
