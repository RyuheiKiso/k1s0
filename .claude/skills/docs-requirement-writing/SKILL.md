# 要件定義書の記述規約（タイプ A）

`docs/03_要件定義/` 配下で IPA 共通フレーム 2013 準拠の要件定義書を書く時に参照する。k1s0 は採用側組織が自律的に検討する個人開発 OSS のため、要件定義書は「採用側の技術アーキテクトや法務担当が独立して読み、採用判断・運用範囲決定ができる」品質が必須になる。過去に「表セルに要件本体を詰め込んだだけ」のドキュメントが、外部レビューで「意図が読み取れない」と差戻しになった経緯がある。本規約は散文優先の記述スタイルを強制する。

**スコープ外**（別 Skill を使うこと）:

- ADR 起票 → `docs-adr-authoring`
- 概要設計書 / 実装設計書 → `docs-design-spec`
- ポストモーテム → `docs-postmortem`

## 要件本体の記述（三幕構成）

- 各要件の本体を表セルに流し込むことは**禁止**。
- 散文で「**現状 → 要件達成後の世界 → 崩れた時の影響**」を段落で展開する。
- 受け入れ基準や手順など「条件の束」は散文の後の箇条書きで列挙する。
- 表は章末のサマリ（ID 一覧 / 充足度 / 優先度分布）にのみ使う。

例（`FR-STATE-01` の場合）:

> **現状**: tier2 の各ドメインサービスは State を独自に実装しており、トランザクション境界と冪等性の扱いがサービスごとに分岐している。この結果、同じ操作が重複実行された時の挙動が予測できず、採用側の業務監査担当からも「同じボタンを 2 回押した時の保証が不明」との指摘を受けている。
>
> **要件達成後**: tier1 State API が single source of truth となり、tier2 は冪等キーを渡すだけで重複排除・ロールバック・監査ログ取得が自動化される。業務担当者は「ボタンを 2 回押しても 1 件として処理される」を SLA として約束できる。
>
> **崩れた時の影響**: tier1 State API が不在だと、tier2 の各サービスが重複排除ロジックを個別実装することになり、実装工数が 5〜10 人月増、かつ挙動不一致が監査リスクに直結する。

受け入れ基準:
- [ ] 同一冪等キーでの 2 回目以降の要求は 1 回目と同じレスポンスを返す。
- [ ] 失敗時のロールバックが Audit ログに記録される。
- [ ] SLO: 99.95% の可用性（`NFR-A-01` に準拠）。

## 散文と図の必須化

- 各章・各節の冒頭に、何を解決するかの導入段落を必ず置く。
- 表の前後に「なぜこの分類なのか」「どう読むか」「重要な関係性」の散文を添える。
- 重要な概念・関係性・フロー・構造は drawio 図を作成し SVG で埋め込む（`drawio-authoring` Skill を併用）。
- アスキーによる図や表の表現は禁止。

## テーブルの品質基準

ラベル羅列のテーブルは不可。各セルに根拠・数値・出典を入れ、1 行読んで納得できる密度にすること。

- **優先度セル**: 判定理由を同居。
  例:「MUST（採用判断の前提。未達で採用検討者の試算が破綻する）」
- **数値セル**: 根拠を併記。
  例:「48 時間以内（業界平均 72 時間を参考に短縮目標）」
- **期待効果セル**: 前後比較を書く。
  例:「年次ライセンス費 2,000 万円削減、承認期間 3 か月 → 0」
- **事例参照セル**: 事例そのものをセルに入れる（URL やドキュメント名だけでは不十分）。

## トレーサビリティ（必須）

要件は単体で閉じず、必ず以下のリンクを持たせる:

- **上流**: 業務背景 / ステークホルダのニーズ
- **下流**: 実装する ADR / 関連する設計書（`04_概要設計/`、`05_実装/`）
- **索引**: `80_トレーサビリティ/` の要件マトリクスに登録

## 自己チェックリスト

1. [ ] ファイルが `docs/03_要件定義/` 配下の該当章にある。
2. [ ] ID が該当章の ID 体系に準拠し、採番に重複がない。
3. [ ] 各要件の本体が散文で書かれている（表セルに詰め込んでいない）。
4. [ ] 「現状 → 達成後 → 崩れた時」の三幕で段落展開されている。
5. [ ] 章・節の冒頭に導入段落がある。
6. [ ] 表の前後に解説散文がある。
7. [ ] 優先度セル・数値セル・期待効果セルに根拠が同居している。
8. [ ] 事例参照が可能な箇所は事例そのものが書かれている。
9. [ ] 受け入れ基準が箇条書きで列挙されている。
10. [ ] 関連 ADR / 下流設計書へのリンクがあり、`80_トレーサビリティ/` マトリクスを更新した。
